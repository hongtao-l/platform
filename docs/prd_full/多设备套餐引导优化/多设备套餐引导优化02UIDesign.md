# 多设备套餐引导优化 — 页面设计说明

> 关联需求：[多设备套餐引导优化01ReqAnalysis.md](多设备套餐引导优化01ReqAnalysis.md)
> 设计端：CareCam Pro PWA（Vue3 + Vant UI）

---

## 1. 整体架构

```
CareCam Pro PWA
├── composables/useServicesStore.js    — 共享状态（服务数据跨页面同步）
├── mock/services.js                   — 套餐 + 设备 mock 数据
└── views/
    ├── me/index.vue                   — 「我的服务」菜单 + 待绑定标签
    ├── store/index.vue                — 服务商城（购买前选设备 + 支付方式）
    └── my-services/
        ├── index.vue                  — 已购套餐列表（仅显示数量）
        └── bind.vue                   — 设备绑定二级页（ID + 名称）
```

## 4. 状态说明

| 状态 | 触发条件 | 展示内容 |
|------|------|------|
| 加载中 | 数据请求中 | van-skeleton × 3 |
| 正常已绑定 | 套餐已绑定设备 | 显示数量 + 无按钮 |
| 待绑定 | 套餐未绑定/未绑满 | 显示数量 + 「去绑定」按钮 |
| 菜单标签 | unboundCount > 0 | 橙色「待绑定」 |
| 离线设备 | 设备 offline | 不在弹窗中显示离线标识 |
| 已绑定设备 | 已在该套餐下 | 页面直接展示 ✓ |
| 不支持设备 | supportedCategories 不含该类型 | 灰色「不支持」标签，不可勾选 |
| 已开通设备 | 已有同 category 生效中套餐 | 紫色「已开通」标签，不可勾选 |
| 默认选中 | 弹窗打开时 | 仅选中可开通设备（≤ 槽位数） |
| 槽位已满 | 选中数达上限 | 未选设备 opacity:0.45 + disabled |
| 设备排序 | — | 可开通 > 已开通 > 不支持 |
| 多设备购买 | type='multi' | 确认订单弹窗（支付方式 + 绑定设备 + 确认支付） |
| 单设备购买 | type='single' | 直接 Toast「购买成功」 |

## 5. 关键交互流程

### 5.1 服务商城选购流程

```
选择套餐 → 主页面选支付方式 → 点击「立即开通」
  ├── 单设备 → 直接购买成功
  └── 多设备 → 确认订单弹窗（确认设备绑定 → 确认支付）
```

### 5.2 设备切换流程

```
点击顶部设备栏 → 设备选择弹窗
  ├── 单设备套餐：radio 单选
  └── 多设备套餐：checkbox 多选（≤ maxSlots）
→ 点击「完成」→ 设备栏更新
```

### 5.3 支付方式选择

支付方式统一在主页面套餐卡片下方展示。确认订单弹窗内也包含支付方式，供用户弹窗内二次确认/修改。
