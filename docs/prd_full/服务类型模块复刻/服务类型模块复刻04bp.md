## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 目标值 | 监控周期 |
|------|------|:--:|------|------|:--:|
| K01 | 服务分类创建数 | 业务 | 统计周期内新建分类数量 | — | 周 |
| K02 | 服务创建数 | 业务 | 统计周期内新建服务数量 | — | 周 |
| K03 | 服务修改率 | 行为 | 修改服务数 / 总服务数 | — | 月 |
| K04 | 服务删除率 | 行为 | 删除服务数 / 总服务数 | — | 月 |
| K05 | 物模型平均关联数 | 行为 | ∑已选物模型数 / 新建服务数 | — | 周 |
| K06 | 新增服务操作时长 | 质量 | 弹窗打开到保存成功的平均时长 | < 2min | 周 |
| K07 | 服务保存成功率 | 质量 | 保存成功次数 / 保存操作总次数 | > 95% | 日 |
| K08 | 分类筛选使用率 | 行为 | 分类点击次数 / 页面 PV | — | 周 |
| K09 | 批量移动使用率 | 行为 | 批量移动次数 / 页面 PV | — | 月 |

### 1.2 指标详细说明

**K01 服务分类创建数**：衡量运营人员对服务分类管理的活跃度。分类数低说明当前分类已满足需求；持续增长说明业务线扩张。

**K02 服务创建数**：直接反映平台服务类型增长。波动需关注是否有批量导入或人工误操作。

**K03 服务修改率**：高频修改可能说明服务配置不够稳定，或业务需求频繁变更。若修改率持续上升需排查服务定义是否合理。

**K04 服务删除率**：删除率过高需排查是否存在"创建-删除"的无意义操作循环，可能是体验问题。

**K05 物模型平均关联数**：关联数过低说明服务定义过简，可能限制套餐能力；过高说明配置冗余。

**K06 新增服务操作时长**：反映配置效率，时长过长说明物模型选择交互有优化空间。

**K07 服务保存成功率**：异常波动直接关联线上问题，需设置告警。

**K08 分类筛选使用率**：衡量左侧分类面板的实际使用价值。若使用率低，说明分类对用户没有筛选需求。

**K09 批量移动使用率**：衡量批量操作的 ROI，过低可考虑下掉该功能。

## 二、埋点方案

### 2.1 事件清单

| 事件ID | 事件名称 | 触发时机 | event_type | 优先级 |
|------|------|------|:--:|:--:|
| svc_page_view | 服务类型页面浏览 | 进入 `/pkg/service` 页面 | 0 | P0 |
| svc_category_add | 新建分类 | 分类弹窗点击「确定」且保存成功 | 0 | P1 |
| svc_category_edit | 编辑分类 | 分类弹窗点击「确定」且保存成功 | 0 | P1 |
| svc_category_click | 分类筛选 | 点击左侧分类项进行筛选 | 0 | P1 |
| svc_list_view | 服务列表浏览 | 服务列表首次加载完成 | 0 | P0 |
| svc_service_add | 新增服务 | 服务弹窗点击「保存」且创建成功 | 1 | P0 |
| svc_service_edit | 修改服务 | 服务弹窗点击「保存」且更新成功 | 1 | P0 |
| svc_service_delete | 删除服务 | 二次确认后点击「确定」且删除成功 | 0 | P0 |
| svc_batch_move | 批量移动 | 批量移动操作完成 | 1 | P2 |
| svc_tm_add | 添加物模型 | 在双栏穿梭框中点击「添加」 | 0 | P2 |
| svc_tm_remove | 移除物模型 | 在双栏穿梭框中点击「移除」 | 0 | P2 |
| svc_tm_search | 物模型搜索 | 在可选/已选物模型搜索框输入关键词 | 0 | P2 |
| svc_algo_add_event | 添加算法事件 | 算法配置区点击「添加事件」 | 0 | P2 |
| svc_algo_toggle | 算法事件开关 | 算法事件行切换「是否开启」开关 | 0 | P2 |
| svc_save_fail | 服务保存失败 | 保存请求返回错误 | 2 | P0 |
| svc_delete_fail | 服务删除失败 | 删除请求返回错误（如被引用） | 2 | P1 |

### 2.2 公共参数（所有事件必带）

> 公共参数固定为7项，不可增减。

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| event_type | String | 是 | 0=普通事件，1=额外参数事件，2=错误事件 | "0" |
| event_id | String | 是 | 事件ID | "svc_page_view" |
| event_time | Int64 | 是 | 事件发生时间戳(ms) | 1715846400000 |
| user_id | String | 是 | 用户唯一标识 | "admin_001" |
| session_id | String | 是 | 会话ID | "sess_abc123" |
| os_type | String | 是 | Web | "Web" |
| app_version | String | 是 | 平台版本号 | "2.1.0" |

### 2.4 上报方式

```
POST /api/pkg/service/event
```

### 2.5 去重策略

| 场景 | 策略 |
|------|------|
| 页面浏览（svc_page_view） | session_id + event_time 窗口 5s 去重 |
| 成功事件（svc_service_add 等） | service_id + event_time 去重，同 service 同秒只保留首条 |
| 错误事件（svc_save_fail） | user_id + event_id + 1s 窗口去重，避免 Toast 闪现多报 |

### 2.6 数据校验

| 校验项 | 规则 | 异常处理 |
|------|------|------|
| event_type 合法 | 必须为 0/1/2 | 丢弃并记录日志 |
| 公共参数完整 | 7 项全部非空 | 丢弃并告警 |
| service_id 格式 | 纯数字 1-6 位 | 丢弃并记录日志 |
| duration_ms 非负 | >= 0 ms | 丢弃该字段，其余保留 |
