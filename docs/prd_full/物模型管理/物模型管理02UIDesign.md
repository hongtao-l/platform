# 物模型管理 — UI设计

## 1. 设计概述

### 1.1 设计依据
基于 [物模型管理需求分析](./物模型管理01ReqAnalysis.md)，参考 IoT 行业物模型定义规范和 Element Plus 设计体系。

### 1.2 设计目标
- 物模型管理作为独立顶级菜单，下设「能力库」和「IOT类目」两个子页面
- 清晰的 类目→模块→能力 三层数据关系
- 能力库页：类目选择器 + 左栏模块列表 + 右栏能力表格，同屏操作
- 单页表单式能力创建抽屉，能力类型切换联动数据定义区，操作高效直观
- 6 种数据类型的能力配置均有独立编辑器
- 完整的状态反馈：空数据、加载、错误、确认弹窗

## 2. 页面结构

### 2.1 导航菜单

```
左侧导航栏
├── ...
├── 物模型管理              ← 顶级菜单（展开）
│   ├── 能力库              ← 子菜单1
│   └── IOT类目             ← 子菜单2
├── ...
```

物模型管理作为独立顶级菜单组，包含两个子菜单项。点击侧栏子菜单切换页面，两个页面为平级关系（无嵌套导航）。

### 2.2 页面层级

```
侧栏菜单                              右侧内容区
├── 物模型管理                         
│   ├── 能力库 ──────────────→  能力库页（能力管理主操作区）
│   │                              ├── 顶部：类目选择器
│   │                              ├── 左栏 260px：模块列表 + CRUD
│   │                              ├── 右栏：能力表格 + 筛选芯片
│   │                              └── 能力抽屉（单步表单） / 参数弹窗
│   │
│   └── IOT类目 ────────────→  IOT类目页（类目管理）
│                                  ├── 类目表格：名称/标识符/模块数/能力数/描述
│                                  └── 添加/编辑类目弹窗
```

**面包屑**：
- 能力库: `物模型管理 / 能力库`
- IOT类目: `物模型管理 / IOT类目`

**顶部类目选择器：**
- 下拉选择当前编辑的类目

- 固定宽度 260px
- 顶部"能力模块"标题 + [+ 添加] 按钮
- 第一个条目为"全部"（伪模块），默认选中
- 每个模块项显示：模块名称 + 模块标识（灰色小字）

### 3.5 添加/编辑能力 — 单步抽屉

**抽屉宽度：** 720px，内容区可滚动

能力类型切换时，下方「数据定义区」联动更新，所有字段在同一抽屉中完成，无需步骤切换。

```
┌─────────────────────────────────────────────────────┐
│  添加标准能力                                      ✕ │
├─────────────────────────────────────────────────────┤
│                                                      │
│  能力类型 *                                          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐            │
│  │ ● 属性   │ │   服务   │ │   事件   │            │ ← 卡片式选择，切换时下方数据定义区联动
│  └──────────┘ └──────────┘ └──────────┘            │
│                                                      │
│  能力名称 *                                          │
│  ┌──────────────────────────────────┐               │
│  │ 请输入能力名称（2-50字符）         │               │
│  └──────────────────────────────────┘               │
│                                                      │
│  标识符 *                                            │
│  ┌──────────────────────────────────┐               │
│  │ 例如 work_mode                    │               │
│  └──────────────────────────────────┘               │
│  字母开头，仅允许字母、数字、下划线，全局唯一          │
│                                                      │
│  数据类型 *（仅"属性"显示）                           │
│  [ 枚举型 ▼ ]                                       │
│                                                      │
│  读写模式（仅"属性"显示）                             │
│  [ 读写 ▼ ]                                         │
│                                                      │
│  描述                                                │
│  ┌──────────────────────────────────┐               │
│  │ 请输入能力描述                     │               │
│  └──────────────────────────────────┘               │
│                                                      │
│  ─────────────── 数据定义 ───────────────              │ ← 分隔线，以下内容随能力类型/数据类型联动
│                                                      │
│  [ 属性-枚举型 / 属性-数值型 / 属性-布尔型 /           │

│                              [取消]        [提交]    │
```

**能力类型切换行为：**
- 选择"属性" → 显示数据类型 + 读写模式；数据定义区渲染对应编辑器
- 选择"服务" → 隐藏数据类型和读写模式；数据定义区渲染输入/输出参数
- 选择"事件" → 隐藏数据类型和读写模式；数据定义区渲染事件类型 + 输出参数
- 编辑已有能力时切换类型，数据定义区重置（符合 R03）

## 5. 交互说明

### 5.1 页面路由
- 点击侧栏"能力库" → 路由 `/thing-model/capability`，加载能力库页
- 点击侧栏"IOT类目" → 路由 `/thing-model/category`，加载类目页
- 两个页面独立，不会相互跳转

### 5.2 能力库 — 类目切换
- 顶部下拉选择类目，模块列表 + 能力表格联动刷新
- 无类目时显示空状态引导"请先在 IOT类目 中创建设备类目"
- 切换类目后模块选中重置为"全部"

### 5.3 IOT类目 — 删除联动
- 删除类目后，能力库页的类目选择器自动移除该项
- 如果能力库当前正选中被删除的类目，自动切换到第一个类目（或无类目空状态）

### 5.4 能力类型修改
- 切换能力类型后数据定义区联动更新，已填内容会丢失
- 切换时如有已填数据，弹窗 confirm 提示"切换类型将清空当前数据定义，是否继续？"

### 5.5 数据类型联动
- "属性"类型下切换数据类型，数据定义区呈现对应编辑器（枚举/数值/布尔/字符串）
- 服务 → 数据定义区显示输入/输出参数编辑器
- 事件 → 数据定义区显示事件类型选择 + 输出参数编辑器

### 5.6 提交校验
- 点击"提交"统一校验所有字段
- 必填项未填 → 滚动到对应区域并标红提示
- 数值范围不合法 → 提示"最大值必须大于最小值"
- 枚举值为空 → 提示"至少需要一个枚举值"

### 5.7 模块筛选
- 选中模块 → 右栏只显示该模块的能力
- 选中"全部" → 显示所有模块的能力
- 与类型筛选芯片是 AND 关系（先模块筛选，再类型筛选）

### 5.8 删除操作
- 删除类目：计算并提示模块数和能力总数，二次确认
- 删除模块：计算并提示能力数，二次确认，删除后自动跳回"全部"
- 删除能力：确认后直接删除，表格刷新

### 5.9 编辑后刷新
- 能力编辑保存后：能力表格 + 类目列表均刷新（模块数和能力数统计可能变化）
- 模块编辑后：模块列表 + 能力表格刷新
- 类目编辑后：能力库的类目选择器同步更新名称
