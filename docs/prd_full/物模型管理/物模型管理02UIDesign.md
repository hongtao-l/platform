## 1. 设计概述

### 1.1 设计依据
基于 [物模型管理需求分析](./物模型管理01ReqAnalysis.md)，参考 Element Plus 设计体系。

### 1.2 设计目标
- 物模型管理作为独立顶级菜单，下设「能力库」和「IOT类目」两个子页面
- **标准能力**作为独立资源池，类目通过穿梭框批量导入能力实现关联
- 能力库页：纯能力池，左栏模块列表 + 右栏全量能力表格
- 类目页：表格 + 行内导入能力按钮 + 点击能力数跳转详情
- 类目能力详情页：二级页面，返回栏 + 模块侧栏 + 能力表格 + 导入/编辑/移除

> 能力库为纯能力池，展示全量能力，不再包含类目选择器和导入按钮。

**标签页「标准能力」：**
- 置于模块侧栏上级，标明层级关系

**左栏（模块侧栏）：**
- 固定宽度 260px
- 顶部"能力模块"标题 + [+ 添加] 按钮
- 第一个条目为"全部"，默认选中
- 每个模块项显示：模块名称 + 模块标识（灰色小字 PascalCase）
- hover 时显示编辑/删除图标按钮
- 选中态：蓝色高亮背景
- 选中模块 → 右栏按模块筛选能力

**右栏能力表格列定义：**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 类型 | 80px | 彩色 tag：属性(蓝) / 服务(绿) / 事件(橙) |
| 能力名称 | auto | 粗体深色文字 |
| 能力标识 | 170px | 灰色等宽字体 PascalCase |
| 数据类型 | 100px | 枚举型/数值型/布尔型/字符串/服务/事件 |
| 数据定义 | auto | 简要说明，如"低功耗(0), AOV(1)..." |
| 描述 | auto | 截断显示 |
| 所属模块 | 130px | 模块名称 |
| 操作 | 150px | 编辑 / 删除 |

**右栏操作按钮：**
- 「添加标准能力」：创建新能力，自动归属当前侧栏选中模块
- 无导入按钮（导入功能在类目页和类目详情页）

**左侧面板（可选能力）：**
- 搜索栏位于面板内部 header 下方
- 按模块分组，每组可折叠/展开（默认展开）
- 模块标题行：展开箭头 + **全选复选框** + 模块名(标识符) + 能力计数
- 每个能力条目：复选框 + [类型标签] + 能力名称 + 标识符（灰色小字）
- 搜索支持按名称和标识符过滤
- 模块全选复选框：全选/全不选可见能力，有部分选中时显示半选态
- 已关联到当前类目 + 已加入待导入的能力不显示

**中间按钮：**
- `→` 将左侧勾选的能力移入右侧待导入
- `←` 清空右侧待导入

**右侧面板（待导入）：**
- 按模块分组，可折叠/展开（新增时自动展开对应模块）
- 每个能力条目：`[类型tag] 名称 标识符 ×`
- 标识符以灰色等宽小字显示
- 点击 × 移除单个能力
- 无归属模块的能力归入"其他"分组

## 5. 交互说明

### 5.1 页面路由
- 点击侧栏"能力库" → 路由 `/thing-model/capability`
- 点击侧栏"IOT类目" → 路由 `/thing-model/category`
- 点击类目行"标准能力"数 → 路由 `/thing-model/category/:id/capabilities`（类目能力详情页）
- 详情页返回 → 路由回到 `/thing-model/category`

### 5.2 能力库 — 模块切换
- 选中模块 → 右栏能力表格按模块筛选
- 选中"全部" → 显示全量能力
- 模块支持 CRUD

### 5.3 导入能力 — 穿梭框
- 从类目页或类目详情页打开
- 左侧模块默认全部展开
- 勾选能力或模块全选后点击 → 移入右侧
- 右侧对应模块自动展开
- 搜索实时过滤左侧能力（按名称+标识符）
- 确认导入后关联建立，表格刷新

### 5.4 添加能力 — 模块自动归属
- 新增时自动取侧栏当前选中模块
- 若侧栏选中"全部"，取第一个模块
- 编辑时模块不可更改

### 5.5 删除操作
- 删除类目：自动解绑关联能力，能力不删除
- 删除模块：同步删除模块下所有能力及关联
- 删除能力：同步删除所有类目与该能力的关联
- 移除类目能力关联：解绑但能力不删除
