## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 落地页曝光UV | 行为 | 进入服务介绍开通页的去重用户数 | 日 |
| 2 | 套餐选择率 | 行为 | 选择套餐UV / 落地页曝光UV | 日 |
| 3 | 支付转化率 | 业务 | 支付成功UV / 落地页曝光UV | 日 |
| 4 | 开通成功率 | 质量 | 支付成功UV / 支付点击UV | 日 |
| 5 | ARPU | 业务 | 总收入 / 支付成功UV | 月 |
| 6 | AI搜索使用率 | 行为 | 使用AI搜索UV / 已激活设备UV | 周 |
| 7 | AI日报生成率 | 行为 | 生成日报UV / 已激活设备UV | 周 |
| 8 | 日报生成成功率 | 质量 | 日报生成成功次数 / 日报生成请求次数 | 日 |
| 9 | 关注事件人均添加数 | 行为 | 用户添加关注事件总数 / 已开通用户数 | 月 |
| 10 | 设备激活率 | 增长 | 已激活设备数 / 总设备数 | 周 |
| 11 | 续费率 | 业务 | 次月续费用户数 / 应续费用户数 | 月 |
| 12 | 内联引导转化率 | 业务 | 从内联引导进入落地页后支付成功UV / 内联引导点击UV | 周 |

### 1.2 核心指标说明

**落地页曝光UV**：衡量服务介绍开通页的流量入口效果。反映首页AI入口、消息页"+关注"、子功能内联引导等入口的引流能力。

**套餐选择率**：衡量两个套餐（月付/年付）的吸引力。用户在落地页选择套餐的比例，反映套餐展示效果和定价合理性。

**支付转化率**：衡量落地页 → 支付的完整转化漏斗，是评估落地页效果和套餐吸引力的核心指标。

**开通成功率**：衡量支付环节的技术稳定性。正常应 > 95%，低于 80% 需排查支付通道问题。

**AI搜索使用率 / AI日报生成率**：衡量已开通用户的功能黏性，反映AI功能是否真正满足了用户需求。搜索使用率基准值约 30-40%，日报生成率约 20-30%。

**内联引导转化率**：衡量"不锁死页面"策略是否有效。内联引导以非阻断方式引导开通，其转化率对比全屏锁死方式是评估交互策略的核心指标。

**关注事件人均添加数**：反映用户对自定义关注的需求深度。默认4个，用户可额外添加，人均添加数 > 1 说明用户有强烈的个性化需求。

**设备激活率**：衡量用户开通后实际激活使用的比例，反映功能渗透率。

## 二、埋点方案

### 2.1 事件清单

| 事件ID | 事件名称 | 触发时机 | event_type | 优先级 |
|------|------|------|------|------|
| page_view_landing | 落地页浏览 | 服务介绍开通页加载完成，首屏可见 | 0（普通事件） | P0 |
| plan_select | 套餐选择 | 用户点击选择套餐卡片 | 0（普通事件） | P0 |
| pay_start | 支付调起 | 调起微信/支付宝支付 | 1（额外参数） | P0 |
| pay_success | 支付成功 | 收到支付成功回调 | 1（额外参数） | P0 |
| pay_fail | 支付失败 | 支付失败或用户取消支付 | 2（错误事件） | P0 |
| ai_search_input | AI搜索执行 | 用户输入搜索文本并执行搜索 | 0（普通事件） | P0 |
| daily_generate_click | 日报生成 | 用户点击"生成日报"按钮 | 0（普通事件） | P0 |
| focus_event_add | 关注事件添加 | 用户添加新的关注事件 | 0（普通事件） | P0 |
| home_ai_entry_click | 首页AI入口 | 用户点击首页设备卡片AI按钮 | 0（普通事件） | P1 |
| inline_guide_click | 内联引导点击 | 用户点击内联引导"立即开通全能AI"按钮 | 0（普通事件） | P1 |
| smart_push_toggle | 智能推送开关 | 用户切换智能推送总开关 | 0（普通事件） | P1 |

### 2.2 公共参数（所有事件必带）

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| event_type | String | 是 | 0=普通事件，1=额外参数事件，2=错误事件 | "0" |
| event_id | String | 是 | 事件ID（对应2.1中的事件ID） | "plan_select" |
| event_time | Int64 | 是 | 事件发生时间戳(ms) | 1716192000000 |
| user_id | String | 是 | 用户唯一标识 | "usr_123456" |
| session_id | String | 是 | 会话ID | "sess_abc123" |
| os_type | String | 是 | iOS / Android | "Android" |
| app_version | String | 是 | APP版本号 | "2.1.0" |

### 2.4 上报方式

APP 端在页面浏览、点击交互、业务转化等关键节点直接上报事件至后台：

```
POST /api/ai/event
```

### 2.5 去重策略

| 场景 | 策略 |
|------|------|
| 同 user_id + event_id 重复 | 1分钟内保留第一条 |
| 页面浏览重复上报 | 同一会话 + 同一 page_view 事件，仅保留首次 |
| pay_success 重复 | 同一 order_id 只保留第一条 |
| 快速切换设备 | device_switch 事件 2 秒内去重 |

### 2.6 数据校验

| 校验项 | 规则 | 异常处理 |
|------|------|------|
| 必填参数 | 公共7项 + 业务参数不得为空 | 丢弃并告警 |
| event_type | ∈ {0, 1, 2} | 标记异常 |
| plan_type | ∈ {monthly, yearly} | 标记异常 |
| pay_method | ∈ {wechat, alipay} | 标记异常 |
| os_type | ∈ {iOS, Android} | 标记异常 |
| fail_reason | ∈ {insufficient_balance, user_cancel, network_error, other} | 标记异常 |
| source | ∈ {home_ai_btn, msg_focus, ai_search_inline, ai_daily_inline} | 标记异常 |
| page | ∈ {ai_search, ai_daily} | 标记异常 |
| action | ∈ {on, off} | 标记异常 |
| 时序校验 | pay_success.event_time ≥ pay_start.event_time | 标记异常 |
| 重复事件 | user_id + event_id 1min去重 | 保留第一条 |
| 重复订单 | pay_success order_id去重 | 保留第一条 |

## 五、转化漏斗分析

### 5.1 主漏斗

```
落地页曝光 (100%)
    ↓
选择套餐 (~68%)
    ↓
点击支付 (~45%)
    ↓
支付成功 (~32%)
    ↓
设备激活 (~30%)
```

### 5.2 分入口漏斗

| 入口 | 曝光占比 | 选择率 | 支付转化率 | 特点 |
|------|------|------|------|------|
| 首页AI入口 | 45% | 72% | 35% | 主动探索，转化高 |
| 消息"+关注" | 30% | 65% | 30% | 功能驱动，转化中 |
| AI搜索内联 | 15% | 58% | 25% | 体验后转化，质量高 |
| AI日报内联 | 10% | 62% | 28% | 需求驱动，转化中高 |

### 5.3 套餐偏好分析

| 套餐 | 选择占比 | 支付成功率 | 续费率 |
|------|------|------|------|
| 月付 | ~70% | 96% | 65% |
| 年付 | ~30% | 94% | —（买断制） |

## 六、功能使用效果评估

### 6.1 功能健康度基准

| 指标 | 优秀 | 良好 | 需优化 |
|------|------|------|------|
| 支付转化率 | > 15% | 10-15% | < 10% |
| 支付成功率 | > 98% | 95-98% | < 95% |
| AI搜索使用率（周活） | > 40% | 25-40% | < 25% |
| AI日报生成率（周活） | > 30% | 20-30% | < 20% |
| 关注事件人均添加数 | > 2 | 1-2 | < 1 |
| 内联引导转化率 | > 10% | 5-10% | < 5% |
| 设备激活率 | > 60% | 40-60% | < 40% |
| 月付续费率 | > 70% | 55-70% | < 55% |

### 6.2 功能迭代评估标准

| 评级 | 条件 |
|------|------|
| S（卓越） | 支付转化率 > 18% + 搜索/日报使用率 > 40% |
| A（优秀） | 支付转化率 > 15% + 搜索/日报使用率 > 30% |
| B（良好） | 各指标均在基准良好线以上 |
| C（一般） | 1-2 项指标低于良好线 |
| D（待优化） | 3 项及以上指标低于良好线 |
