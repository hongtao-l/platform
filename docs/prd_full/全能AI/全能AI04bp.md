## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 落地页曝光UV | 行为 | 进入服务介绍开通页的去重用户数 | 日 |
| 2 | 套餐选择率 | 行为 | 选择套餐UV / 落地页曝光UV | 日 |
| 3 | 支付转化率 | 业务 | 支付成功UV / 落地页曝光UV | 日 |
| 4 | 开通成功率 | 质量 | 支付成功UV / 支付点击UV | 日 |
| 5 | ARPU | 业务 | 总收入 / 支付成功UV | 月 |
| 6 | AI搜索使用率 | 行为 | 使用AI搜索UV / 已激活设备UV | 周 |
| 7 | AI日报生成率 | 行为 | 生成日报UV / 已激活设备UV | 周 |
| 8 | 日报生成成功率 | 质量 | 日报生成成功次数 / 日报生成请求次数 | 日 |
| 9 | 关注事件人均添加数 | 行为 | 用户添加关注事件总数 / 已开通用户数 | 月 |
| 10 | 设备激活率 | 增长 | 已激活设备数 / 总设备数 | 周 |
| 11 | 续费率 | 业务 | 次月续费用户数 / 应续费用户数 | 月 |
| 12 | 内联引导转化率 | 业务 | 从内联引导进入落地页后支付成功UV / 内联引导点击UV | 周 |

### 1.2 核心指标说明

**落地页曝光UV**：衡量服务介绍开通页的流量入口效果。反映首页AI入口、消息页"+关注"、子功能内联引导等入口的引流能力。

**套餐选择率**：衡量两个套餐（月付/年付）的吸引力。用户在落地页选择套餐的比例，反映套餐展示效果和定价合理性。

**支付转化率**：衡量落地页 → 支付的完整转化漏斗，是评估落地页效果和套餐吸引力的核心指标。

**开通成功率**：衡量支付环节的技术稳定性。正常应 > 95%，低于 80% 需排查支付通道问题。

**AI搜索使用率 / AI日报生成率**：衡量已开通用户的功能黏性，反映AI功能是否真正满足了用户需求。搜索使用率基准值约 30-40%，日报生成率约 20-30%。

**内联引导转化率**：衡量"不锁死页面"策略是否有效。内联引导以非阻断方式引导开通，其转化率对比全屏锁死方式是评估交互策略的核心指标。

**关注事件人均添加数**：反映用户对自定义关注的需求深度。默认4个，用户可额外添加，人均添加数 > 1 说明用户有强烈的个性化需求。

**设备激活率**：衡量用户开通后实际激活使用的比例，反映功能渗透率。

## 二、埋点方案

### 2.1 事件清单

#### 2.1.1 页面浏览埋点

| 页面名称 | 触发时机 | 上报参数 |
|------|------|------|
| 服务介绍开通页 | 页面加载完成，首屏可见 | user_id, source（入口来源：home_ai_btn / msg_focus / ai_search_inline / ai_daily_inline） |
| AI配置页 | 页面加载完成 | user_id, activated_device_count（已激活设备数） |
| AI搜索页 | 页面加载完成 | user_id, device_id, device_activated（是否已激活） |
| AI日报页 | 页面加载完成 | user_id, device_id, device_activated, selected_date |
| 消息页 | 页面加载完成 | user_id, device_id, device_activated |

| 事件名称 | 触发时机 | 所属页面 | 上报参数 | 优先级 |

| agreement_dialog_show | 协议确认弹窗弹出 | 服务介绍开通页 | — | P1 |
| agreement_dialog_agree | 点击弹窗"同意" | 协议确认弹窗 | — | P1 |
| agreement_dialog_cancel | 点击弹窗"取消" | 协议确认弹窗 | — | P1 |
| ai_search_input | 用户执行AI搜索 | AI搜索页 | device_id, query_length（搜索文本长度） | P0 |
| guess_click | 用户点击"猜你想搜"标签 | AI搜索页 | device_id, guess_text | P1 |

| daily_regenerate_click | 用户点击"重新生成" | AI日报页 | device_id, selected_date | P1 |
| daily_history_open | 用户打开历史日报弹窗 | AI日报页 | device_id | P1 |
| daily_history_select | 用户选择某条历史日报 | 历史日报弹窗 | device_id, history_date | P1 |
| focus_event_add | 用户添加关注事件 | AI配置页 | event_name, current_count（当前事件数） | P0 |
| focus_event_edit | 用户编辑关注事件 | AI配置页 | event_name | P1 |
| focus_event_delete | 用户删除关注事件 | AI配置页 | event_name | P1 |
| focus_event_toggle | 用户启停关注事件 | AI配置页 | event_name, action（enable/disable） | P1 |
| smart_push_toggle | 用户切换智能推送开关 | AI配置页 | action（on/off） | P1 |
| device_switch | 用户在AI子页面切换设备 | AI搜索/AI日报/消息 | from_device, to_device, page | P1 |
| msg_filter_tab_click | 用户点击消息筛选标签 | 消息页 | device_id, tab_key, device_activated | P1 |
| focus_btn_click | 用户点击"+关注" | 消息页 | device_id, device_activated | P0 |
| home_ai_entry_click | 用户点击首页设备卡片AI入口 | 首页 | device_id, device_activated | P0 |
| inline_guide_click | 用户点击内联引导"立即开通全能AI" | AI搜索/AI日报 | page, device_id | P0 |

#### 2.1.3 业务转化埋点

| 事件名称 | 触发时机 | 上报参数 | 优先级 |
|------|------|------|------|
| pay_start | 调起支付 | plan_type, pay_method, amount | P0 |
| pay_success | 收到支付成功回调 | plan_type, amount, order_id | P0 |
| pay_fail | 支付失败或取消 | plan_type, fail_reason（insufficient_balance / user_cancel / network_error / other） | P0 |
| activation_complete | 开通完成（含支付成功后激活） | plan_type, activated_device_id | P0 |

### 2.2 公共参数（所有事件必带）

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| user_id | String | 是 | 用户唯一标识 | "u_123456" |
| device_id | String | 是 | 当前操作关联的设备标识 | "frontdoor" |
| timestamp | Int64 | 是 | 事件发生时间戳(ms) | 1716192000000 |
| app_version | String | 是 | APP版本号 | "1.2.0" |
| platform | String | 是 | iOS / Android | "android" |
| session_id | String | 是 | 会话ID | "sess_abc123" |

### 2.3 上报方式

APP 端在页面浏览、点击交互、业务转化等关键节点直接上报事件至后台：

```
POST /api/ai/event
```

### 2.4 去重策略

| 场景 | 策略 |
|------|------|
| 同 user_id + event_id 重复 | 1分钟内保留第一条 |
| 页面浏览重复上报 | 同一会话 + 同一页面，仅保留首次 |
| pay_success 重复 | 同一 order_id 只保留第一条 |
| 快速切换设备 | device_switch 事件 2 秒内去重 |

### 2.5 数据校验

| 校验项 | 规则 | 异常处理 |
|------|------|------|
| 必填参数 | 公共6项 + 业务参数不得为空 | 丢弃并告警 |
| plan_type | ∈ {monthly, yearly} | 标记异常 |
| pay_method | ∈ {wechat, alipay} | 标记异常 |
| platform | ∈ {ios, android} | 标记异常 |
| fail_reason | ∈ {insufficient_balance, user_cancel, network_error, other} | 标记异常 |
| 时序校验 | pay_success.time ≥ pay_start.time | 标记异常 |
| 重复事件 | user_id + event_id 1min去重 | 保留第一条 |
| 重复订单 | pay_success order_id去重 | 保留第一条 |

## 五、转化漏斗分析

### 5.1 主漏斗

```
落地页曝光 (100%)
    ↓
选择套餐 (~68%)
    ↓
点击支付 (~45%)
    ↓
支付成功 (~32%)
    ↓
设备激活 (~30%)
```

### 5.2 分入口漏斗

| 入口 | 曝光占比 | 选择率 | 支付转化率 | 特点 |
|------|------|------|------|------|
| 首页AI入口 | 45% | 72% | 35% | 主动探索，转化高 |
| 消息"+关注" | 30% | 65% | 30% | 功能驱动，转化中 |
| AI搜索内联 | 15% | 58% | 25% | 体验后转化，质量高 |
| AI日报内联 | 10% | 62% | 28% | 需求驱动，转化中高 |

### 5.3 套餐偏好分析

| 套餐 | 选择占比 | 支付成功率 | 续费率 |
|------|------|------|------|
| 月付 ¥9.9 | ~70% | 96% | 65% |
| 年付 ¥99 | ~30% | 94% | —（买断制） |

## 六、功能使用效果评估

### 6.1 功能健康度基准

| 指标 | 优秀 | 良好 | 需优化 |
|------|------|------|------|
| 支付转化率 | > 15% | 10-15% | < 10% |
| 支付成功率 | > 98% | 95-98% | < 95% |
| AI搜索使用率（周活） | > 40% | 25-40% | < 25% |
| AI日报生成率（周活） | > 30% | 20-30% | < 20% |
| 关注事件人均添加数 | > 2 | 1-2 | < 1 |
| 内联引导转化率 | > 10% | 5-10% | < 5% |
| 设备激活率 | > 60% | 40-60% | < 40% |
| 月付续费率 | > 70% | 55-70% | < 55% |

### 6.2 功能迭代评估标准

| 评级 | 条件 |
|------|------|
| S（卓越） | 支付转化率 > 18% + 搜索/日报使用率 > 40% |
| A（优秀） | 支付转化率 > 15% + 搜索/日报使用率 > 30% |
| B（良好） | 各指标均在基准良好线以上 |
| C（一般） | 1-2 项指标低于良好线 |
| D（待优化） | 3 项及以上指标低于良好线 |
