# 全能AI — 需求分析

## 1. 业务背景与目标

### 1.1 背景
全能AI 是安防摄像头的智能看护管家功能。通过 AI 视觉识别，摄像头能"看懂"画面中的事件（人、宠物、车辆、包裹、异常行为等），并根据用户设定的关注偏好，主动推送有意义的事件消息。

**用户痛点**：
- 传统摄像头产生大量无效告警（风吹草动、光线变化等），用户消息疲劳
- 用户无法快速定位关键事件（如"奶奶今天出门了吗"），只能逐条翻看录像
- 缺乏每日汇总，用户需要手动查看才能了解家中全天动态

### 1.2 目标
- 通过 AI 识别降低无效推送，让用户只收到关心的内容
- 提供自然语言搜索，降低事件查找门槛
- 自动生成日报，让用户一眼掌握家中全天情况
- 按月订阅或按年买断（具体价格由运营配置），按设备维度激活

## 2. 角色与职责

| 角色 | 职责 |
|------|------|
| 终端用户 | 浏览服务介绍、选择套餐、完成购买、管理关注事件 |
| 终端用户 | 在已激活设备上使用 AI消息、AI日报、AI搜索 |
| 运营管理员 | 监控转化漏斗、功能使用数据、收入情况 |
| 运营管理员 | 配置套餐价格、管理支付通道 |

## 3. 名词解释

| 术语 | 说明 |
|------|------|
| **全能AI** | 摄像头智能看护管家功能，包含 AI消息推送、AI日报、AI搜索、关注事件管理 |
| **设备激活** | 对单台设备开通全能AI服务的行为，激活后该设备可使用全部AI功能 |
| **已激活设备** | 已开通全能AI服务的设备 |
| **未激活设备** | 未开通全能AI服务的设备 |
| **关注事件** | 用户自定义的AI识别事件类型（如老人看护、宠物活动），AI据此过滤和推送 |
| **基础事件** | 设备固有的人形侦测、运动侦测等非AI事件类型 |
| **AI消息推送** | 摄像头识别画面中的人、宠物、车辆等，筛选关注事件后推送通知 |
| **AI日报** | 每日自动生成的图文报告，汇总当天关注事件完整时间线 + AI一句总结 |
| **AI搜索** | 通过自然语言描述搜索历史事件记录 |
| **智能推送开关** | 控制是否仅推送关注事件的全局开关，关闭后推送所有事件 |
| **连续订阅** | 按月付费，到期自动续费，可随时取消 |
| **买断制** | 按年付费，一次性购买，到期不自动续费 |

## 4. 核心业务流程

### 4.1 开通全能AI流程
```
用户进入落地页 → 浏览服务介绍
  → 选择套餐（月付/年付）
  → 选择支付方式（微信/支付宝）
  → 勾选协议
  → 点击"立即开通"
  → 确认支付（调起支付）
  → 支付成功 → 开通完成
  → 后续在配置页可管理各设备的激活状态
```

### 4.2 设备切换行为流程
```
用户进入AI搜索/AI日报/AI消息页面
  → 页面默认选中第一台已激活设备
  → 用户切换设备
    → 已激活设备：正常显示功能内容
    → 未激活设备：保留框架UI，内容区显示内联开通引导
    → 不弹Toast、不全屏锁死
```

### 4.3 AI日报生成流程
```
用户进入AI日报页 → 选择设备（默认当日）
  → 已激活设备 → 点击"生成日报"（每日最多3次）
  → 逐步展示生成进度：分析关注事件 → 整理时间线 → AI生成总结
  → 生成完成 → 展示事件汇总卡片（最早→…→最晚）+ AI总结
  → 每日20:00自动生成（次数允许时，按设备本地时区）
```

### 4.4 消息筛选流程
```
用户进入消息页 → 默认显示第一台设备 + 当天消息
  → 切换设备 → 自动重置筛选标签为"全部"
  → 已激活设备：显示全部/关注事件标签/其他
  → 未激活设备：显示全部/运动侦测/人形侦测
  → 点击"+关注" → 根据设备激活状态跳转不同页面
```

## 6. 详细场景分析

### 6.1 正常场景

#### S01: 用户首次开通全能AI
1. 用户从首页设备卡片点击"AI"入口
2. 进入服务介绍开通页，浏览Banner和服务介绍
3. 查看两个套餐：月付（连续订阅）和年付（买断制），具体价格由运营配置
4. 选择月付套餐，下方显示续费提示
5. 选择微信支付
6. 勾选协议（含自动续费协议）
7. 点击"立即开通"
8. 调起支付 → 支付成功 → 开通完成
9. 跳转AI配置页，显示"✓ 守护中"状态

#### S02: 用户管理关注事件
1. 已开通用户进入AI配置页
2. 查看4个默认关注事件（儿童守护、包裹送达、厨房守护、宠物守护）
3. 关闭不需要的"包裹送达"
4. 点击"添加"按钮，输入"老人看护" + 描述
5. 保存成功，列表显示5个关注事件
6. 智能推送开关保持开启

#### S03: 用户在AI搜索页查询
1. 用户进入AI搜索页
2. 默认选中第一台已激活设备
3. 看到"猜你想搜"推荐标签
4. 点击"奶奶今天出门了吗"标签
5. 自动填入并搜索，显示"共找到 3 条相关事件"
6. 点击某条结果进入事件详情

#### S04: 用户生成AI日报
1. 用户进入AI日报页
2. 默认选中第一台已激活设备，仅支持当日日报
3. 点击"生成日报"（每日最多3次）
4. 逐步展示进度：分析关注事件 → 整理时间线 → AI生成总结
5. 生成完成，显示事件汇总卡片（最早→…→最晚）+ AI总结
6. 事件汇总包含关注事件（人员经过、宠物活动、告警等）+ "其他"类别
7. 底部显示AI免责声明"本功能由AI大模型提供，个别结果可能存在误差"

#### S05: 内联引导 — 未激活设备
1. 用户在AI搜索页切换到未激活设备
2. 设备筛选栏和搜索输入区保持可见
3. 内容区显示内联引导（AI搜索图标 + 描述 + "立即开通全能AI"按钮）
4. 不弹Toast、不全屏锁死
5. 点击按钮跳转服务介绍开通页

#### S06: 消息页事件筛选
1. 用户进入消息页
2. 默认选中有激活设备，标签栏显示：全部、老人看护、宠物活动、人员经过、其他
3. 切换设备为未激活设备 → 标签自动重置为"全部"，标签栏变为：全部、运动侦测、人形侦测
4. 点击"+关注" → 根据设备状态跳转不同页面

### 6.2 异常场景

#### E01: 网络异常
| 场景 | 表现 |
|------|------|
| 加载落地页时断网 | 显示已有静态内容；购买按钮置灰或提示"网络异常，请稍后重试" |
| 支付过程中断网 | 弹窗提示"网络异常，支付结果请稍后在订单中查看" |
| AI日报生成中断网 | 保留当前已生成的步骤结果，提示重试 |
| AI搜索请求断网 | 提示"网络异常，搜索未完成，请重试" |

#### E02: 数据为空
| 场景 | 表现 |
|------|------|
| 搜索无结果 | "未找到相关事件，试试其他描述" |
| 当日无日报 | 显示"今日日报尚未生成"，提示点击下方按钮生成 |
| 消息页当天无消息 | "暂无消息" |
| 历史日报列表为空 | 隐藏历史日报入口图标 |

#### E03: 操作失败
| 场景 | 表现 |
|------|------|
| 支付失败 | 弹窗提示失败原因，可返回重试 |
| 开通失败 | 弹窗提示"开通失败，请稍后重试"，保留已选套餐信息 |
| 关注事件添加失败 | Toast提示"添加失败，请重试" |
| 日报生成失败 | 提示"生成失败，请重试"，保留生成按钮 |

#### E04: 权限与限制
| 场景 | 表现 |
|------|------|
| 关注事件达上限（10个） | 点击添加 → Toast"最多添加10个关注事件" |
| 日报次数用完 | 底部提示"今日生成次数已用完（每日最多3次），明天自动重置" |
| 未勾选协议点支付 | 弹窗"请先阅读并同意云服务协议" |

## 7. 数据模型

### 7.1 全能AI服务 (AIService)
```javascript
{
  userId: String,          // 用户ID
  status: String,          // active | expired | cancelled
  planType: String,        // 'monthly' | 'yearly'
  startTime: String,       // 开通时间
  endTime: String,         // 到期时间
  activatedDevices: [String], // 已激活设备ID列表
  orderId: String,         // 订单ID
}
```

### 7.2 关注事件 (FocusEvent)
```javascript
{
  id: String,              // 事件ID
  userId: String,          // 用户ID
  name: String,            // 事件名称
  description: String,     // 事件描述
  enabled: Boolean,        // 启停状态
  isDefault: Boolean,      // 是否系统默认
  sortOrder: Number,       // 排序序号
  createdAt: String,       // 创建时间
}
```

### 7.3 AI日报 (DailyReport)
```javascript
{
  id: String,              // 日报ID
  userId: String,          // 用户ID
  deviceId: String,        // 设备ID
  date: String,            // 日期 YYYY-MM-DD（仅当日，不支持历史日期）
  events: [Object],        // 事件汇总列表（关注事件 + "其他"）
  summary: String,         // AI总结文本
  generatedAt: String,     // 生成时间
  genCount: Number,        // 当日已生成次数（上限3次）
}
```

### 7.4 实体关系
```
用户 1──N 设备
设备 1──1 AI激活状态
设备 1──N 消息
用户 1──N 关注事件（跨设备共享）
关注事件 1──1 启停状态
设备 1──N AI日报（每日每设备最多1份）
```

## 8. 业务规则

### 8.1 设备激活规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 设备级激活 | 全能AI服务按设备维度激活，非全局开关；每台设备独立控制 |
| R02 | 默认设备 | 包含AI功能的页面默认选中第一台已激活设备 |
| R03 | 未激活引导 | 未激活设备在各AI子功能页面显示内联开通引导，不弹Toast、不全屏锁死 |
| R04 | 激活联动 | 设备激活状态影响：消息筛选标签、首页AI入口文案、AI各子页面内容区 |

### 8.2 关注事件规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R05 | 默认事件 | 开通后系统预设4个关注事件：儿童守护、包裹送达、厨房守护、宠物守护 |
| R06 | 数量上限 | 用户最多添加10个关注事件（含默认），达到上限后Toast提示 |
| R07 | 智能推送 | 智能推送开关默认开启；开启后仅推送关注事件消息 |
| R08 | 事件操作 | 每个关注事件支持：启停开关、编辑名称和描述、删除 |

### 8.3 套餐规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R09 | 两个选项 | 套餐A：按月连续订阅；套餐B：按年买断制。权益完全相同，具体价格由运营配置 |
| R10 | 连续订阅提示 | 选中月付时，下方显示自动续费提示文案 |
| R11 | 协议联动 | 月付协议含"自动续费协议"，年付仅含"云服务协议" |
| R12 | 协议前置 | 未勾选协议点击支付 → 弹窗提示 → 取消或同意并继续 |
| R13 | 支付方式 | 仅支持微信支付和支付宝 |

### 8.4 消息筛选规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R14 | 标签差异化 | 已激活设备：关注事件标签在前，基础事件在后 |
| R15 | 标签默认 | 未激活设备：仅基础事件标签（全部、运动侦测、人形侦测） |
| R16 | 切换重置 | 切换设备时自动重置筛选标签为"全部" |
| R17 | 无全部设备 | 设备筛选为单选，无"全部设备"选项 |
| R18 | 横向滚动 | 标签栏超出屏幕时支持横向滚动 |

### 8.5 AI日报规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R19 | 仅当日 | 日报仅支持当日生成，不支持日期选择和历史日期切换 |
| R20 | 生成进度 | 分三步展示：分析关注事件 → 整理时间线 → AI生成总结 |
| R21 | 历史入口 | 已激活设备右上角显示历史日报入口（时钟图标），仅展示近7日；未激活设备隐藏 |
| R22 | 生成次数 | 每日每台设备最多生成3次（按日期+设备维度计数，localStorage持久化） |
| R23 | 自动生成 | 每日20:00在次数允许时自动生成日报（按设备本地时区，国内用北京时间、海外用当地时区；检查autoGenKey防重复触发） |
| R24 | 事件汇总 | 每个关注事件显示最早和最晚一条事件描述，中间省略号；其他事件归入"其他" |
| R25 | 次数重置 | 离开页面时重置当日生成次数（支持演示场景） |
| R26 | 免责声明 | 日报底部固定显示"本功能由AI大模型提供，个别结果可能存在误差" |
