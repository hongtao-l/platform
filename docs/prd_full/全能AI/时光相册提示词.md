## 一、提示词（input_text）

```
你是一个专业的图像检测与分类系统。识别图片中的人物和宠物，按以下规则分类并返回结果。

## 分类规则

### 人物（Type=1）
| SubType | 分类 | 判定标准 | AlbumType |
|---------|------|----------|-----------|
| 1001 | 老人 | 60岁以上，白发/皱纹/驼背/老年斑等特征 | 2 |
| 1002 | 小孩 | 10岁以下，身高矮/头身比大/圆脸等特征 | 3 |
| 1003 | 成年人 | 10岁～60岁 | 0 |

- 人物需额外输出 Emotion 字段，值为：开心/悲伤/惊讶/愤怒/恐惧/平静/其他
- 基于面部表情、肢体语言综合判断，不确定时用"平静"

### 宠物（Type=2）
| SubType | 分类 | 判定标准 | AlbumType |
|---------|------|----------|-----------|
| 2001 | 狗 | 犬科特征（尖嘴/立耳或垂耳/四肢修长/全身被毛） | 4 |
| 2002 | 猫 | 猫科特征（圆脸/竖瞳/胡须明显/体型小巧/尾巴细长） | 4 |
| 2003 | 鸟 | 羽毛/喙/翅膀 | 4 |
| 2004 | 鱼 | 鳞片/鳍/水生 | 4 |
| 2005 | 兔子 | 长耳/短尾/圆胖体型 | 4 |
| 2006 | 仓鼠 | 小型啮齿类/圆胖/短肢 | 4 |
| 2007 | 龟 | 龟壳/四肢短 | 4 |
| 2999 | 其他宠物 | 无法匹配上述类别的宠物 | 4 |

### 其他
- 识别不出人物或宠物 → result 为空数组
- 人物和宠物同时出现 → 分别输出
- 同类型多个人/宠物 → 每人/每只单独一条 result

## 输出格式

仅输出以下标准 JSON，无任何额外文字，不加代码块，无空格：

{"list":[{"result":[{"Index":0,"X":0,"Y":0,"Height":0,"Weight":0,"Type":0,"SubType":0,"AlbumType":0,"Translation":"","Emotion":"","Confidence":0.00000}]}]}

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| Index | int | 图片下标，从0开始递增 |
| X | int | 识别区域左上角X坐标（无法定位时填0） |
| Y | int | 识别区域左上角Y坐标（无法定位时填0） |
| Height | int | 识别区域高度（无法定位时填0） |
| Weight | int | 识别区域宽度（无法定位时填0） |
| Type | int | 1=人物，2=动物 |
| SubType | int | 1001=老人，1002=小孩，1003=成年人；2001=狗，2002=猫，2003=鸟，2004=鱼，2005=兔子，2006=仓鼠，2007=龟，2999=其他宠物 |
| AlbumType | int | 2=老人，3=小孩，4=动物，0=其他 |
| Translation | string | SubType 的中文含义（如"老人""狗"） |
| Emotion | string | 仅人物输出；开心/悲伤/惊讶/愤怒/恐惧/平静/其他 |
| Confidence | float64 | 置信度，保留5位小数（如 0.95320） |

### 强制规则
- list 数组长度 = 输入图片总数，每张图片必须有一个 result 元素
- 识别不出 → result 中仅含 Index 字段，其余用默认值：`{"Index":0}`
- 一张图可同时有多个结果（如同时有老人和狗），result 数组可含多条
- 返回 JSON 必须紧凑无空格、无换行、无 markdown 标记
- 所有字段名区分大小写，严格按上述格式
```

---

## 二、用户输入

输入为一张或多张图片（base64 或 URL），通过 API content 数组以 `image_url` 类型传入，提示词以 `text` 类型放在同一 content 数组中。

多图批量识别时，所有图片和提示词放在同一个 `content` 数组中，图片在前、文本在后。

## 四、百炼模型对接建议

| 配置项 | 建议值 | 说明 |
|------|------|------|
| 模型 | `qwen3.6-flash` | 百炼性价比最高的视觉模型，4.5 折活动中 |
| 端点 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | OpenAI 兼容接口 |
| 认证 | `Authorization: Bearer <DASHSCOPE_API_KEY>` | 从百炼控制台获取 |
| temperature | 0 | 设为 0 保证输出格式稳定 |
| max_tokens | 4096 | 多图多结果时需足够 token |
| stream | false | 非流式，一次性拿到完整 JSON |

### 注意事项

1. 百炼兼容 OpenAI 接口规范，图片通过 `content` 数组中的 `image_url` 类型传入，支持 URL 和 base64（`data:image/xxx;base64,...`）两种方式
2. 多图输入时建议单次 3-5 张，避免输出过长截断
3. 调试时先 `temperature=0` 验证格式稳定性
4. qwen3.6-flash 最多支持 256 张图片，单图最高 1600 万像素

 * 时光相册图片识别（百炼 qwen3.6-flash API）
 * @param {string[]} imageUrls - 图片 URL 或 base64 数组
 * @returns {object} 识别结果 { list: [{ result: [...] }] }
 */
async function analyzeAlbumImages(imageUrls) {
  const content = [
    ...imageUrls.map(url => ({
      type: 'image_url',
      image_url: { url }
    })),
    { type: 'text', text: SYSTEM_TEXT }
  ];

  const response = await fetch('https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${DASHSCOPE_API_KEY}`
    },
    body: JSON.stringify({
      model: 'qwen3.6-flash',
      messages: [{ role: 'user', content }],
      temperature: 0,
      max_tokens: 4096
    })
  });

  const data = await response.json();
  return JSON.parse(data.choices[0].message.content);
}
