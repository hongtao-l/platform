# 全能AI — UI设计

## 1. 设计概述

### 1.1 设计依据
基于 APP 端现有设计系统（Vant UI 组件库），采用单套餐方案（月付连续订阅 或 年付买断制，具体价格由运营配置），设备维度激活。设计遵循现有 APP 的视觉风格和交互习惯，保证与消息页、首页等现有页面的协调统一。

### 1.2 设计目标
- 清晰展示全能AI功能价值，降低用户理解门槛
- 套餐选择直观，支付流程顺畅
- 设备激活状态一目了然，内联引导不打断用户操作
- AI搜索、AI日报等功能页面保持简洁高效

## 2. 页面结构

### 2.1 页面层级
```
首页 (home)
├── 全能AI落地页 (ai)
│   ├── [未开通态] 服务介绍开通页
│   │   └── 支付弹窗
│   │   └── 协议确认弹窗
│   ├── [已开通态] AI配置页
│   │   └── 添加/编辑关注事件弹窗
│   │   └── Toast提示（上限提醒）
├── AI搜索页 (ai/search)
│   ├── [已激活] 猜你想搜 + 搜索结果
│   ├── [未激活] 内联开通引导
│   └── 事件详情页
├── AI日报页 (ai/daily)
│   ├── [已激活] 事件汇总（最早→…→最晚） + AI总结 + 免责声明
│   ├── [未激活] 内联开通引导
│   ├── 历史日报弹窗（近7日）
│   └── 事件详情页
└── 消息页 (message) — 联动修改
    ├── 设备筛选弹窗
    ├── 日期选择弹窗
    └── 事件详情页
```

## 5. 交互说明

### 5.1 套餐选择与支付
1. 默认选中月付套餐
2. 点击套餐卡片切换选中态（边框高亮 + 轻微放大动效）
3. 选中月付 → 显示续费提示文字；选中年付 → 隐藏
4. 协议文本随套餐切换联动
5. 未勾选协议点击支付 → 弹窗拦截（取消/同意）
6. 同意后自动勾选协议 → 进入支付弹窗

### 5.2 设备切换（内联引导）
1. 所有AI子页面默认选中第一台已激活设备
2. 切换到未激活设备时 → 保留框架UI，替换内容区为内联引导
3. 不弹Toast、不全屏锁死
4. 切换回已激活设备 → 恢复正常内容

### 5.3 AI搜索交互
1. 搜索前展示"猜你想搜"推荐标签
2. 点击标签 → 自动填入并搜索
3. 输入文字 + 点击搜索/回车 → 执行搜索
4. 清除按钮 → 恢复猜你想搜状态
5. 点击搜索结果 → 跳转事件详情

### 5.4 AI日报交互
1. 未生成态 → 点击"生成日报 · 剩余N次"（每日最多3次）
2. 生成中态 → 三步进度展示（idle→generating→done状态枚举）
3. 已生成态 → 事件汇总卡片（最早→…→最晚） + AI总结 + 重新生成按钮
4. 点击事件卡片（"其他"除外） → 跳转事件详情时间线
5. 切换设备 → 重置日报状态
6. 历史日报入口仅在已激活设备显示，弹窗标题"历史日报（近7日）"
7. 每日20:00自动生成（按设备本地时区，次数允许时）
8. 离开页面重置次数
9. 次数用完仅显示提示，不引导套餐

### 5.5 消息页联动
1. 切换设备 → 自动重置筛选标签为"全部"
2. 标签栏支持横向滚动（手势+鼠标滚轮）
3. 点击"+关注" → 根据设备激活状态跳转不同页面
