## 一、业务背景

全能AI 是安防摄像头的智能看护管家功能。通过 AI 视觉识别，摄像头能"看懂"画面中的事件（人、宠物、车辆、包裹、异常行为等），并根据用户设定的关注偏好，主动推送有意义的事件消息。

**要解决的用户痛点**：
- 传统摄像头产生大量无效告警（风吹草动、光线变化等），用户消息疲劳
- 用户无法快速定位关键事件（如"奶奶今天出门了吗"），只能逐条翻看录像
- 缺乏每日汇总，用户需要手动查看才能了解家中全天动态

**产品目标**：
- 通过 AI 识别降低无效推送，让用户只收到关心的内容
- 提供自然语言搜索，降低事件查找门槛
- 自动生成日报，让用户一眼掌握家中全天情况

**商业模式**：按月订阅或按年买断（具体价格由运营配置），按设备维度激活。

## 二、名词解释

| 术语 | 说明 |
|------|------|
| 全能AI | 摄像头智能看护管家功能，包含 AI消息推送、AI日报、AI搜索、关注事件管理 |
| 设备激活 | 对单台设备开通全能AI服务的行为，激活后该设备可使用全部AI功能 |
| 已激活设备 | 已开通全能AI服务的设备 |
| 未激活设备 | 未开通全能AI服务的设备 |
| 关注事件 | 用户自定义的AI识别事件类型（如老人看护、宠物活动），AI据此过滤和推送 |
| 基础事件 | 设备固有的人形侦测、运动侦测等非AI事件类型 |
| AI消息推送 | 摄像头识别画面中的人、宠物、车辆等，筛选关注事件后推送通知 |
| AI日报 | 每日自动生成的图文报告，汇总当天关注事件完整时间线 + AI一句总结 |
| AI搜索 | 通过自然语言描述搜索历史事件记录 |
| 智能推送开关 | 控制是否仅推送关注事件的全局开关，关闭后推送所有事件 |
| 连续订阅 | 按月付费，到期自动续费，可随时取消 |
| 买断制 | 按年付费，一次性购买，到期不自动续费 |

## 三、业务实体说明

### 3.1 核心实体

**用户（User）**
- 拥有多台摄像头设备
- 可开通全能AI服务
- 开通后按设备维度独立激活

**设备（Device）**
- 归属于用户
- 具有唯一名称和分组属性
- 独立的AI激活状态（已激活/未激活）
- 在线状态（在线/离线/休眠）

**全能AI服务（AIService）**
- 单一套餐方案
- 两种付费模式：连续订阅（月付）、买断制（年付），具体价格由运营配置
- 按设备维度激活，非全局开通

**关注事件（FocusEvent）**
- 用户自定义的AI关注事件类型
- 包含：名称、事件描述、启停状态
- 系统默认提供4个，用户可自定义最多10个

**AI日报（DailyReport）**
- 单日、单设备的AI事件汇总报告
- 包含：事件汇总、时间线、AI总结

**消息（Message）**
- 摄像头检测到的事件记录
- 分为关注事件（AI识别）和基础事件（运动/人形侦测）
- 携带设备、时间、事件类型、标签等信息

### 3.2 实体关系

```
用户 1──N 设备
设备 1──1 AI激活状态
设备 1──N 消息
用户 1──N 关注事件（跨设备共享）
关注事件 1──1 启停状态
设备 1──N AI日报（每日每设备最多1份）
```

## 四、核心业务流程

### 4.1 开通全能AI流程

```
用户进入落地页 → 浏览服务介绍
  → 选择套餐（月付/年付）
  → 选择支付方式（微信/支付宝）
  → 勾选协议
  → 点击"立即开通"
  → 确认支付（调起支付）
  → 支付成功 → 开通完成
  → 后续在配置页可管理各设备的激活状态
```

### 4.2 设备切换行为流程

```
用户进入AI搜索/AI日报/AI消息页面
  → 页面默认选中第一台已激活设备
  → 用户切换设备
    → 已激活设备：正常显示功能内容
    → 未激活设备：保留框架UI，内容区显示内联开通引导
    → 不弹Toast、不全屏锁死
```

### 4.3 AI日报生成流程

```
用户进入AI日报页 → 选择设备（默认当日，不支持日期选择）
  → 已激活设备 → 点击"生成日报"（每日最多3次）
  → 逐步展示生成进度：分析关注事件 → 整理时间线 → AI生成总结
  → 生成完成 → 展示事件汇总卡片（最早→…→最晚）+ AI总结 + 免责声明
  → 每日20:00自动生成（次数允许时，按设备本地时区，localStorage防重复）
```

### 4.4 消息筛选流程

```
用户进入消息页 → 默认显示第一台设备 + 当天消息
  → 切换设备 → 自动重置筛选标签为"全部"
  → 已激活设备：显示全部/关注事件标签/其他
  → 未激活设备：显示全部/运动侦测/人形侦测
  → 点击"+关注" → 根据设备激活状态跳转不同页面
```

## 五、业务规则

### 5.1 设备激活规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 设备级激活 | 全能AI服务按设备维度激活，非全局开关；每台设备独立控制 |
| R02 | 默认设备 | 包含AI功能的页面默认选中第一台已激活设备 |
| R03 | 未激活引导 | 未激活设备在各AI子功能页面显示内联开通引导，不弹Toast、不全屏锁死 |
| R04 | 激活联动 | 设备激活状态影响：消息筛选标签、首页AI入口文案、AI各子页面内容区 |

### 5.2 关注事件规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R05 | 默认事件 | 开通后系统预设4个关注事件：儿童守护、包裹送达、厨房守护、宠物守护 |
| R06 | 数量上限 | 用户最多添加10个关注事件（含默认），达到上限后提示"最多添加10个关注事件" |
| R07 | 智能推送 | 智能推送开关默认开启；开启后仅推送关注事件消息，关闭后推送所有事件 |
| R08 | 事件操作 | 每个关注事件支持：启停开关、编辑名称和描述、删除 |

### 5.3 套餐规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R09 | 两个选项 | 套餐A：按月连续订阅；套餐B：按年买断制。权益完全相同，具体价格由运营配置 |
| R10 | 连续订阅提示 | 选中月付时，下方显示自动续费提示文案 |
| R11 | 协议联动 | 月付勾选协议文本含"自动续费协议"，年付仅含"云服务协议" |
| R12 | 协议前置 | 未勾选协议点击支付 → 弹窗提示 → 用户可选择取消或同意并继续 |
| R13 | 支付方式 | 仅支持微信支付和支付宝 |

### 5.4 消息筛选规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R14 | 标签差异化 | 已激活设备：关注事件标签在前，基础事件在后（全部、老人看护、宠物活动、人员经过、其他） |
| R15 | 标签默认 | 未激活设备：仅基础事件标签（全部、运动侦测、人形侦测） |
| R16 | 切换重置 | 切换设备时自动重置筛选标签为"全部" |
| R17 | 无全部设备 | 设备筛选为单选，无"全部设备"选项 |
| R18 | 横向滚动 | 标签栏超出屏幕时支持横向滚动（手势滑动 + 鼠标滚轮） |

### 5.5 AI日报规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R19 | 仅当日 | 日报仅支持当日生成，不支持日期选择和历史日期切换 |
| R20 | 生成进度 | 分三步展示：分析关注事件 → 整理时间线 → AI生成总结，使用单一状态枚举（idle/generating/done） |
| R21 | 历史入口 | 已激活设备右上角显示历史日报入口（时钟图标），仅展示近7日；未激活设备隐藏 |
| R22 | 生成次数 | 每日每台设备最多生成3次（按日期+设备维度计数，localStorage持久化），次数用完仅显示提示不引导套餐 |
| R23 | 自动生成 | 每日20:00在次数允许时自动生成日报（按设备本地时区，国内用北京时间、海外用当地时区；检查autoGenKey防重复触发） |
| R24 | 事件汇总 | 每个关注事件显示最早和最晚一条事件描述，中间省略号；非关注事件归入"其他"类别 |
| R25 | 次数重置 | 离开页面时重置当日生成次数（支持演示场景） |
| R26 | 免责声明 | 日报底部固定显示"本功能由AI大模型提供，个别结果可能存在误差" |

## 七、详细功能描述

### 7.1 全能AI落地页

#### 功能应用场景
- 用户首次了解全能AI功能，浏览服务介绍和套餐信息
- 用户选择套餐并完成购买开通
- 已开通用户管理设备激活状态和关注事件

#### 7.1.1 服务介绍开通页（未开通态）

**页面由5个区域从上至下组成：**

**① 顶部 Banner**
- 全能AI图标 + 主标题"全能AI · 智能看护管家"
- 副标题简述核心价值
- 三张功能卡片横向排列：AI消息、AI日报、AI搜索
- 每张卡片含图标、功能名、一句话描述

**② 套餐选择区（左右并排）**

两个套餐卡片竖版布局，每个卡片包含：套餐名称 → 价格 → 标签 → 四项权益（打勾列表）

| 属性 | 套餐A | 套餐B |
|------|------|------|
| 名称 | 全能AI套餐 | 全能AI套餐 |
| 价格 | 月付（具体价格运营配置） | 年付（具体价格运营配置） |
| 付费模式 | 连续订阅 | 买断制 |
| 标签 | "连续订阅" | "买断制" |
| 权益 | AI消息推送、AI日报、AI搜索、关注事件管理 | 同套餐A |

- 选中连续订阅后，套餐区下方显示自动续费提示文案
- 选中买断制时此提示不显示

**③ 支付方式选择**
- 两个选项并排：微信支付 / 支付宝
- 单选，默认选中支付宝支付

**④ 服务介绍区**
- AI消息推送：摄像头自动识别画面中的人、宠物、车辆等，只推送你关注的事件
- AI日报总结：每日自动生成图文报告，汇总关注事件的完整时间线和一句话总结
- AI搜索：用自然语言搜索事件记录，快速定位关键片段
- 关注事件管理：自定义关注的人和事物，设置关键词，AI精准识别并优先推送

**⑤ 底部固定栏**
- 协议勾选区（整行可点击）：
  - 方形复选框 + 协议文字
  - 未勾选：灰边框空心框
  - 已勾选：蓝底白对勾
  - 连续订阅时协议文本："已阅读并同意《云服务协议（含自动续费协议）》"
  - 买断制时协议文本："已阅读并同意《云服务协议》"
- "立即开通"按钮，显示当前选中套餐价格（由运营配置）
- 未勾选协议点击按钮 → 弹窗提示"请先阅读并同意云服务协议"
  - 弹窗按钮：取消 / 同意（点击同意自动勾选协议，进入支付流程）

#### 7.1.2 AI配置页（已开通态）

**状态卡片**
- 标题"全能AI"
- 有效期信息（如"有效期至 2026-06-20"）
- 绿色"✓ 守护中"状态标识

**智能推送设置**
- 智能推送开关（Switch组件），默认开启
- 开关下方提示："开启后仅推送关注事件消息"

**关注事件管理**
- 4个默认关注事件：

| 事件名称 | 事件描述 |
|------|------|
| 儿童守护 | 小孩哭喊、跌倒、出门 |
| 包裹送达 | 快递或者外卖送达 |
| 厨房守护 | 厨房异常烟雾、明火 |
| 宠物守护 | 宠物乱拉乱尿、吃饭、喝水 |

- 每个事件条目包含：事件名称、事件描述、启停开关、编辑按钮、删除按钮
- "添加"按钮显示当前数量/上限（如"添加 4/10"）
- 点击添加 → 弹出输入弹窗（关注名称 + 事件描述）
- 最多添加10个关注事件，达到上限 → Toast提示"最多添加10个关注事件"

#### 7.1.3 支付弹窗

- 顶部显示支付金额（随选中套餐变化，具体金额由运营配置）
- 支付方式：微信支付 / 支付宝（与落地页选择联动）
- 确认支付按钮

### 7.2 AI搜索页

#### 功能应用场景
- 用户想快速查找历史事件（如"奶奶今天出门了吗""猫咪上午在干嘛"）
- 用户不知道具体时间，只记得事件特征，通过自然语言描述查找

#### 页面结构

**设备筛选栏（始终可见）**
- 横向排列所有设备名称
- 当前选中设备高亮
- 默认选中第一台已激活设备
- 切换设备时不清除搜索内容

**搜索输入区（始终可见）**
- 自然语言输入框 + 清除按钮（输入内容后显示）+ 搜索按钮
- 输入框下方示例提示："👉 试试：'奶奶今天出门了吗' '猫咪上午在干嘛'"

**内容区 — 已激活设备**
- 搜索前：显示"猜你想搜"推荐标签（如"奶奶今天出门了吗""猫咪上午在干嘛""门口有快递""晚上有人经过"），点击标签自动填入并搜索
- 搜索后：显示"共找到 N 条相关事件"，结果列表（缩略图 + 事件类型 + 设备名 + 时间 + 描述）
- 搜索结果可点击进入事件详情

**内容区 — 未激活设备**
- 内联引导：AI搜索图标 + "AI搜索" + 功能描述 + "立即开通全能AI"按钮
- 点击按钮跳转服务介绍开通页
- 设备筛选栏和搜索输入区保持可见，仅内容区替换

#### 交互说明
- 切换设备：点击设备标签切换，不弹Toast、不全屏锁死
- 搜索触发：点击搜索按钮或按回车键
- 清除搜索：点击清除按钮，恢复猜你想搜状态

### 7.3 AI日报页

#### 功能应用场景
- 用户想看今天家中发生了什么事，一键生成汇总报告
- 用户回顾近7日的历史日报

#### 页面结构

**设备筛选栏（始终可见）**

## 九、异常说明

### 9.1 网络异常

| 场景 | 表现 |
|------|------|
| 加载落地页时断网 | 显示已有的静态内容（Banner、套餐介绍、服务介绍）；购买按钮置灰或点击后提示"网络异常，请稍后重试" |

| AI搜索请求断网 | 提示"网络异常，搜索未完成，请重试" |

### 9.2 数据为空

| 场景 | 表现 |
|------|------|
| 搜索无结果 | 显示空态文案"未找到相关事件，试试其他描述" |

| 历史日报列表为空 | 隐藏历史日报入口图标 |

### 9.3 操作失败

| 场景 | 表现 |
|------|------|
| 支付失败 | 弹窗提示失败原因（余额不足、支付取消等），按钮可返回重试 |

| 关注事件添加失败 | Toast提示"添加失败，请重试" |
| 日报生成失败 | 提示"生成失败，请重试"，保留生成按钮可再次点击 |

### 9.4 权限与限制

| 场景 | 表现 |
|------|------|

## 十、APP埋点与运营数据看板

### 10.1 核心指标体系

#### 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 落地页曝光UV | 行为 | 进入服务介绍开通页的去重用户数 | 日 |
| 2 | 套餐选择率 | 行为 | 选择套餐UV / 落地页曝光UV | 日 |
| 3 | 支付转化率 | 业务 | 支付成功UV / 落地页曝光UV | 日 |
| 4 | 开通成功率 | 质量 | 支付成功UV / 支付点击UV | 日 |
| 5 | ARPU | 业务 | 总收入 / 支付成功UV | 月 |
| 6 | AI搜索使用率 | 行为 | 使用AI搜索UV / 已激活设备UV | 周 |
| 7 | AI日报生成率 | 行为 | 生成日报UV / 已激活设备UV | 周 |
| 8 | 日报生成成功率 | 质量 | 日报生成成功次数 / 日报生成请求次数 | 日 |
| 9 | 关注事件人均添加数 | 行为 | 用户添加关注事件总数 / 已开通用户数 | 月 |
| 10 | 设备激活率 | 增长 | 已激活设备数 / 总设备数 | 周 |
| 11 | 续费率 | 业务 | 次月续费用户数 / 应续费用户数 | 月 |
| 12 | 内联引导转化率 | 业务 | 从内联引导进入落地页后支付成功UV / 内联引导点击UV | 周 |

#### 核心指标说明

**落地页曝光UV**：衡量服务介绍开通页的流量入口效果。反映首页AI入口、消息页"+关注"、子功能内联引导等入口的引流能力。

**支付转化率**：衡量落地页 → 支付的完整转化漏斗，是评估落地页效果和套餐吸引力的核心指标。

**AI搜索使用率 / AI日报生成率**：衡量已开通用户的功能黏性，反映AI功能是否真正满足了用户需求。

**内联引导转化率**：衡量内联引导这一非阻断式引导方式的转化效果，验证「不锁死页面」策略是否有效。
