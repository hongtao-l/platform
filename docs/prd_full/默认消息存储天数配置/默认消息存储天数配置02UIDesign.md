# 默认消息存储天数配置 — 设计说明

> v3.0：方案更新 — 对齐线上平台实际结构，物模型配置为对话框、算法配置为抽屉

## 页面结构

本需求在 IoT 管理后台产品审核详情页中改造按钮行为并新增弹窗组件：

```
IoT 管理后台 — 产品审核模块
├── 产品审核列表页（已有）
│   └── 「管理产品」→ 跳转至产品详情页
├── 产品详情页（已有，改造按钮）
│   ├── 基本信息卡片
│   ├── 物模型配置卡片 →「查看/修改」→ 打开物模型配置对话框（设备配置）
│   ├── 算法服务配置表格 →「配置」→ 打开算法配置抽屉
│   ├── 平台服务卡片
│   ├── App配置卡片 →「前往查看/修改」→ 跳转至App功能配置页
│   ├── 语音助手配置卡片
│   └── 固件版本历史卡片
├── 物模型配置对话框（新建弹窗）
│   └── 云端抓图 Tab → 存储时长配置 ← 核心配置 A
├── 算法配置抽屉（新建抽屉）
│   └── 事件记录(ID 1013)「配置」→ 存储天数配置 ← 核心配置 B
└── App功能配置页（已有，移除存储天数配置项）
```

物模型配置以对话框（Dialog，900px）打开，内含 border-card 类型 Tabs。算法配置以右侧抽屉（Drawer，900px）打开。

## 布局详述

### 区域1: 产品详情页（改造部分）

仅改造两个卡片的按钮行为，布局不变：

**区域1.1: 物模型配置卡片**

- **位置**：基本信息卡片下方
- **内容**：左侧设置图标 + "已配置 N 个子项"文字；右侧「查看/修改」文字链接按钮
- **交互变更**：点击「查看/修改」→ 打开物模型配置对话框

**区域1.2: 算法服务配置表格**

- **位置**：交互控制详情下方
- **表格列**：ID(AIIoTType)、算法名称、服务ID、描述（溢出省略）、是否支持（switch）、是否免开通（disabled switch）、算法配置（按钮）、关联事件（按钮）
- **关键数据行**：
  - 云端抓图 (ID 1011)：有点击「配置」→ 打开算法配置抽屉
  - 事件记录 (ID 1013)：点击「配置」→ 打开算法配置抽屉，顶部多一个「事件记录配置」区域
- **交互变更**：点击某算法行的「配置」→ 打开算法配置抽屉，携带算法信息

### 区域2: 物模型配置对话框

- **组件**：el-dialog，标题"设备配置"，宽度 900px，top 5vh
- **内容**：el-tabs（type="border-card"），6 个 Tab：
  - 设备能力、镜头配置、麦克风配置、屏显配置、喇叭配置 → 占位提示
  - **云端抓图** → 存储时长配置

**区域2.1: 云端抓图 Tab**

- **位置**：最后一个 Tab
- **顶部警示**：el-alert type="warning" 提示"该配置会影响设备的报警抓图能力，请谨慎选择"
- **内容**：表单（标签宽度 140px），包含以下配置项：
  - **默认开启**：Radio 单选组（开启/关闭），默认开启
  - **支持动图**：Radio 单选组（是/否），默认否
  - **动图开关**：Radio 单选组（开启/关闭），仅"支持动图=是"时显示
  - **抓图质量**：Select 下拉选择（小图/标清/高清），默认小图
  - **存储时长(天)**：InputNumber 1-30 天，默认 1，带成本提示说明 ← 核心配置
  - **存储方式**：Radio 单选组（事件/定时），默认事件
  - **存储格式**：Radio 单选组（已有格式带私有头/标准jpg）
  - **抓图数目**：InputNumber 0-99
  - **抓拍间隔**：InputNumber 5-300，单位"秒"，默认 30

- **存储时长辅助说明**：信息图标 + "设置云端抓图在服务器的存储时长（1-30天）。仅平台可见，涉及云存储成本，请谨慎配置。"
- **其他 Tab**（设备能力/镜头配置/麦克风配置/屏显配置/喇叭配置）：各有对应配置选项 + 标准能力/自定义能力空表格

**区域2.2: 对话框底部按钮**

- **位置**：对话框 footer 区域
- **按钮**：左「上一步」（首个 Tab 隐藏）+ 左「跳过」+ 右「保存,下一步」（末 Tab 为「保存,完成」）
- **交互**：点击「保存,完成」→ Toast "物模型配置已保存"，关闭对话框；「跳过」→ 前进到下一 Tab

### 区域3: 算法配置抽屉

- **组件**：el-drawer，标题"算法配置"，右侧滑出，宽度 900px
- **内容**：根据算法类型不同，内容有差异

**区域3.1: 事件记录配置（仅事件记录 ID 1013）**

- **位置**：抽屉内容顶部，能力绑定上方
- **区域标题**：h4 "事件记录配置"
- **内容**：表单（标签宽度 140px），包含：
  - **存储天数**：InputNumber 1-30 天，默认 1，带成本提示说明 ← **核心配置 B**
  - **成本提示**：信息图标 + "设置该算法事件记录的云端存储天数（1-30天）。仅平台可见，涉及云存储成本，请谨慎配置。"
- **控件规格**：InputNumber 宽度 140px，最小值 1，最大值 30，步长 1；单位"天"在输入框右侧间距 8px

**区域3.2: 能力绑定（所有算法通用）**

- **位置**：事件记录配置下方（如有），或抽屉顶部（其他算法）
- **内容**：border-card Tabs，分为「标准能力」和「自定义能力」
- 每个 Tab 内含表格：类型、能力名称、能力标识、数据类型、数据定义、描述、所属模块、操作（删除）
- 表头操作：「添加标准能力」/「添加自定义能力」按钮
- **自定义能力** Tab 额外显示提示："您所定义的每一个功能，都与固件、APP控制台面板相关"

**区域3.3: 抽屉底部按钮**

- **位置**：抽屉 footer 区域
- **按钮**：右对齐，「取消」（关闭抽屉）+「确定」（蓝色主按钮）
- **交互**：点击「确定」→ Toast "算法配置已保存"（事件记录额外提示存储天数），关闭抽屉

### 区域4: App功能配置页（改造部分）

- **改动**：移除默认配置 Tab 底部的「默认消息存储天数」配置项（含 el-divider 和辅助说明行）
- **保留**：默认配置 Tab 其余 7 个配置项不变（默认分辨率、图片插值、视频插值、全彩模式时段、一键报警配置、假三目设置、灯控开关显示）

## 状态说明

| 状态 | 触发条件 | 展示内容 |
|------|------|------|
| 正常态 | 所有数据就绪 | 各页面/弹窗完整内容渲染 |
| 空态 — 算法服务 | 产品无算法服务配置 | 表格内显示"暂无算法服务配置" |
| 空态 — 能力绑定 | 算法无绑定能力 | 标准/自定义能力表格显示"暂无数据" |
| 边界态 — 最小值 | 存储时长/天数设为 1 | 减号按钮置灰，不可继续减少 |
| 边界态 — 最大值 | 存储时长/天数设为 30 | 加号按钮置灰，不可继续增加 |
| 错误态 — 保存失败 | 网络异常或服务端错误 | Toast"保存失败，请重试"，值回退 |
| 错误态 — 参数缺失 | URL 缺少必要参数 | 页面渲染，对应信息区显示空值 |

## 组件清单

| 组件 | 用途 | 所在区域 |
|------|------|------|
| 对话框（Dialog） | 物模型配置弹窗，含 border-card Tabs | 物模型配置 |
| 抽屉（Drawer） | 算法配置，右侧滑出 | 算法配置 |
| 分页标签（Tabs, border-card） | 能力绑定标准/自定义切换 | 算法配置抽屉 |
| 数据表格（Table） | 列表数据展示 | 算法服务卡片、能力绑定 |
| 开关（Switch） | 布尔值切换 | 平台服务、算法表格 |
| 单选组（RadioGroup） | 多选一 | 平台服务（码流） |
| 数字输入（InputNumber） | 整数输入，支持步进 | 云端抓图存储时长、事件记录存储天数 |
| 按钮（Button） | 触发操作 | 各处操作和保存按钮 |
| 信息图标（InfoFilled） | 成本提示图标 | 存储时长/天数辅助说明行 |
| 设置图标（SetUp） | 配置图标 | 物模型配置卡片 |
| 状态标签（Tag） | 状态徽标 | 基本信息、固件版本 |
