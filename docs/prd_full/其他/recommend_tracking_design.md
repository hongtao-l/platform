# 推荐位埋点设计文档

> 文档版本: v1.0  
> 更新日期: 2026-04-16  
> 目标: 通过精细化埋点数据，实现用户画像精准刻画，支撑精细化运营决策

---

## 一、埋点设计目标

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| 用户画像刻画 | 了解用户偏好、消费能力、行为习惯 |
| 策略效果评估 | 评估不同策略的转化效果，优化投放 |
| 漏斗分析 | 分析用户从曝光到购买的完整路径 |
| 个性化推荐 | 基于行为数据实现千人千面的套餐推荐 |

### 1.2 数据应用场景

```
埋点数据 → 数据仓库 → 用户标签 → 精细化运营
                              ↓
                    ┌─────────┼─────────┐
                    ↓         ↓         ↓
              用户分群    策略优化    效果归因
```

## 二、埋点事件体系

### 2.1 事件分类总览

| 事件类型 | 事件名称 | 事件ID | 触发时机 |
|----------|----------|--------|----------|
| **曝光类** | 推荐位曝光 | rec_slot_expose | 推荐位内容展示在用户可视区域 |
| **浏览类** | 推荐位浏览 | rec_slot_view | 用户在推荐位停留超过阈值时间 |
| **点击类** | 套餐卡片点击 | rec_pkg_click | 用户点击套餐卡片 |
| **转化类** | 套餐购买成功 | rec_pkg_purchase | 用户完成套餐购买 |
| **交互类** | 漏斗层级切换 | rec_funnel_switch | 用户在漏斗不同层级间切换 |

### 2.2 详细事件定义

#### 2.2.1 推荐位曝光事件 (rec_slot_expose)

**触发条件**: 推荐位内容进入用户可视区域（超过50%面积）

**事件属性**:

| 属性名 | 属性ID | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| 用户ID | user_id | string | 是 | 用户唯一标识 |
| 设备ID | device_id | string | 是 | 设备唯一标识 |
| 应用ID | app_id | string | 是 | 牵心PRO/鹤梦之家 |
| 推荐位编码 | slot_code | string | 是 | rec_slot_home |
| 推荐位名称 | slot_name | string | 是 | 首页推荐位 |
| 策略ID | strategy_id | string | 是 | 当前生效的策略ID |
| 漏斗ID | funnel_id | string | 是 | 当前展示的漏斗ID |
| 漏斗层级 | funnel_level | int | 是 | 当前展示的漏斗层级(1-4) |
| 套餐ID | pkg_id | string | 是 | 当前展示的套餐ID |
| 套餐名称 | pkg_name | string | 是 | 套餐名称 |
| 套餐分类 | pkg_category | string | 是 | 云存储/AI服务/综合套餐 |
| 套餐价格 | pkg_price | float | 是 | 套餐售价 |
| 投放区域 | region | string | 是 | 国内/海外 |
| 用户分群 | user_group | string | 是 | 用户所属分群ID |
| 曝光时长 | expose_duration | int | 否 | 曝光持续毫秒数 |
| 屏幕位置 | screen_position | string | 否 | 首屏/二屏/三屏 |
| 时间戳 | timestamp | long | 是 | 事件发生时间 |

**示例数据**:
```json
{
  "event_id": "rec_slot_expose",
  "user_id": "U12345678",
  "device_id": "D87654321",
  "app_id": "QX_PRO_001",
  "slot_code": "rec_slot_home",
  "slot_name": "首页推荐位",
  "strategy_id": "STR_001",
  "funnel_id": "FUN_001",
  "funnel_level": 1,
  "pkg_id": "REC_001",
  "pkg_name": "7天事件云存储基础版",
  "pkg_category": "云存储",
  "pkg_price": 6.90,
  "region": "domestic",
  "user_group": "ug_rec_high_value",
  "expose_duration": 3000,
  "screen_position": "first_screen",
  "timestamp": 1713244800000
}
```

#### 2.2.2 推荐位浏览事件 (rec_slot_view)

**触发条件**: 用户在推荐位停留时间超过阈值（如2秒）

**事件属性**:

| 属性名 | 属性ID | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| 基础属性 | - | - | - | 同曝光事件 |
| 浏览时长 | view_duration | int | 是 | 用户停留毫秒数 |
| 滚动深度 | scroll_depth | float | 是 | 页面滚动百分比 |
| 是否首屏 | is_first_screen | boolean | 是 | 是否在首屏展示 |

**业务价值**:
- 区分"路过"和"真正关注"的用户
- 计算内容吸引力指数 = 浏览时长 / 曝光时长

---

#### 2.2.3 套餐卡片点击事件 (rec_pkg_click)

**触发条件**: 用户点击推荐位中的套餐卡片

**事件属性**:

| 属性名 | 属性ID | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| 基础属性 | - | - | - | 同曝光事件 |
| 点击位置 | click_position | string | 是 | 卡片/按钮/图片 |
| 跳转页面 | target_page | string | 是 | 套餐详情页/购买页 |
| 前序行为 | prev_action | string | 否 | 点击前用户行为路径 |

**业务价值**:
- 计算点击率 CTR = 点击数 / 曝光数
- 分析用户对不同套餐的兴趣度

---

#### 2.2.4 套餐购买成功事件 (rec_pkg_purchase)

**触发条件**: 用户完成套餐购买支付

**事件属性**:

| 属性名 | 属性ID | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| 基础属性 | - | - | - | 同曝光事件 |
| 订单ID | order_id | string | 是 | 订单唯一标识 |
| 支付金额 | pay_amount | float | 是 | 实际支付金额 |
| 支付方式 | pay_method | string | 是 | 微信/支付宝/Apple Pay |
| 优惠金额 | discount_amount | float | 否 | 优惠券/活动优惠金额 |
| 购买时长 | purchase_duration | int | 是 | 从曝光到购买的时长(秒) |
| 转化路径 | conversion_path | string | 是 | 曝光→点击→详情→购买 |

**业务价值**:
- 计算转化率 CVR = 购买数 / 点击数
- 计算ROI = 收入 / 运营成本
- 分析用户决策周期

---

#### 2.2.5 漏斗层级切换事件 (rec_funnel_switch)

**触发条件**: 用户在漏斗不同层级间切换（如从第1层切换到第2层）

**事件属性**:

| 属性名 | 属性ID | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| 基础属性 | - | - | - | 同曝光事件 |
| 原层级 | from_level | int | 是 | 切换前的漏斗层级 |
| 目标层级 | to_level | int | 是 | 切换后的漏斗层级 |
| 切换原因 | switch_reason | string | 是 | 用户拒绝/超时/条件变化 |
| 展示套餐数 | pkg_count | int | 是 | 该层级展示的套餐数量 |

**业务价值**:
- 分析漏斗各层级的转化效率
- 优化漏斗策略配置

## 三、用户画像标签体系

### 3.1 标签分类

```
用户画像标签
├── 基础属性标签
│   ├── 人口属性: 年龄、性别、地域
│   ├── 设备属性: 设备型号、系统版本
│   └── 会员属性: 会员等级、注册时长
│
├── 行为偏好标签
│   ├── 浏览偏好: 常浏览时段、浏览频次
│   ├── 点击偏好: 高点击套餐类型
│   └── 购买偏好: 高购买套餐类型
│
├── 消费能力标签
│   ├── 消费金额: 累计消费、客单价
│   ├── 消费频次: 购买次数、复购率
│   └── 价格敏感度: 优惠敏感/价格不敏感
│
└── 运营响应标签
    ├── 策略响应: 对不同策略的响应率
    ├── 漏斗响应: 漏斗各层级转化率
    └── 分群匹配: 当前所属分群
```

### 3.2 标签计算规则

#### 3.2.1 行为偏好标签

| 标签名称 | 计算规则 | 数据来源 |
|----------|----------|----------|
| 浏览时段偏好 | 统计用户浏览最多的时段(0-6/6-12/12-18/18-24) | rec_slot_view |
| 套餐类型偏好 | 统计用户点击最多的套餐类型占比 | rec_pkg_click |
| 推荐位活跃度 | 近7天浏览次数 / 7 | rec_slot_view |
| 点击活跃度 | 近7天点击次数 / 7 | rec_pkg_click |

#### 3.2.2 消费能力标签

| 标签名称 | 计算规则 | 数据来源 |
|----------|----------|----------|
| 累计消费金额 | 用户历史总消费金额 | rec_pkg_purchase |
| 客单价 | 累计消费金额 / 购买次数 | rec_pkg_purchase |
| 复购率 | 购买次数 > 1 的用户占比 | rec_pkg_purchase |
| 价格敏感度 | 优惠订单占比 > 50% 为敏感 | rec_pkg_purchase |

#### 3.2.3 运营响应标签

| 标签名称 | 计算规则 | 数据来源 |
|----------|----------|----------|
| 策略响应率 | 该策略下用户点击率 | rec_slot_expose + rec_pkg_click |
| 漏斗转化率 | 用户在各层级的转化率 | rec_funnel_switch |
| 分群匹配度 | 用户行为与分群定义的匹配程度 | 综合计算 |

### 3.3 标签更新频率

| 标签类型 | 更新频率 | 说明 |
|----------|----------|------|
| 实时标签 | 实时 | 当前状态、在线状态 |
| 日标签 | T+1 | 行为偏好、消费能力 |
| 周标签 | T+7 | 长期偏好、用户价值 |
| 月标签 | T+30 | 用户生命周期阶段 |

## 四、数据分析指标体系

### 4.1 核心指标

#### 4.1.1 曝光指标

| 指标名称 | 计算公式 | 说明 |
|----------|----------|------|
| 曝光量(PV) | 曝光事件数 | 推荐位总曝光次数 |
| 曝光用户数(UV) | 去重用户数 | 看到推荐位的用户数 |
| 人均曝光次数 | PV / UV | 平均每个用户看到推荐位的次数 |
| 曝光覆盖率 | 曝光UV / 活跃UV | 推荐位触达用户比例 |

#### 4.1.2 点击指标

| 指标名称 | 计算公式 | 说明 |
|----------|----------|------|
| 点击量 | 点击事件数 | 套餐卡片点击次数 |
| 点击用户数 | 去重用户数 | 点击过套餐的用户数 |
| 点击率(CTR) | 点击量 / 曝光量 | 衡量内容吸引力 |
| 人均点击次数 | 点击量 / 点击UV | 用户点击深度 |

#### 4.1.3 转化指标

| 指标名称 | 计算公式 | 说明 |
|----------|----------|------|
| 转化量 | 购买事件数 | 套餐购买次数 |
| 转化用户数 | 去重用户数 | 购买过套餐的用户数 |
| 转化率(CVR) | 转化量 / 点击量 | 衡量转化效果 |
| 客单价 | 总收入 / 转化量 | 平均订单金额 |
| GMV | 转化金额总和 | 商品交易总额 |

#### 4.1.4 漏斗指标

| 指标名称 | 计算公式 | 说明 |
|----------|----------|------|
| 漏斗转化率 | 各层级转化率乘积 | 整体漏斗效率 |
| 层级流失率 | 1 - 层级转化率 | 各层级流失比例 |
| 平均决策时长 | 购买时长中位数 | 用户决策周期 |

### 4.2 分析维度

#### 4.2.1 时间维度

- 小时级: 分析用户活跃时段
- 日级: 分析日常波动趋势
- 周级: 分析周期性规律
- 月级: 分析长期趋势

#### 4.2.2 用户维度

- 新老用户: 新用户 vs 老用户行为差异
- 分群用户: 不同分群的响应差异
- 会员等级: 不同等级用户的偏好差异
- 地域分布: 国内 vs 海外用户差异

#### 4.2.3 内容维度

- 套餐类型: 云存储/AI服务/综合套餐
- 价格区间: 低/中/高价位套餐
- 推荐位: 不同推荐位的效果差异
- 策略: 不同策略的效果差异

### 4.3 分析报表

#### 4.3.1 实时监控看板

| 报表名称 | 刷新频率 | 核心指标 |
|----------|----------|----------|
| 实时流量看板 | 1分钟 | 实时曝光、点击、转化 |
| 策略效果看板 | 5分钟 | 各策略实时CTR、CVR |
| 异常告警看板 | 实时 | 数据异常波动告警 |

#### 4.3.2 日常分析报表

| 报表名称 | 生成频率 | 分析内容 |
|----------|----------|----------|
| 日报 | 每日 | 昨日核心指标、环比变化 |
| 周报 | 每周 | 本周趋势分析、分群效果 |
| 月报 | 每月 | 月度总结、策略优化建议 |

## 五、精细化运营应用场景

### 5.1 用户分群优化

**场景**: 基于埋点数据优化用户分群规则

**数据支撑**:
```
用户行为数据 → 标签计算 → 分群规则调整 → 策略匹配优化
```

**示例**:
- 发现"高价值用户"分群对AI服务套餐点击率更高
- 调整策略: 为该分群优先展示AI服务套餐

### 5.2 策略效果评估

**场景**: 评估不同策略的投放效果

**数据支撑**:
```
策略A vs 策略B
├── 曝光量对比
├── CTR对比
├── CVR对比
├── GMV对比
└── ROI对比
```

**决策依据**:
- ROI高的策略: 增加投放
- ROI低的策略: 优化或下线

### 5.3 漏斗策略优化

**场景**: 优化漏斗各层级的套餐配置

**数据支撑**:
```
漏斗分析
├── 第1层: 曝光→点击转化率
├── 第2层: 点击→详情转化率
├── 第3层: 详情→购买转化率
└── 第4层: 购买→复购转化率
```

**优化方向**:
- 流失率高的层级: 优化套餐选择
- 转化率低的层级: 调整价格或文案

### 5.4 个性化推荐

**场景**: 基于用户画像实现千人千面

**数据支撑**:
```
用户画像
├── 偏好标签: 喜欢云存储套餐
├── 消费能力: 客单价50-100元
├── 活跃时段: 晚上8-10点
└── 价格敏感度: 中等敏感
```

**推荐策略**:
- 推荐云存储相关套餐
- 价格区间50-100元
- 晚上8点推送优惠信息

## 六、数据采集技术方案

### 6.1 SDK集成

```javascript
// 埋点SDK初始化
Tracker.init({
  appId: 'QX_PRO_001',
  userId: 'U12345678',
  deviceId: 'D87654321',
  region: 'domestic',
  userGroup: 'ug_rec_high_value'
});

// 曝光事件上报
Tracker.track('rec_slot_expose', {
  slot_code: 'rec_slot_home',
  strategy_id: 'STR_001',
  funnel_id: 'FUN_001',
  funnel_level: 1,
  pkg_id: 'REC_001',
  pkg_name: '7天事件云存储基础版',
  pkg_category: '云存储',
  pkg_price: 6.90
});

// 点击事件上报
Tracker.track('rec_pkg_click', {
  slot_code: 'rec_slot_home',
  pkg_id: 'REC_001',
  click_position: 'card'
});

// 购买事件上报
Tracker.track('rec_pkg_purchase', {
  slot_code: 'rec_slot_home',
  pkg_id: 'REC_001',
  order_id: 'ORD_123456',
  pay_amount: 6.90,
  pay_method: 'wechat'
});
```

### 6.2 数据上报策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 实时上报 | 事件发生立即上报 | 购买、点击等关键事件 |
| 批量上报 | 累积一定数量后上报 | 曝光、浏览等高频事件 |
| 延迟上报 | 网络恢复后上报 | 弱网环境 |

### 6.3 数据质量保障

| 保障措施 | 说明 |
|----------|------|
| 数据校验 | 上报前校验必填字段 |
| 去重处理 | 相同事件去重 |
| 异常过滤 | 过滤异常数据(如超长时长) |
| 数据补全 | 补充缺失的公共字段 |

```
事件ID格式: {模块}_{动作}
```

```
属性ID格式: {实体}_{属性}
示例: pkg_name (套餐_名称)
```
