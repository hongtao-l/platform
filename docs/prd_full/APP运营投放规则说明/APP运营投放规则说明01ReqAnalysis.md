# APP 运营投放规则说明 — 需求分析

## 1. 业务背景与目标

### 1.1 背景

生态平台当前管理多个 APP（如牵心PRO、鹤梦之家），所有 APP 统一由平台运营方进行运营投放管理。随着业务发展，出现了两种 APP 运营模式：

- **代运营 APP**：平台运营方全权负责 APP 的运营投放，包括商城位、推荐位、广告位、活动位的策略配置
- **企业自运营 APP**：企业客户自行管理 APP 运营，平台仅为其提供套餐配置能力，不介入日常运营投放

一个企业下可能存在多个 APP，其中部分由平台代运营、部分由企业自运营。需要一套清晰的规则来区分两类 APP 的运营能力边界。

### 1.2 目标

- 明确「代运营 APP」和「企业自运营 APP」的定义与运营能力边界
- 规范代运营 APP 的四个运营位（商城位、推荐位、广告位、活动位）的投放规则
- 规范企业自运营 APP 仅开放套餐配置模块的权限规则
- 确保套餐创建时，代运营 APP 和自运营 APP 的投放范围可正确区分
- 为企业多 APP 场景（混合运营模式）提供清晰的规则说明

## 2. 角色与职责

| 角色 | 职责 |
|------|------|
| 平台运营管理员 | 管理代运营 APP 的商城位、推荐位、广告位、活动位策略 |
| 平台运营管理员 | 创建套餐并指定可投放的 APP 范围 |
| 企业客户 | 在自运营 APP 中配置套餐，管理套餐上下架 |
| 终端用户 | 在 APP 内看到运营位内容（商城套餐、推荐弹窗、广告、活动） |

## 3. 名词解释

| 术语 | 说明 |
|------|------|
| **代运营 APP** | 由平台运营方全权负责运营投放的 APP。平台可为其配置商城位、推荐位、广告位、活动位四大运营模块 |
| **企业自运营 APP** | 由企业客户自行管理的 APP。平台仅提供套餐配置功能，不介入其他运营投放 |
| **运营位** | APP 内用于展示运营内容的区域，包含商城位、推荐位、广告位、活动位 |
| **商城位** | APP 商城页面的套餐展示区域，通过策略控制不同用户看到不同套餐 |
| **推荐位** | APP 推荐场景的套餐展示区域，通过漏斗 + 策略控制套餐推荐 |
| **广告位** | APP 内预留的广告展示区域（开屏、插屏、Banner 等），通过广告策略控制展示规则 |
| **活动位** | APP 首页活动弹窗区域，通过活动策略控制活动投放 |
| **投放范围** | 套餐/策略/广告可投放的目标 APP 集合 |
| **混合运营** | 一个企业下同时存在代运营 APP 和自运营 APP 的场景 |

## 4. 核心业务规则

### 4.1 APP 分类规则

```
APP 运营类型
├── 代运营 APP（平台运营）
│   ├── 商城位运营 ✓
│   ├── 推荐位运营 ✓
│   └── 活动位运营 ✓
│
└── 企业自运营 APP（企业自管）
    └── 套餐配置 ✓（仅此一项）
```

### 4.2 运营能力矩阵

| 能力模块 | 代运营 APP | 企业自运营 APP |
|----------|:----------:|:--------------:|
| 套餐配置 | ✓ | ✓ |
| 商城位运营 | ✓ | ✗ |
| 推荐位运营 | ✓ | ✗ |
| 广告位运营 | ✓（后续版本） | ✗ |
| 活动位运营 | ✓ | ✗ |

### 4.3 套餐配置规则

- **创建套餐时**：可选择投放的 APP，但仅限「企业自运营 APP」。代运营 APP 的套餐由平台运营方统一配置
- **套餐上架配置**：区域、APP、运营位（商城位/推荐位）的选择范围仅包含企业自运营 APP
- **代运营 APP 的套餐**由平台运营方在商城位/推荐位策略中统一管理，不与具体企业挂钩

### 4.4 企业多 APP 混合场景

一个企业下可能有多个 APP：

```
企业A
├── APP-1（代运营）→ 平台运营方管理四个运营位
├── APP-2（自运营）→ 企业A自行管理套餐配置
└── APP-3（自运营）→ 企业A自行管理套餐配置
```

- 企业A 登录平台后，仅可见 APP-2、APP-3 的套餐配置功能
- APP-1 的运营投放由平台运营方独立管理，企业A 不可见、不可操作
- 平台运营方可查看和管理所有 APP

### 5.2 本次新增/调整（仅文档规则说明）

本次不涉及新功能开发，仅梳理和文档化以下规则：

1. **APP 运营类型属性**：APP 增加「运营类型」标识（代运营 / 自运营）
2. **运营模块可见性规则**：根据 APP 运营类型决定可用的运营模块
3. **套餐投放范围规则**：创建套餐时仅可选择自运营 APP
4. **代运营 APP 运营规则**：明确四个运营位的投放规则与关系

## 6. 详细场景分析

### 6.1 代运营 APP 场景

#### S01: 平台为代运营 APP 配置全套运营位

1. 平台运营管理员在「应用管理」中创建 APP，运营类型选择「代运营」
2. 该 APP 在运营管理菜单下生效四个运营模块：
   - 商城位运营 → 可为该 APP 创建商城策略
   - 推荐位运营 → 可为该 APP 创建推荐漏斗和策略
   - 广告位运营 → 可为该 APP 配置广告位和广告策略
   - 活动位运营 → 可为该 APP 创建活动策略
3. 各运营位策略中，「投放 APP」可选择该代运营 APP

#### S02: 平台为代运营 APP 配置商城位

1. 进入「商城位运营」→ 左侧 APP 列表显示所有代运营 APP
2. 选择目标 APP，添加商城策略
3. 配置投放区域、投放用户分群
4. 进入套餐配置页，从套餐池选择套餐上架
5. 终端用户在 APP 商城页看到对应套餐

#### S03: 平台为代运营 APP 配置活动位

1. 进入「活动位运营」→ 添加活动策略
2. 配置活动周期、弹窗频率、活动地址
3. 投放配置中选择代运营 APP
4. 发布活动 → 终端用户在 APP 首页看到活动弹窗

### 6.2 企业自运营 APP 场景

#### S04: 企业创建自运营 APP 并配置套餐

1. 企业在平台注册 APP，运营类型为「自运营」
2. 平台为该企业开通套餐管理权限
3. 企业登录后，仅可见「套餐管理」菜单
4. 企业创建套餐时：
   - 可绑定设备数、定价、内购 Key 等配置正常填写
   - 上架配置中，可选 APP 仅为企业名下的自运营 APP
   - 不可选择代运营 APP 或其他企业的 APP

#### S05: 企业有多 APP 混合运营

1. 企业A 有 3 个 APP：APP-1（代运营）、APP-2（自运营）、APP-3（自运营）
2. 企业A 登录平台，套餐管理模块中仅显示 APP-2、APP-3
3. 创建套餐时，投放 APP 仅可选 APP-2、APP-3
4. APP-1 的套餐由平台运营方通过商城位/推荐位策略统一配置
5. 企业A 不可见 APP-1 的任何运营配置

### 6.3 套餐与运营位联动场景

#### S06: 套餐被商城位策略引用

1. 平台在套餐管理中创建套餐 A，投放范围为代运营 APP
2. 在商城位运营中，将套餐 A 加入某代运营 APP 的商城策略
3. 套餐 A 的上架/下架/编辑影响商城位展示
4. 企业自运营 APP 的套餐与代运营 APP 的套餐池相互独立

#### S07: 套餐被自运营企业使用

1. 企业B 创建套餐 B，投放范围为企业B 的自运营 APP
2. 套餐 B 只对企业B 的 APP 生效
3. 平台运营方在商城位策略中看不到套餐 B（不在代运营套餐池内）
4. 各企业套餐数据隔离

## 7. 关键规则总结

| 规则编号 | 规则描述 |
|----------|----------|
| R01 | 平台审核 APP 时设定运营类型：代运营 / 自运营 |
| R02 | 代运营 APP 开放全部三个运营位：商城位、推荐位、活动位（广告位暂不纳入本期） |
| R03 | 企业自运营 APP 仅开放套餐配置模块 |
| R04 | 平台创建套餐时可选择代运营 APP 作为投放范围；企业创建套餐时仅可选择名下自运营 APP |
| R05 | 代运营 APP 的套餐和运营策略由平台运营方统一管理 |
| R06 | 企业创建套餐、配置套餐投放 APP 时，仅可选择该企业名下的自运营 APP |
| R07 | 企业仅可见名下自运营 APP 的套餐管理，不可见代运营 APP 的任何配置 |
| R08 | APP 运营类型允许变更，仅平台运营方在审核环节可变更，企业不可操作 |

## 8. 确认结论

| 议题 | 结论 |
|------|------|
| APP 运营类型是否允许变更 | **允许**，变更后按新类型的运营能力边界生效 |
| 广告位运营模块是否纳入本期 | **不纳入**，待后续版本开发 |
| 企业自运营是否需要用户分群 | **不需要**，用户分群仅平台运营侧使用 |
| 企业自运营套餐是否需要平台审核 | **不需要**，企业自行管理套餐上下架 |
