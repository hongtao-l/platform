# 活动位运营 — UI设计

## 1. 设计概述

### 1.1 设计依据
基于 [商城位运营 UI 模式](../../../project/design/platform_pkg/ops-mall.html)，复用其策略列表 + 步骤式抽屉的成熟交互模式，新增活动周期、弹窗频率、活动地址选择、活动状态流转等差异化功能。

### 1.2 设计目标
- 与商城位运营保持一致的视觉风格和交互习惯
- 活动特有字段清晰突出，不增加认知负担
- 状态流转可视化
- 活动地址独立管理

## 5. 交互说明

### 5.1 提交策略
1. 3步表单全部填写完毕，点击"提交"/"发布"
2. 前端校验：必填项、日期合法性、频率范围、地址是否选择
3. 校验通过 → 策略创建，状态设为"草稿"；点击"发布" → 状态变为"进行中"
4. 关闭抽屉，列表刷新

### 5.2 弹窗频率
- 数字输入框，范围 1-720 小时
- 默认值 24 小时

### 5.3 活动地址选择
- 下拉选择已配置的地址
- 选中后显示地址预览

### 5.4 状态操作
- 列表操作列动态渲染按钮
- 草稿策略 → 可见"发布"按钮，点击后状态变"进行中"
- 进行中策略 → 可"暂停"，暂停后可"恢复"或"结束"
- 已结束/已过期策略 → 可"复制新建"

### 5.5 活动到期处理
- 前端定时检查，到达 endTime 自动将"进行中"改为"已过期"
- 列表样式置灰

### 8.3 ActivityPopup — 首页弹窗

**组件定位**：展示有效活动弹窗，支持频率控制。

**UI 规格**：
```
┌───────────────────────┐
│                       │
│  ┌─────────────────┐  │
│  │                  │  │
│  │   [活动素材]      │  │  ← 300×200 比例
│  │                  │  │    渐变背景兜底
│  │                  │  │
│  └─────────────────┘  │
│                       │
│   618大促弹窗活动       │  ← 活动名称 title
│                       │
│  ┌─────────────────┐  │
│  │    立即查看       │  │  ← van-button round block primary
│  └─────────────────┘  │
│       暂不参与         │  ← 灰色文字链接
│                       │
│                    ✕  │  ← 右上角关闭
└───────────────────────┘
  van-popup: position="center", round, width: 300px
```

| 属性 | 值 |
|------|------|
| 弹窗宽度 | 300px |
| 弹出延迟 | 500ms |
| "立即查看" | 跳转活动地址 + 关闭弹窗 + 记录 dismiss |
| "暂不参与" | 关闭弹窗 + 记录 dismiss |
| 关闭按钮 | 同"暂不参与" |

**频率控制**：
- 弹窗关闭后记录时间戳到 localStorage
- 距上次弹窗超过 `popupInterval` 小时才再次弹窗

**状态定义**：
- **有弹窗活动 + 未 dismissed**：延迟 500ms 弹出
- **同一会话已弹过**：`sessionDismissed` flag，不重复弹
- **间隔未到**：不弹
- **无有效弹窗活动**：不弹

### 8.5 交互流程总览

```
首页加载
  │
  ├─→ ActivityPopup: watch activePopups + forcePopup
  │     └─→ 有数据 + 未dismiss + 间隔满足 → 延迟500ms → 弹窗
  │           ├─→ "立即查看" → 跳转 + dismiss
  │           └─→ "暂不参与" → dismiss
  │
  └─→ ActivityDemoPanel: 手动操作
        ├─→ 添加演示 → store.addDemo()
        ├─→ 触发弹窗 → store.triggerPopupNow()
        └─→ 重置 → store.resetAll() + initMockData()
```
