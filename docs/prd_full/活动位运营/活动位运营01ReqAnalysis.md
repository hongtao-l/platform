# 活动位运营 — 需求分析

## 1. 业务背景与目标

### 1.1 背景
平台运营人员在"商城位运营"和"推荐位运营"的基础上，需要一个新的"活动位运营"模块来管理 App 内的营销活动展示。活动位不同于商城位（长期套餐售卖）和推荐位（内容推荐），活动位有明确的**活动周期**、**多样化展示位置**（banner、弹窗等）、**活动媒资**（图片、视频素材）以及**活动落地链接**。

### 1.2 目标
- 提供统一的活动位策略配置能力，支持按区域、APP、用户分群精准投放活动
- 支持多种活动展示位置（banner、弹窗），每种位置对应不同的媒资规格
- 支持活动周期管控，超期自动下线
- 支持活动套餐绑定，活动可关联套餐促进转化
- 支持活动状态流转：草稿 → 进行中 → 已暂停/已结束/已过期

## 2. 角色与职责

| 角色 | 职责 |
|------|------|
| 运营管理员 | 创建/编辑/删除活动策略，配置活动位、活动周期、媒资、链接、套餐 |
| 运营管理员 | 发布活动（草稿→进行中）/ 暂停 / 恢复 / 结束 |
| 终端用户 | 在 App 内看到投放的活动（banner/弹窗），点击进入活动页 |

## 3. 名词解释

| 术语 | 说明 |
|------|------|
| **活动位** | App 内用于展示营销活动的固定/动态位置，如首页 banner、启动弹窗等 |
| **活动策略** | 定义一套完整的活动投放规则：什么时候、在哪个位置、向哪些用户、展示什么内容 |
| **活动周期** | 活动生效的时间范围 [startTime, endTime]，超期自动下线 |
| **活动位置** | 活动展示的具体位置类型：banner（横幅）、popup（弹窗）、floating（悬浮） |
| **活动媒资** | 活动展示所需的图片/视频素材，不同位置有不同尺寸规格 |
| **活动套餐** | 活动关联的套餐（复用商城位套餐体系），活动期间可进行套餐促销 |
| **活动链接** | 用户点击活动后跳转的目标 URL（H5 活动页 / 原生页面路由） |
| **活动状态** | 草稿 → 进行中 / 已暂停 / 已结束 / 已过期 |

## 4. 核心业务流程

```
创建活动策略 → 配置基础信息 → 配置活动周期 → 配置投放位置(Banner/弹窗等)
→ 配置投放区域 → 配置投放APP → 配置投放用户 → 配置媒资素材
→ 配置活动链接 → 配置活动套餐 → 发布 → 活动上线
→ 活动到期自动下线
```

## 5. 功能架构

### 5.1 页面结构
- **活动位策略列表** — 主页面，展示所有活动策略，支持筛选/搜索
- **添加/编辑活动策略** — 步骤式抽屉（3步）：
  - Step 1: 基础信息（活动名称、活动周期、活动优先级、活动备注）
  - Step 2: 投放配置（投放位置、投放区域、投放APP、投放用户分群）
  - Step 3: 活动内容（活动媒资、活动链接、活动套餐配置）
  - 提交按钮：草稿 → 保存草稿；信息完整 → 发布
- **套餐配置页** — 复用商城位套餐穿梭框模式

### 5.2 活动位置枚举
| 位置编码 | 位置名称 | 媒资规格 | 说明 |
|----------|----------|----------|------|
| `slot_banner` | 首页Banner | 750×400 | 全宽横幅轮播 |
| `slot_popup` | 启动弹窗 | 600×800 | 应用启动时弹出 |
| `slot_floating` | 悬浮按钮 | 120×120 | 页面右下角悬浮 |

### 5.3 活动状态流转
```
草稿(draft) ──发布──→ 进行中(active) ──手动暂停──→ 已暂停(paused)
  │                       │                          │
  └──保存草稿             └──到期自动──→ 已过期(expired)
                         └──手动结束──→ 已结束(ended)
```

## 6. 详细场景分析

### 6.1 正常场景

#### S01: 创建活动策略
1. 运营管理员点击"添加活动"
2. 输入活动名称（如"618大促Banner活动"）
3. 设置活动周期：2026-06-01 ~ 2026-06-18
4. 选择投放位置：首页Banner
5. 选择投放区域：国内
6. 选择投放APP：牵心PRO
7. 选择投放用户：高价值用户、活跃用户
8. 上传Banner媒资图片（750×400）
9. 填写活动链接：`https://m.example.com/activity/618`
10. 配置关联套餐（可选）
11. 发布 → 状态变为"进行中"

#### S02: 发布上线
1. 运营管理员确认活动配置无误
2. 点击"发布" → 状态变为"进行中"
3. 活动在生效时间到达后自动出现在 App 首页 Banner

#### S03: 活动到期自动下线
1. 活动结束时间到达
2. 系统自动将状态改为"已过期"
3. App 不再展示该活动

#### S04: 弹窗活动
1. 创建活动，选择投放位置为"启动弹窗"
2. 上传弹窗媒资（600×800）
3. 配置弹窗频率：每次启动/每天一次/每周一次
4. 活动上线后，用户打开 App 看到弹窗

### 6.2 异常场景

#### E01: 活动周期冲突
同一位置 + 同一APP + 同一区域的两个活动周期重叠时，系统提示冲突，按优先级高者展示。

#### E02: 媒资尺寸不符
上传媒资时校验尺寸，不符合该位置规格时提示并拒绝上传。

#### E03: 链接校验
活动链接必须以 `http://` 或 `https://` 开头，支持 App 内部路由 scheme（如 `carecam://...`）。

#### E04: 发布前校验
发布时前端校验必填项（名称、周期、位置、区域、APP、分群、媒资、链接），校验不通过则阻止发布并定位到对应步骤。

## 7. 数据模型

### 7.1 活动策略 (ActivityStrategy)
```javascript
{
  id: String,           // 策略ID AS_XXXX
  name: String,         // 活动名称
  status: String,       // draft | active | paused | ended | expired
  startTime: String,    // 活动开始时间 ISO-8601
  endTime: String,      // 活动结束时间 ISO-8601
  position: String,     // 投放位置 slot_banner | slot_popup | slot_floating
  regions: [String],    // ['国内'] | ['海外'] | ['国内','海外']
  apps: [String],       // ['牵心PRO'] | ['鹤梦之家'] | ['牵心PRO','鹤梦之家']
  userGroups: [String], // 用户分群ID数组
  mediaUrl: String,     // 活动媒资URL
  mediaSize: String,    // 媒资实际尺寸（校验用）
  activityUrl: String,  // 活动链接
  popupFrequency: String, // 弹窗频率: 'always' | 'once_daily' | 'once_weekly'（仅popup位置）
  packages: [Object],   // 关联套餐列表
  remark: String,       // 备注
  createdBy: String,    // 创建人
  createdAt: String,    // 创建时间
  priority: Number,     // 优先级（数字越大越高）
}
```

### 7.2 活动套餐关联
复用商城位套餐体系，`packages` 字段引用套餐ID列表。

## 8. 业务规则

| 规则编号 | 规则内容 |
|----------|----------|
| R01 | 活动名称不能为空，最长 50 字 |
| R02 | 活动结束时间必须晚于开始时间 |
| R03 | 每个策略至少选择一个投放位置 |
| R04 | 每个策略至少选择一个投放区域 |
| R05 | 每个策略至少选择一个投放APP |
| R06 | 每个策略至少选择一个投放用户分群 |
| R07 | 媒资图片必须符合所选位置的尺寸规格 |
| R08 | 活动链接不能为空 |
| R09 | 同一位置冲突时按优先级和创建时间排序 |
| R10 | 已结束/已过期的活动不可编辑，只能复制新建 |
| R11 | 进行中活动不可删除，需先暂停或结束 |
| R12 | 删除活动为软删除，保留记录 |

## 9. 用户分群

复用现有商城位运营的 5 个用户分群：
- 高价值用户 (`ug_high_value`)
- 新注册用户 (`ug_new_reg`)
- 活跃用户 (`ug_active`)
- 沉默用户 (`ug_silent`)
- 付费用户 (`ug_paid`)

## 10. 异常场景总览

| 异常 | 处理方式 |
|------|----------|
| 媒资上传失败 | 提示重试，不阻塞策略提交 |
| 活动周期重叠 | 前端提示，展示冲突列表，确认后仍可提交 |
| 活动已过期仍展示 | 后端定时任务 + 客户端本地时间双校验 |
| 进行中活动误删除 | 前端置灰删除按钮，后端校验拦截 |

## 11. APP侧 — 活动展示场景

### S05: 首页Banner轮播
1. 用户打开 App 进入首页
2. App 请求活动列表 API（`GET /activity/strategy/list?status=active&app=牵心PRO`）
3. 过滤出 `position=slot_banner` 且在有效期内的活动
4. 在首页家庭分组标签下方展示 `<van-swipe>` 轮播横幅
5. 按 priority 降序排列，自动轮播（3s 间隔）
6. 用户点击 Banner → 跳转 `activityUrl`（内部路由或外部 H5）
7. 无有效 Banner 时，区域不渲染

### S06: 启动弹窗
1. 用户打开 App 进入首页
2. 检查是否有 `position=slot_popup` 的有效活动
3. 过滤规则：
   - 活动在时间范围内
   - `popupFrequency === 'always'` → 每次都弹
   - `popupFrequency === 'once_daily'` → 当天未关闭才弹
   - `popupFrequency === 'once_weekly'` → 本周未关闭才弹
4. 延迟 500ms 弹出居中弹窗（避免与页面加载动画冲突）
5. 弹窗展示：媒资图片 + 活动名称 + "立即查看"按钮 + "暂不参与"
6. 用户点击"立即查看" → 跳转链接 + 关闭弹窗 + 记录当日关闭
7. 用户点击"暂不参与"或关闭按钮 → 关闭弹窗 + 记录当日关闭
8. 同一会话只弹一次（session 级标记）

### S07: 悬浮按钮
1. 用户进入首页
2. 检查是否有 `position=slot_floating` 的有效活动
3. 取 priority 最高的一条
4. 在首页右下角（bottom: 80px, right: 16px）展示圆形悬浮按钮
5. 按钮显示活动媒资图片（120×120），无图片时显示默认图标
6. 用户点击 → 跳转链接
7. 悬浮按钮右上角有关闭按钮（×），点击后当天隐藏
8. 无有效悬浮活动时不渲染

## 12. APP侧业务规则

| 规则编号 | 规则内容 |
|----------|----------|
| R13 | APP 仅拉取 `status=active` 且在 `[startTime, endTime]` 范围内的活动 |
| R14 | 客户端本地时间与服务器时间双重校验，防止时区偏差导致过期活动仍展示 |
| R15 | 弹窗关闭记录存储在 localStorage，key 为 `carecam_activity_dismiss` |
| R16 | `once_daily` 频率：以日期 `YYYY-MM-DD` 为粒度判断 |
| R17 | `once_weekly` 频率：以周一日期为粒度判断 |
| R18 | 同一位置多个活动按 priority 降序排列，Banner 全部轮播，弹窗和悬浮只取最高优先级 |
| R19 | Banner 图片加载失败时显示渐变占位 + 活动标题 |
| R20 | API 不可用时降级使用前端内置 Mock 数据 |

## 13. 技术架构

```
iot-platform (管理后台)          carecam_pro (APP端)
  ├── 活动策略CRUD               ├── ActivityBanner (轮播)
  ├── 状态流转(草稿→进行中)       ├── ActivityPopup (弹窗)
  ├── 套餐关联                   ├── ActivityFab (悬浮按钮)
  ├── 状态流转(草稿→进行中)       ├── Pinia Store (状态管理)
  └── 冲突检测                   └── localStorage (关闭缓存)
           │                              │
           └────────── API ───────────────┘
                GET /activity/strategy/list
```
