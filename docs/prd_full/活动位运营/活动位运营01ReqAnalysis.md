# 活动位运营 — 需求分析

## 1. 业务背景与目标

### 1.1 背景
平台运营人员在"商城位运营"和"推荐位运营"的基础上，需要一个新的"活动位运营"模块来管理 App 内的营销活动展示。活动位不同于商城位（长期套餐售卖）和推荐位（内容推荐），活动位有明确的**活动周期**、**活动弹窗**、**活动落地地址**。

### 1.2 目标
- 提供统一的活动位策略配置能力，支持按区域、APP、用户分群精准投放活动
- 支持首页弹窗活动，配置弹窗频率（每N小时1次）
- 支持活动周期管控，超期自动下线
- 支持活动地址管理（H5地址配置）
- 支持活动状态流转：草稿 → 进行中 → 已暂停/已结束/已过期

## 2. 角色与职责

| 角色 | 职责 |
|------|------|
| 运营管理员 | 创建/编辑/删除活动策略，配置活动周期、弹窗频率、活动地址 |
| 运营管理员 | 发布活动（草稿→进行中）/ 暂停 / 恢复 / 结束 |
| 终端用户 | 在 App 首页看到活动弹窗，点击跳转活动页 |

## 3. 名词解释

| 术语 | 说明 |
|------|------|
| **活动位** | App 首页用于展示营销活动的弹窗位置 |
| **活动策略** | 定义一套完整的活动投放规则：什么时候、向哪些用户、展示什么内容 |
| **活动周期** | 活动生效的时间范围 [startTime, endTime]，超期自动下线 |
| **弹窗频率** | 弹窗展示频率，每N小时弹1次 |
| **活动地址** | 用户点击活动后跳转的目标 H5 地址 |
| **活动状态** | 草稿 → 进行中 / 已暂停 / 已结束 / 已过期 |

## 4. 核心业务流程

```
创建活动策略 → 配置基础信息 → 配置活动周期 → 配置弹窗频率
→ 配置投放区域 → 配置投放APP → 配置投放用户 → 选择活动地址
→ 发布 → 活动上线 → 活动到期自动下线
```

## 5. 功能架构

### 5.1 页面结构
- **活动位策略列表** — 主页面，展示所有活动策略，支持筛选/搜索
- **添加/编辑活动策略** — 步骤式抽屉（3步）：
  - Step 1: 基础信息（活动名称、活动周期、活动优先级、活动备注）
  - Step 2: 投放配置（弹窗频率、投放区域、投放APP、投放用户分群）
  - Step 3: 活动内容（选择活动地址）
  - 提交按钮：草稿 → 保存草稿；信息完整 → 发布
- **活动地址配置页** — 管理活动H5地址列表

### 5.2 活动状态流转
```
草稿(draft) ──发布──→ 进行中(active) ──手动暂停──→ 已暂停(paused)
  │                       │                          │
  └──保存草稿             └──到期自动──→ 已过期(expired)
                         └──手动结束──→ 已结束(ended)
```

## 6. 详细场景分析

### 6.1 正常场景

#### S01: 创建活动策略
1. 运营管理员点击"添加活动"
2. 输入活动名称（如"618大促弹窗活动"）
3. 设置活动周期：2026-06-01 ~ 2026-06-18
4. 配置弹窗频率：每24小时1次
5. 选择投放区域：国内
6. 选择投放APP：牵心PRO
7. 选择投放用户：高价值用户、活跃用户
8. 选择活动地址：618大促活动页
9. 发布 → 状态变为"进行中"

#### S02: 发布上线
1. 运营管理员确认活动配置无误
2. 点击"发布" → 状态变为"进行中"
3. 活动在生效时间到达后，用户打开 App 时首页弹出活动弹窗

#### S03: 活动到期自动下线
1. 活动结束时间到达
2. 系统自动将状态改为"已过期"
3. App 不再展示该活动

### 6.2 异常场景

#### E01: 活动周期冲突
同一APP + 同一区域的两个活动周期重叠时，系统提示冲突，按优先级高者展示。

#### E02: 地址无效
活动地址必须以 `http://` 或 `https://` 开头。

#### E03: 发布前校验
发布时前端校验必填项（名称、周期、频率、区域、APP、分群、地址），校验不通过则阻止发布并定位到对应步骤。

## 7. 数据模型

### 7.1 活动策略 (ActivityStrategy)
```javascript
{
  id: String,           // 策略ID AS_XXXX
  name: String,         // 活动名称
  status: String,       // draft | active | paused | ended | expired
  startTime: String,    // 活动开始时间 ISO-8601
  endTime: String,      // 活动结束时间 ISO-8601
  regions: [String],    // ['国内'] | ['海外'] | ['国内','海外']
  apps: [String],       // ['牵心PRO'] | ['鹤梦之家'] | ['牵心PRO','鹤梦之家']
  userGroups: [String], // 用户分群ID数组
  addressId: String,    // 活动地址ID
  popupInterval: Number,// 弹窗频率（每N小时1次）
  remark: String,       // 备注
  createdBy: String,    // 创建人
  createdAt: String,    // 创建时间
  priority: Number,     // 优先级（数字越大越高）
}
```

### 7.2 活动地址 (ActivityAddress)
```javascript
{
  id: String,           // 地址ID
  name: String,         // 地址名称
  url: String,          // H5地址
}
```

## 8. 业务规则

| 规则编号 | 规则内容 |
|----------|----------|
| R01 | 活动名称不能为空，最长 50 字 |
| R02 | 活动结束时间必须晚于开始时间 |
| R03 | 弹窗频率必须设置（1-720小时） |
| R04 | 每个策略至少选择一个投放区域 |
| R05 | 每个策略至少选择一个投放APP |
| R06 | 每个策略至少选择一个投放用户分群 |
| R07 | 活动地址不能为空 |
| R08 | 同一APP+区域冲突时按优先级和创建时间排序 |
| R09 | 已结束/已过期的活动不可编辑，只能复制新建 |
| R10 | 进行中活动不可删除，需先暂停或结束 |
| R11 | 删除活动为软删除，保留记录 |

## 9. 用户分群

复用现有商城位运营的 5 个用户分群：
- 高价值用户 (`ug_high_value`)
- 新注册用户 (`ug_new_reg`)
- 活跃用户 (`ug_active`)
- 沉默用户 (`ug_silent`)
- 付费用户 (`ug_paid`)

## 10. 异常场景总览

| 异常 | 处理方式 |
|------|----------|
| 活动地址无效 | 前端校验URL格式，提示修改 |
| 活动周期重叠 | 前端提示，展示冲突列表，确认后仍可提交 |
| 活动已过期仍展示 | 后端定时任务 + 客户端本地时间双校验 |
| 进行中活动误删除 | 前端置灰删除按钮，后端校验拦截 |

## 11. APP侧 — 活动展示场景

### S04: 首页弹窗
1. 用户打开 App 进入首页
2. App 请求活动列表 API，获取有效活动
3. 过滤规则：活动在时间范围内，且距上次弹窗超过 popupInterval 小时
4. 延迟 500ms 弹出居中弹窗（避免与页面加载动画冲突）
5. 弹窗展示：活动名称 + "立即查看"按钮 + "暂不参与"
6. 用户点击"立即查看" → 跳转活动地址
7. 用户点击"暂不参与"或关闭按钮 → 关闭弹窗 + 记录关闭时间
8. 同一会话只弹一次（session 级标记）

## 12. APP侧业务规则

| 规则编号 | 规则内容 |
|----------|----------|
| R12 | APP 仅拉取 `status=active` 且在 `[startTime, endTime]` 范围内的活动 |
| R13 | 客户端本地时间与服务器时间双重校验，防止时区偏差导致过期活动仍展示 |
| R14 | 弹窗关闭记录存储在 localStorage，key 为 `carecam_activity_dismiss` |
| R15 | 弹窗频率以小时为粒度，距上次弹窗超过 popupInterval 小时才弹 |
| R16 | 多个活动按 priority 降序排列，取最高优先级 |
| R17 | API 不可用时降级使用前端内置 Mock 数据 |

## 13. 技术架构

```
iot-platform (管理后台)          carecam_pro (APP端)
  ├── 活动策略CRUD               ├── ActivityPopup (弹窗)
  ├── 活动地址管理                ├── Pinia Store (状态管理)
  ├── 状态流转(草稿→进行中)       └── localStorage (关闭缓存)
  └── 冲突检测                           │
           │                              │
           └────────── API ───────────────┘
                GET /activity/strategy/list
```
