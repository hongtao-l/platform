## 一、业务背景

平台运营在"商城位运营"和"推荐位运营"基础上，需要一个独立的"活动位运营"模块来管理 App 内的营销活动展示。

**与商城位、推荐位的本质区别**：
- **商城位**：长期套餐售卖，无时效限制，重在套餐内容
- **推荐位**：内容/设备推荐，重在个性化匹配
- **活动位**：营销活动推广，有明确的活动周期、多样化展示位置（Banner/弹窗/悬浮/侧边栏）、活动媒资素材和落地链接

**要解决的痛点**：
- 活动投放散乱，缺乏统一的策略配置管理
- 活动到期后仍展示，引发用户投诉和合规风险
- 不同展示位置的媒资规格不统一，运营上传尺寸混乱
- 无法按区域、APP、用户分群精准投放活动
- 活动效果缺少数据追踪

**产品目标**：
- 提供统一的活动位策略配置后台，支持按区域、APP、用户分群精准投放
- 支持多种展示位置（Banner、弹窗、悬浮按钮、侧边栏），每种位置对应不同媒资规格
- 活动周期管控，超期自动下线
- 活动状态流转：草稿 → 进行中 → 已暂停/已结束/已过期
- APP 端展示效果与后台配置无缝衔接

## 二、名词解释

| 术语 | 说明 |
|------|------|
| 活动位 | App 内用于展示营销活动的固定/动态位置，如首页 Banner、启动弹窗、悬浮按钮、侧边栏入口 |
| 活动策略 | 一套完整的活动投放规则：什么时间、在什么位置、向哪些用户、展示什么内容 |
| 活动周期 | 活动生效的时间范围 [startTime, endTime]，超期自动下线 |
| 活动位置 | 活动展示的具体位置类型：Banner、弹窗、悬浮按钮、侧边栏 |
| 活动媒资 | 活动展示所需的图片/视频素材，不同位置有不同尺寸规格 |
| 活动套餐 | 活动关联的套餐（复用商城位套餐体系），活动期间进行套餐促销 |
| 活动链接 | 用户点击活动后跳转的目标 URL（H5 活动页 / App 内部路由） |
| 活动状态 | 草稿 → 进行中 / 已暂停 / 已结束 / 已过期 |
| 弹窗频率 | 弹窗活动的展示频率：每次启动 / 每天一次 / 每周一次 |
| 用户分群 | 投放的目标用户群体：高价值用户、新注册用户、活跃用户、沉默用户、付费用户 |

## 三、业务实体说明

### 3.1 核心实体

**活动策略（ActivityStrategy）**
- 属性：名称、状态（draft/active/paused/ended/expired）、开始/结束时间、投放位置、投放区域、投放APP、用户分群、媒资URL、活动链接、弹窗频率、套餐列表、优先级、备注
- 生命周期：草稿 → 进行中 ⇄ 已暂停 → 已结束/已过期
- 唯一标识：AS_XXXX 格式策略ID

**活动位置（ActivitySlot）**
- 4种固定类型：Banner（750×400）、弹窗（600×800）、悬浮按钮（120×120）、侧边栏入口（200×200）
- 每种位置有固定媒资尺寸规格，上传时校验

**活动套餐关联**
- 复用商城位套餐体系，活动策略可关联 0~N 个套餐
- 套餐支持拖拽排序，影响 APP 端套餐展示优先级

### 3.2 实体关系

```
活动策略 1──1 活动位置（4选1）
活动策略 1──N 投放区域（国内/海外，至少选1）
活动策略 1──N 投放APP（牵心PRO/鹤梦之家，至少选1）
活动策略 1──N 用户分群（5个分群，至少选1）
活动策略 1──N 套餐（可选，拖拽排序）
```

## 四、核心业务流程

### 4.1 活动策略全生命周期

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1. 创建   │ →  │ 2. 发布   │ →  │ 3. 运行   │ →  │ 4. 结束   │
│ 填写配置  │    │ 状态→进行中│    │ 用户可见  │    │ 自动/手动 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                      │
                                      └─ 手动暂停 ⇄ 恢复
```

### 4.2 3步创建流程

```
Step 1: 基础信息
  → 活动名称（必填，最长50字）
  → 活动周期（必填，结束>开始）
  → 活动优先级（默认50，1-100）
  → 备注（选填）

Step 2: 投放配置
  → 投放位置（4选1，卡片式单选）
  → 投放区域（多选，至少选1）
  → 投放APP（多选，至少选1）
  → 投放用户分群（多选，至少选1）
  → 弹窗频率（conditional：仅弹窗位置显示）

Step 3: 活动内容
  → 活动媒资（必填，尺寸校验联动位置）
  → 活动链接（必填，http/https/scheme校验）
  → 关联套餐（选填，穿梭框选择，拖拽排序）
  → 提交 → 状态为"草稿"；发布 → 状态变为"进行中"
```

### 4.3 活动状态流转

```
草稿(draft) ──发布──→ 进行中(active) ──手动暂停──→ 已暂停(paused)
  │                       │                          │
  └──保存草稿             └──到期自动──→ 已过期(expired)
                         └──手动结束──→ 已结束(ended)
```

### 4.4 APP 端展示流程

```
用户打开 App
  → APP 请求活动列表（status=active）
  → 本地时间 + 服务器时间双重校验有效期
  → 按位置分类处理：
      ├─ Banner: 按 priority 排序，van-swipe 轮播
      ├─ 弹窗: 取 priority 最高，检查频率 + dismiss 缓存，延迟弹出
      ├─ 悬浮: 取 priority 最高，右下角固定
      └─ 侧边栏: 入口展示
  → 用户点击 → 跳转活动链接
```

## 五、业务规则

### 5.1 策略管理

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 名称必填 | 活动名称不能为空，最长50字 |
| R02 | 周期合法 | 活动结束时间必须晚于开始时间 |
| R03 | 位置必选 | 每个策略至少选择一个投放位置 |
| R04 | 区域必选 | 每个策略至少选择一个投放区域 |
| R05 | APP必选 | 每个策略至少选择一个投放APP |
| R06 | 分群必选 | 每个策略至少选择一个用户分群 |
| R07 | 媒资尺寸校验 | 媒资图片必须符合所选位置的尺寸规格 |
| R08 | 链接必填 | 活动链接不能为空，须以 http://、https:// 或 App scheme 开头 |
| R09 | 冲突排序 | 同一位置冲突时按优先级和创建时间排序 |
| R10 | 终态不可编辑 | 已结束/已过期的活动不可编辑，只能复制新建 |
| R11 | 进行中不可删除 | 进行中活动需先暂停或结束后才能删除 |
| R12 | 软删除 | 删除活动为软删除，保留记录 |

### 5.2 APP 端展示

| 编号 | 规则 | 说明 |
|------|------|------|
| R13 | 仅拉取有效 | 仅拉取 status=active 且在 [startTime, endTime] 范围内的活动 |
| R14 | 双重校验 | 客户端本地时间与服务器时间双重校验，防止时区偏差 |
| R15 | 关闭缓存 | 弹窗关闭记录存储 key: `carecam_activity_dismiss` |
| R16 | 每日频率 | `once_daily` 以日期 YYYY-MM-DD 为粒度判断 |
| R17 | 每周频率 | `once_weekly` 以周一日期为粒度判断 |
| R18 | 优先级排序 | 同一位置多活动按 priority 降序，Banner 全部轮播，弹窗/悬浮取最高优先级 |
| R19 | 图片降级 | Banner 图片加载失败时显示渐变占位 + 活动标题 |
| R20 | API降级 | API 不可用时使用前端内置 Mock 数据 |

## 七、详细功能描述

### 7.1 活动策略列表

#### 功能应用场景
运营管理员查看所有活动策略，了解当前投放状态，进行筛选和管理操作。

#### 列表展示
- 每行显示：活动名称+ID、投放位置（彩色tag）、活动周期、状态标签、套餐数量、备注摘要、操作按钮
- 支持按状态/位置/区域/APP筛选
- 支持按活动名称搜索
- 列表按创建时间倒序排列

#### 操作按钮（按状态动态变化）

| 状态 | 可用操作 |
|------|----------|
| 草稿 | 编辑 + 删除 + 发布 |
| 进行中 | 查看 + 暂停 |
| 已暂停 | 查看 + 恢复 + 结束 |
| 已结束 | 查看 + 复制新建 |
| 已过期 | 查看 + 复制新建 |

---

### 7.2 创建/编辑活动策略

#### 功能应用场景
运营管理员创建新的活动投放策略，或编辑已有的草稿策略。

#### Step 1: 基础信息
- 活动名称：文本输入，最长50字符，必填
- 活动周期：日期时间范围选择器，必填，结束时间必须晚于开始时间
- 活动优先级：数字输入，范围 1-100，默认50，越高越优先展示
- 备注：文本域，选填

#### Step 2: 投放配置
- 投放位置：4张卡片单选，每张显示位置名称+尺寸规格，必选
- 投放区域：复选框（国内/海外），至少选1
- 投放APP：复选框（牵心PRO/鹤梦之家），至少选1
- 投放用户分群：复选框（5个分群），至少选1
- 弹窗频率（条件显示）：仅选"弹窗"位置时出现，单选：每次启动/每天一次/每周一次

#### Step 3: 活动内容
- 活动媒资：上传区，动态显示当前选择位置的尺寸规格，校验上传图片尺寸
- 活动链接：URL输入框，校验 http/https/scheme 格式，必填
- 活动套餐：选填，穿梭框选择后显示已选套餐列表，支持拖拽排序
- 保存草稿：校验通过后保存，状态为"草稿"
- 发布：校验通过后状态变为"进行中"

#### 编辑限制
- 已结束/已过期的策略不可编辑，只能复制新建
- 进行中的策略不可编辑

---

### 7.3 查看活动详情

#### 功能应用场景
运营管理员查看进行中/已结束/已过期活动策略的完整信息。

#### 详情抽屉
- 只读展示策略全部信息（名称、周期、位置、区域、APP、分群、媒资、链接、套餐）
- 包含操作按钮（按状态动态显示）

---

### 7.4 套餐配置

#### 功能应用场景
运营管理员为活动策略关联套餐，管理套餐展示顺序。

#### 功能要点
- 复用商城位套餐穿梭框，左侧可选套餐池，右侧已选套餐列表
- 已选套餐支持拖拽排序（九宫格拖拽手柄）
- 拖拽时的视觉反馈：拖拽行半透明 + 蓝色虚线，目标位置蓝色横线指示
- 支持分类筛选（云存储/AI/综合）

---

### 7.5 APP 端 — Banner 轮播

#### 功能应用场景
用户在 App 首页家庭分组标签下方看到活动横幅轮播，点击进入活动详情页。

#### 功能要点
- 仅展示 `position=slot_banner` 且在有效期内的活动
- 按 priority 降序排列，自动轮播（3秒间隔）
- 点击跳转 activityUrl（支持 http/https 外部链接和 App 内部路由）
- 图片加载失败时显示渐变占位 + 活动标题
- 无有效 Banner 时区域不渲染

---

### 7.6 APP 端 — 启动弹窗

#### 功能应用场景
用户打开 App 时看到活动弹窗，引导参与限时活动。

#### 功能要点
- 仅展示 `position=slot_popup` 且在有效期内的活动
- 取 priority 最高的一条
- 延迟 500ms 弹出（避免与页面加载动画冲突）
- 弹窗内容：媒资图片 + 活动名称 + "立即查看"按钮 + "暂不参与"
- 频率控制：`always` 每次都弹，`once_daily` 每天一次，`once_weekly` 每周一次
- 关闭记录存储在 localStorage
- 同一会话只弹一次

---

### 7.7 APP 端 — 悬浮按钮

#### 功能应用场景
用户在 App 首页右下角看到悬浮活动按钮，点击参与活动。

#### 功能要点
- 仅展示 `position=slot_floating` 且在有效期内的活动
- 取 priority 最高的一条
- 固定定位：bottom: 80px, right: 16px, z-index: 100
- 52×52px 圆形，显示活动媒资图片
- 右上角关闭按钮，点击后当天隐藏
- pulse 动画吸引注意
- 无有效悬浮活动时不渲染

## 九、异常说明

| 分类 | 场景 | 处理方式 |
|------|------|------|
| 网络异常 | 媒资上传失败 | 提示重试，不阻塞策略保存（可先保存草稿） |
| 数据异常 | 活动周期重叠 | 前端提示冲突列表，确认后仍可提交 |
| 数据异常 | 活动已过期仍展示 | 后端定时任务 + 客户端本地时间双校验 |
| 数据异常 | 发布时必填项缺失 | 前端校验，阻止发布并定位到对应步骤 |
| 网络异常 | API 不可用 | APP 降级使用前端内置 Mock 数据 |
| 操作异常 | 已结束活动被编辑 | 前端置灰编辑按钮，后端校验拦截 |
| 操作异常 | 进行中活动被删除 | 前端置灰删除按钮，后端校验拦截 |
| 极限场景 | 同一位置大量冲突 | 按 priority 降序 + 创建时间排序，前端提示 |
| 边界条件 | 活动周期跨年 | 正常支持，时间范围无年份限制 |
| 边界条件 | 弹窗 dismiss 缓存在 localStorage | 清理 APP 缓存后重置，属于预期行为 |
