## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 活动曝光UV | 行为 | COUNT(DISTINCT user_id) WHERE event=activity_view | 实时 |
| 2 | 活动曝光PV | 行为 | COUNT(event=activity_view) | 实时 |
| 3 | 活动点击UV | 行为 | COUNT(DISTINCT user_id) WHERE event=activity_click | 实时 |
| 4 | 活动点击率 | 业务 | 点击UV / 曝光UV × 100 | 日 |
| 5 | 活动转化UV | 业务 | COUNT(DISTINCT user_id) WHERE event=activity_conversion | 日 |
| 6 | 活动转化率 | 业务 | 转化UV / 点击UV × 100 | 日 |
| 7 | 弹窗关闭率 | 行为 | 弹窗关闭次数 / 弹窗曝光次数 × 100 | 日 |
| 8 | 套餐关联转化 | 业务 | 从活动页进入套餐购买的成功数 | 周 |
| 9 | 活动营收贡献 | 业务 | SUM(price) WHERE event_source=activity | 周 |
| 10 | 活动ROI | 业务 | 活动营收 / 活动成本 × 100 | 月 |

### 1.2 核心指标说明

**活动曝光UV/曝光PV**：衡量活动素材触达用户的数量。UV 用于统计有效触达用户数，PV 用于评估轮播/Banner 的重复曝光效果。

**活动点击率**：衡量活动素材对用户的吸引力。是判断活动设计（图片、标题、位置）是否有效的核心指标。Banner 点击率通常 3-8%，弹窗点击率 10-25%。

**弹窗关闭率**：关闭率过高说明弹窗频次过高或内容不匹配用户期望。健康值应 < 60%。

**活动转化率**：衡量活动落地页的转化效果。从点击到目标行为（购买/注册/下载）的完成率。

**套餐关联转化**：评估活动→套餐的引流效果，验证活动套餐绑定的 ROI。

## 二、埋点方案

### 2.1 事件清单

| 事件ID | 事件名称 | 触发时机 | event_type | 优先级 |
|------|------|------|------|------|
| activity_view | 活动曝光 | 活动素材可见（Banner进入视口、弹窗弹出、悬浮按钮渲染） | 0（普通事件） | P0 |
| activity_click | 活动点击 | 用户点击活动素材跳转 | 0（普通事件） | P0 |
| activity_dismiss | 活动关闭 | 弹窗点"暂不参与"、悬浮点关闭按钮 | 0（普通事件） | P1 |
| activity_conversion | 活动转化 | 用户通过活动完成目标行为（购买/注册） | 1（带额外参数） | P0 |
| activity_error | 活动异常 | 媒资加载失败、链接跳转失败 | 2（错误事件） | P1 |

### 2.2 公共参数（所有事件必带）

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| event_type | String | 是 | 0=普通事件，1=转化事件，2=错误事件 | "0" |
| event_id | String | 是 | 事件ID | "activity_view" |
| event_time | Int64 | 是 | 事件发生时间戳(ms) | 1715846400000 |
| user_id | String | 是 | 用户唯一标识 | "usr_123456" |
| session_id | String | 是 | 会话ID | "sess_abc123" |
| os_type | String | 是 | iOS / Android | "Android" |
| app_version | String | 是 | APP版本号 | "2.1.0" |

### 2.3 业务参数（按事件）

**activity_view**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| activity_id | String | 是 | 活动策略ID |
| position | String | 是 | 展示位置：slot_banner / slot_popup / slot_floating / slot_sidebar |
| activity_name | String | 是 | 活动名称 |
| priority | Number | 是 | 活动优先级 |
| user_group | String | 否 | 用户所属分群（如果可判断） |

**activity_click**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| activity_id | String | 是 | 活动策略ID |
| position | String | 是 | 展示位置 |
| activity_name | String | 是 | 活动名称 |
| target_url | String | 是 | 跳转目标URL |
| url_type | String | 是 | 链接类型：h5 / app_route / scheme |

**activity_dismiss**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| activity_id | String | 是 | 活动策略ID |
| position | String | 是 | 展示位置 |
| dismiss_reason | String | 是 | 关闭原因：skip / close_btn / timeout |
| frequency | String | 否 | 弹窗频率设置 |

**activity_conversion**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| activity_id | String | 是 | 活动策略ID |
| position | String | 是 | 展示位置 |
| conversion_type | String | 是 | 转化类型：purchase / register / download |
| conversion_value | Number | 否 | 转化金额（购买时） |
| order_id | String | 否 | 订单ID（购买时） |
| package_ids | [String] | 否 | 关联套餐ID列表 |

**activity_error**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| activity_id | String | 是 | 活动策略ID |
| error_type | String | 是 | 错误类型：media_load_fail / link_invalid / link_open_fail |
| error_msg | String | 是 | 错误描述 |

### 2.4 上报方式

APP 端在活动展示/点击/关闭/转化等关键节点直接上报事件至后台：

```
POST /api/activity/event
```

### 2.5 去重策略

| 场景 | 策略 |
|------|------|
| 同 user_id + activity_id + event_id 重复 | 1分钟内保留第一条 |
| activity_view 重复曝光 | 同一会话 + 同一活动 + 同一位置，10秒内去重 |
| activity_conversion 重复 | 同一 order_id 只保留第一条 |

### 2.6 数据校验

| 校验项 | 规则 | 异常处理 |
|------|------|------|
| 必填参数 | 公共7项 + 业务参数不得为空 | 丢弃并告警 |
| event_type | ∈ {0, 1, 2} | 标记异常 |
| position | ∈ {slot_banner, slot_popup, slot_floating, slot_sidebar} | 标记异常 |
| url_type | ∈ {h5, app_route, scheme} | 标记异常 |
| conversion_type | ∈ {purchase, register, download} | 标记异常 |
| error_type | ∈ {media_load_fail, link_invalid, link_open_fail} | 标记异常 |
| 时序校验 | activity_conversion.time ≥ activity_click.time | 标记异常 |
| 重复事件 | user_id + activity_id + event_id 1min去重 | 保留第一条 |
| 重复曝光 | 同会话同活动同位置10s去重 | 保留第一条 |
| 重复订单 | activity_conversion order_id去重 | 保留第一条 |

## 三、数据看板

### 3.1 看板概览

**看板名称**：活动位运营数据看板

**看板用途**：展示活动位运营的整体数据，按活动策略和位置维度分析曝光、点击、转化效果。

**目标查看人群**：运营经理、产品经理

**访问路径**：管理后台 → 运营管理 → 活动位运营 → 数据看板

| 报表名称 | 类型 | 内容 | 刷新频率 | 接收人 |
|------|------|------|------|------|

### 3.5 告警规则

| 告警名称 | 触发条件 | 级别 | 通知方式 |
|------|------|------|------|
| 点击率骤降 | 活动点击率环比下降 > 50% | P1 | 站内消息 |
| 活动媒资异常 | activity_error 5分钟 > 10条 | P1 | 站内 + 推送 |
| 活动到期未下线 | 过期活动仍有曝光上报 | P1 | 站内消息 |
| 转化断流 | 有曝光有点击但连续1小时无转化 | P2 | 站内消息 |
| 弹窗关闭率过高 | 弹窗关闭率 > 80% | P2 | 站内消息 |

## 五、活动效果评估标准

### 5.1 分位置效果基准

| 位置 | 优秀点击率 | 良好点击率 | 需优化 | 优秀关闭率 | 需优化关闭率 |
|------|------|------|------|------|------|
| Banner | > 8% | 4-8% | < 4% | — | — |
| 弹窗 | > 25% | 15-25% | < 15% | < 30% | > 60% |
| 悬浮按钮 | > 10% | 5-10% | < 5% | < 20% | > 50% |
| 侧边栏 | > 12% | 6-12% | < 6% | — | — |

### 5.2 活动效果评级

| 评级 | 条件 |
|------|------|
| S（卓越） | 点击率 > 基准优秀线 + 转化率 > 5% |
| A（优秀） | 点击率 > 基准优秀线 |
| B（良好） | 点击率在良好区间 |
| C（一般） | 点击率低于良好线但转化率 > 3% |
| D（待优化） | 点击率 < 良好线且转化率 < 3% |
