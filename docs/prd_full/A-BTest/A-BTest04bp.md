## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 总曝光UV | 行为 | COUNT(DISTINCT user_id) WHERE event=experiment_view | 实时 |
| 2 | 总转化UV | 业务 | COUNT(DISTINCT user_id) WHERE event=pay_success | 实时 |
| 3 | 整体转化率 | 业务 | 总转化UV / 总曝光UV × 100 | 日 |
| 4 | 总收入 | 业务 | SUM(price) WHERE event=pay_success | 日 |
| 5 | 变体A转化率 | 业务 | A的pay_success UV / A的experiment_view UV | 日 |
| 6 | 变体B转化率 | 业务 | B的pay_success UV / B的experiment_view UV | 日 |
| 7 | 套餐选择率 | 行为 | plan_select UV / experiment_view UV | 日 |
| 8 | 支付成功率 | 质量 | pay_success UV / pay_click UV | 日 |
| 9 | 客单价 | 业务 | SUM(price) / pay_success UV | 周 |
| 10 | 流量均匀度 | 质量 | max(A曝光,B曝光) / min(A曝光,B曝光) | 日 |

### 1.2 核心指标说明

**转化率（按变体）**：衡量各变体从曝光到支付的端到端转化效果。是判断实验优胜方的核心指标。

**流量均匀度**：监控Hash分流是否均匀分配。正常值应 < 1.2，超过说明分流异常，需排查Hash算法或数据上报是否有遗漏。

**套餐选择率**：反映落地页内容对用户的吸引力，是转化漏斗中曝光→选择的关键节点。

**支付成功率**：反映支付流程的顺畅度。如果A/B的支付成功率一致但转化率不同，说明差异来自页面内容；如果支付成功率也不同，需排查支付流程是否受实验干扰。

## 二、埋点方案

### 2.1 事件清单

| 事件ID | 事件名称 | 触发时机 | event_type | 优先级 |
|------|------|------|------|------|
| experiment_view | 页面曝光 | 落地页加载完成，首屏可见 | 0（普通事件） | P0 |
| plan_select | 套餐选择 | 用户点击套餐卡片 | 0（普通事件） | P0 |
| pay_click | 支付点击 | 用户点击支付按钮 | 0（普通事件） | P0 |
| pay_success | 支付成功 | 收到支付成功回调 | 1（带额外参数） | P0 |

### 2.2 公共参数（所有事件必带）

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| event_type | String | 是 | 0=普通事件，1=带额外参数事件 | "0" |
| event_id | String | 是 | 事件ID | "experiment_view" |
| event_time | Int64 | 是 | 事件发生时间戳(ms) | 1715846400000 |
| user_id | String | 是 | 用户唯一标识 | "usr_123456" |
| session_id | String | 是 | 会话ID | "sess_abc123" |
| os_type | String | 是 | iOS / Android | "iOS" |
| app_version | String | 是 | APP版本号 | "2.1.0" |

### 2.3 业务参数（按事件）

**experiment_view**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| experiment_key | String | 是 | 实验标识 |
| variant | String | 是 | A / B |

**plan_select**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| experiment_key | String | 是 | 实验标识 |
| variant | String | 是 | A / B |
| plan_type | String | 是 | 套餐类型（basic/pro） |
| plan_price | Number | 是 | 套餐价格 |

**pay_click**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| experiment_key | String | 是 | 实验标识 |
| variant | String | 是 | A / B |
| plan_type | String | 是 | 当前选中套餐 |
| plan_price | Number | 是 | 套餐价格 |

**pay_success**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| experiment_key | String | 是 | 实验标识 |
| variant | String | 是 | A / B |
| plan_type | String | 是 | 购买套餐类型 |
| plan_price | Number | 是 | 实际支付金额 |
| order_id | String | 是 | 订单号 |

### 2.4 上报方式

APP在调用分流接口获取 experiment_key 和 variant 后，在本地原生页面完成关键行为时直接上报事件数据至后台。

```
POST /api/abtest/event
```

### 2.5 去重策略

| 场景 | 策略 |
|------|------|
| 同 user_id + event_id 重复 | 1分钟内保留第一条 |
| pay_success 重复 | 同一 order_id 只保留第一条 |

### 2.6 数据校验

| 校验项 | 规则 | 异常处理 |
|------|------|------|
| 必填参数 | 公共7项 + 业务参数不得为空 | 丢弃并告警 |
| event_type | ∈ {0, 1, 2} | 标记异常 |
| variant | ∈ {A, B} | 标记异常 |
| plan_type | ∈ {basic, pro} | 标记异常 |
| plan_price | ∈ 定价范围 ±20% | 标记异常 |
| 时序校验 | pay_success.time ≥ pay_click.time | 标记异常 |
| 重复事件 | user_id + event_id 1min去重 | 保留第一条 |
| 重复订单 | pay_success order_id去重 | 保留第一条 |

## 三、数据看板

### 3.1 看板概览

**看板名称**：A/B Test 数据看板

**看板用途**：展示单个实验的完整数据，对比变体A/B在各指标上的差异，辅助运营判断胜出变体。

**目标查看人群**：产品经理、运营经理

**访问路径**：实验列表 → 点击行「数据」→ 数据看板页

### 3.4 报表需求

| 报表名称 | 类型 | 内容 | 刷新频率 | 接收人 |
|------|------|------|------|------|
| 运行中实验日报 | 日报 | 各运行中实验的核心数据概要 | 每日9:00 | 产品、运营 |
| 实验结案报告 | 一次性 | 实验全周期数据 + 结论 + 建议 | 实验结束时 | 产品、运营、管理层 |

### 3.5 告警规则

| 告警名称 | 触发条件 | 级别 | 通知方式 |
|------|------|------|------|
| 流量分配异常 | max(A曝光,B曝光)/min > 1.2 | P1 | 站内消息 |
| 数据上报中断 | 连续30分钟无新事件 | P1 | 站内 + 推送 |
| 支付成功率骤降 | 成功率环比下降 > 20% | P0 | 站内 + 推送 |
| 实验长期无结论 | 运行 > 30天仍不达标 | P2 | 站内消息 |

## 四、埋点测试验证

| 测试场景 | 预期结果 | 验证方式 |
|------|------|------|
| 打开落地页 | experiment_view 上报，公共属性齐全 | 抓包验证参数 |
| 点击套餐卡片 | plan_select 上报，plan_type 正确 | 抓包验证 |
| 点击支付按钮 | pay_click 上报 | 抓包验证 |
| 支付成功回调 | pay_success 上报，order_id非空，event_type=1 | 抓包验证 |
| 重复进入页面 | experiment_view 1min内去重 | 验证去重生效 |
| 无实验时 | 不调用埋点接口 | 无异常日志 |
| 公共属性完整性 | 7项公共属性均非空 | 逐项校验 |
| 枚举值校验 | variant仅A/B，plan_type仅basic/pro | 异常值告警 |
