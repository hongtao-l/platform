## 一、业务背景

### 业务场景

IoT 平台物模型能力库中，属性类型支持 enum、int、boolean、string 四种基础类型。但在实际设备协议设计中，存在以下场景需要数组结构：

1. **定时侦测**：设备可按多个时段执行侦测，每个时段包含开始时间、结束时间、循环周期等字段。需要 `array<struct>` 类型表达。
2. **区域侦测**：设备支持多个侦测区域，每个区域包含名称和顶点坐标。需要 `array<struct>` 类型表达，且顶点坐标本身又是 `array<int>`。
3. **循环周期**：一周内重复的星期几，如周一、周三、周五。需要 `array<int>` 类型表达。
4. **标签列表**：设备标签、能力标签等可变长字符串集合。需要 `array<string>` 类型表达。

当前能力库仅支持 enum/int/boolean/string，以上场景无法在能力定义中表达，导致协议设计需要绕过能力库直接硬编码。

### 用户痛点

- 能力定义缺少数组类型，struct 和 array 组合场景无法建模
- 定时侦测、区域侦测等复杂配置项无法在能力库中标准化
- 算法管理中编辑能力参数时，缺少 array 类型的编辑支持

### 产品目标

- 属性型能力新增 `array` 数据类型，支持 elementType: int / string / struct
- `struct` 类型和 `array<struct>` 支持字段编辑（参数弹窗），字段本身可嵌套 array
- 能力池 mock 数据中加入 array 类型示例（定时侦测、区域侦测）

## 二、名词解释

| 术语 | 说明 |
|------|------|
| 物模型 | 设备能力的抽象定义模型，包含属性（prop）、服务（svc）、事件（evt）三种能力类型 |
| 属性型能力（prop） | 描述设备的状态或配置参数，有具体的数据类型和读写模式 |
| 数据类型（dataType） | 属性的数据存储类型：enum / int / boolean / string / **array（新增）** / struct |
| elementType | 数组元素的类型，取值 int / string / struct。仅 dataType=array 时有效 |
| struct | 结构体类型，含若干子字段，每个子字段有独立名称、标识符和数据类型 |
| array\<struct\> | 元素类型为 struct 的数组，每个元素都是同构的结构体，支持字段编辑 |
| 能力池 | 能力库中「标准能力」Tab 下的能力列表，全品类共用的原子能力定义 |
| 参数弹窗 | 编辑服务出参、事件出参、struct 字段时弹出的编辑对话框，需同样支持 array 类型 |

## 三、功能范围

### 3.1 功能列表总览

| 序号 | 模块 | 功能 | 优先级 | 说明 | 备注 |
|------|------|------|:--:|------|------|
| F1 | 能力库 | 属性型容器支持 array 数据类型 | P0 | 在 enum/int/boolean/string 基础上新增 array，编辑器含元素类型选择 + 最大长度 | **核心需求** |
| F2 | 能力库 | array 支持三种 elementType | P0 | int / string / struct，下拉切换 | — |
| F3 | 能力库 | array\<struct\> 字段编辑 | P0 | 元素类型选 struct 时，展示字段列表编辑器（添加/编辑/删除字段） | 复用 struct 编辑器逻辑 |
| F4 | 能力库 | struct 和 array\<struct\> 字段支持嵌套 array | P0 | struct 字段的子字段 dataType 可再选 array，支持多层嵌套 | 参数弹窗同样支持 |
| F5 | 能力库 | 列表展示适配 array 类型 | P1 | 表格中能力行的「数据类型」列、「数据定义」列适配 array | — |
| F6 | 算法管理 | 参数弹窗支持 array 类型 | P1 | 添加/编辑出参时，数据类型下拉增加 array 选项 | elementType: int / string（参数弹窗不涉及 struct） |

### 3.2 不做/暂不做的功能

| 功能 | 原因 | 后续计划 |
|------|------|------|
| array 元素默认值逐项编辑 | array 默认值场景复杂（长度不定），暂用文本输入 | 按需评估可视化编辑器 |
| array 元素的校验规则（如每个元素范围） | 一期用描述文本补充 | 后续按需 |
| 参数弹窗中 array\<struct\> | 参数弹窗主要用于简单出参，struct 嵌套场景极少 | 后续评估 |

## 四、场景穷举分析

### 4.1 添加 array 类型能力

#### 4.1.1 功能概述

- **功能目的**：在能力库添加属性型能力时，数据类型可选择 array
- **前置条件**：用户已登录 IoT 管理后台，进入能力库 → 标准能力/自定义能力 Tab
- **操作路径**：物模型管理 → 能力库 → 添加能力 → 选择属性 → 数据类型选「数组型」

#### 4.1.2 正常场景

| 场景编号 | 场景名称 | 前置条件 | 操作步骤 | 预期结果 | 状态变化 |
|------|------|------|------|------|------|
| N-001 | 添加 array\<int\> 能力 | 能力库已打开 | 1. 点击「添加标准能力」 2. 选属性类型 3. 数据类型选「数组型」 4. 元素类型选 Int 5. 设置最大长度 10 6. 填写名称/标识符后保存 | 列表新增一条能力，数据类型列显示「数组型」，数据定义列显示「元素Int, 最大10项」 | 能力列表 +1 |
| N-002 | 添加 array\<string\> 能力 | 同上 | 同上，元素类���选 String，最大长度 20 | 数据定义列显示「元素String, 最大20项」 | 能力列表 +1 |
| N-003 | 添加 array\<struct\> 能力 | 同上 | 1. 元素类型选 Struct 2. 添加字段「开始时间」(string) 和「结束时间」(string) 3. 保存 | 列表新增能力，数据定义列显示「元素Struct, 最大100项」 | 能力列表 +1 |
| N-004 | struct 字段内嵌套 array | 能力弹窗已打开，dataType=struct | 1. 添加字段「循环周期」 2. 数据类型选 array 3. 元素类型选 Int 4. 保存 struct 字段 | 字段列表显示「循环周期」带「元素Int, 最大X项」摘要 | struct 字段列表 +1 |

#### 4.1.3 异常场景

| 场景编号 | 异常类型 | 触发条件 | 系统行为 | 用户感知 |
|------|------|------|------|------|
| E-001 | 必填缺失 | 数组型未选元素类型 | 保存校验失败 | Toast「请选择元素类型」 |
| E-002 | 必填缺失 | 数组型未填最大长度 | 保存校验失败 | Toast「请填写最大长度」 |
| E-003 | 结构体无字段 | array\<struct\> 字段列表为空 | 保存校验失败 | Toast「请至少添加一个结构体字段」 |
| E-004 | 越界 | 最大长度输入 0 或 >1000 | 控件自动限制 | InputNumber 自动修正 |

#### 4.1.4 边界条件

| 条件项 | 最小值 | 最大值 | 默认值 | 超限行为 |
|------|------|------|------|------|
| array maxLength | 1 | 1000 | 100 | 自动限制 |
| struct 字段数 | 1 | 50 | — | 前端不硬限制 |

### 4.2 编辑 array 类型能力

#### 4.2.1 功能概述

- **功能目的**：编辑已有 array 类型能力的数据定义
- **前置条件**：已有 array 类型的能力记录
- **操作路径**：能力库 → 某能力行的「编辑」按钮 → 弹窗回填

#### 4.2.2 正常场景

| 场景编号 | 场景名称 | 前置条件 | 操作步骤 | 预期结果 |
|------|------|------|------|------|
| N-101 | 回填 array\<int\> | 已有 array\<int\> 能力 | 点击编辑 | 弹窗回填：数据类型=数组型，元素类型=Int，最大长度=已有值 |
| N-102 | 回填 array\<struct\> | 已有 array\<struct\> 能力 | 点击编辑 | 弹窗回填：元素类型=Struct，字段列表完整还原 |
| N-103 | 修改 elementType | 已有 array\<int\> 能力 | 将 Int 改为 String | 保存后数据定义更新 |

### 4.3 参数弹窗中使用 array

#### 4.3.1 功能概述

- **功能目的**：在编辑服务出参/事件出参时，参数的数据类型可选 array
- **前置条件**：在能力弹窗中编辑 svc/evt 的出参，或编辑 struct 字段

#### 4.3.2 正常场景

| 场景编号 | 场景名称 | 前置条件 | 操作步骤 | 预期结果 |
|------|------|------|------|------|
| N-201 | 出参使用 array\<int\> | 参数弹窗已打开 | 数据类型选 Array → 元素类型选 Int → 填最大长度 | 参数显示类型「Array」摘要「元素Int, 最大N项」 |
| N-202 | 出参使用 array\<string\> | 同上 | 数据类型选 Array → 元素类型选 String | 同预期 |

## 五、跨模块联动分析

### 5.1 与现有功能的关联

| 关联模块 | 关联方式 | 影响 |
|------|------|------|
| 能力库 — 能力池 | array 作为新的 dataType 选项 | 添加/编辑能力弹窗增加 UI |
| 能力库 — 列表展示 | 数据类型列和数据定义列需要适配 | `dataTypeLabel` 和 `dataDefDetail` 函数已支持 |
| 算法管理 — 参数弹窗 | 关联能力出参时可�� array | 参数弹窗增加 array 类型的编辑 UI |
| 能力库 — 自定义能力 | 自定义能力同样支持 array | 复用标准能力的 array 编辑逻辑 |
| data.js mock 数据 | 已有定时侦测、区域侦测的 array mock | 示例数据充当测试用例 |

### 5.2 数据结构影响

```
属性 dataDef 扩展:
  dataType = 'array' 时新增:
    elementType: 'int' | 'string' | 'struct'   ← 元素类型
    maxLength: number (1-1000)                   ← 最大长度
    fields?: Field[]                             ← elementType='struct' 时的字段定义
```
