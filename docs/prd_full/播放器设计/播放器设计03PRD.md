## 一、业务背景

CareCam Pro 应用的核心功能之一是实时视频预览。用户通过 APP 查看家中或场所的摄像机实时画面，并进行云台控制、语音对讲、录制等操作。

当前存在的主要问题：

- **手动切换设备类型**：用户需要在直播页内手动选择摄像机品类（标准/广角/鱼眼），增加操作步骤且不直观
- **多目设备支持不足**：市面上越来越多的摄像机具备多个镜头（最多4个），现有页面只支持单画面显示
- **代码重复**：不同摄像机类型页面大量逻辑重复，缺乏统一的布局框架和可复用组件
- **扩展困难**：新增摄像机类型需要新增路由和页面，不符合开闭原则

**产品目标**：
- 通过设备命名自动识别摄像机类型，消除用户手动切换
- 统一的直播页布局框架，支持单屏和多屏（最多4屏）两种模式
- 组件化设计，功能按钮按设备类型可配置
- 为未来更多设备类型提供可扩展基础

## 二、名词解释

| 术语 | 说明 |
|------|------|
| 标准摄像机 | 单镜头 PTZ 云台摄像机，支持方向控制和变焦 |
| 多目单流广角摄像机 | 多镜头设备，输出单路超广角视频流（32:9比例），最多4个镜头 |
| 多目设备 | 具有多个镜头的摄像机统称 |
| 子设备 | 绑定到主摄像机上的附属传感器（温湿度、烟雾、门磁、雷达等） |
| PTZ | Pan/Tilt/Zoom — 云台水平旋转/垂直旋转/变焦控制 |
| 清晰度 | 视频流质量档位：标清(SD)、高清(HD)、超清(UHD) |
| 设备命名规则 | 后台配置的设备名称，APP通过关键词匹配判断设备类型 |
| 全屏模式 | 视频画面占据整个屏幕，隐藏系统状态栏和导航栏 |

## 三、业务实体说明

### 3.1 核心实体

**摄像机设备（Camera Device）**
- 属性：设备ID、设备名称、在线状态、镜头数量、子设备列表
- 设备类型通过设备名称关键词推断，不作为独立字段
- 一个设备可绑定多个子设备

**设备类型（Device Type）**
- 标准摄像机（Standard）：默认类型
- 多目单流广角摄像机（WideAngle）：名称含「广角/全景/多目」
- 识别规则：不区分大小写，子串匹配，优先级匹配

**视频流（Video Stream）**
- 标准摄像机：1路流
- 多目广角摄像机：1-N路流（N=镜头数，最多4）
- 每路流支持多个清晰度档位

**子设备（Sub Device）**
- 类型：温湿度传感器、烟雾传感器、门磁、毫米波雷达
- 属性：类型、在线状态、当前读数/状态值

### 3.2 实体关系

```
摄像机设备 1──N 视频流
摄像机设备 1──N 子设备
设备类型 1──1 直播页布局
功能按钮 n──m 设备类型（按设备类型配置可见性）
```

## 四、核心业务流程

### 4.1 设备类型识别与路由

```
用户在设备列表点击「实时预览」
  → APP 获取设备信息（设备名称、镜头数量等）
  → 关键词匹配判断设备类型
    → 含「广角」or「全景」or「多目」→ 广角直播页
    → 其他 → 标准直播页
  → 携带设备参数路由到对应页面
  → 页面按类型渲染布局
```

### 4.2 标准摄像机直播流程

```
进入标准直播页
  → 显示单路视频画面
  → 画面底部显示清晰度选择和全屏按钮
  → 底部显示功能按钮组
  → 用户点击PTZ → 展开PTZ面板（方向键+变焦滑块）
  → 用户点击子设备 → 弹出子设备列表面板
  → 用户点击回放 → 跳转回放页
  → 用户点击对讲 → 按住说话，松手发送
```

### 4.3 多目广角摄像机直播流程

```
进入广角直播页
  → 根据 lensCount 渲染对应数量画面
  → 单画面模式：32:9宽画布，支持缩放/平移
  → 多画面模式：等分纵向空间，各画面独立
  → 点击某画面 → 活跃高亮
  → 点击全屏 → 该画面全屏
  → 底部功能按钮组（PTZ替换为广角专用控制）
```

### 4.4 多屏交互流程

```
多屏模式进入
  → 默认第一屏为活跃状态（蓝色边框）
  → 点击其他屏 → 切换活跃状态
  → 清晰度切换仅影响活跃屏
  → 点击全屏 → 活跃屏全屏，顶部显示镜头编号
  → 全屏下可缩放/平移（32:9画面）
  → 返回 → 回到多屏并列
```

## 五、业务规则

### 5.1 设备识别规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 关键词匹配 | 设备名含「广角」「全景」「多目」任一 → 广角类型 |
| R02 | 不区分大小写 | 匹配时忽略大小写，如「WIDE ANGLE」→ 广角 |
| R03 | 默认兜底 | 不含上述关键词 → 标准摄像机类型 |
| R04 | 优先级 | 匹配到即停止，顺序：广角 > 全景 > 多目 |

### 5.2 多屏规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R05 | 屏幕数量 | 有效范围 1-4，超过按4处理，0按1处理 |
| R06 | 排列方向 | 从上到下垂直排列，等分纵向空间 |
| R07 | 活跃屏 | 每次只有一屏为活跃状态，全屏/清晰度切换仅影响活跃屏 |
| R08 | 分隔 | 屏与屏之间 2px 分隔线 |

### 5.3 功能按钮规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R09 | 按钮可配置 | 功能按钮按设备类型显示/隐藏 |
| R10 | PTZ按钮 | 仅标准摄像机显示 |
| R11 | 变倍按钮 | 广角摄像机显示，标准不显示 |
| R12 | 补光/警笛 | 广角摄像机显示 |

## 七、详细功能描述

### 7.1 设备类型自动识别

#### 功能应用场景
用户从设备列表进入实时预览，APP 自动判断设备类型并加载对应页面，无需用户手动选择。

#### 识别逻辑
1. 获取设备名称字符串
2. 转为小写后检查是否包含关键词
3. 匹配成功 → 确定类型 → 路由
4. 匹配失败 → 标准类型兜底

### 7.2 标准摄像机直播页

#### 功能应用场景
查看单镜头 PTZ 摄像机的实时画面，控制云台方向、变焦，并进行对讲、录制等操作。

#### 页面组成
| 区域 | 内容 |
|------|------|
| 顶部栏 | 返回按钮 + 设备名称 + 信号强度 + 设置入口 |
| 视频画面 | 16:9单路实时视频 |
| 画面叠加 | 清晰度选择（SD/HD/UHD）+ 全屏按钮 |
| 底部功能区 | 功能按钮网格 + PTZ控制面板（条件展开） |

#### PTZ 控制
- 3×3 方向键：上/下/左/右 + 中心复位
- 变焦滑块：1x - 10x，实时显示当前倍数
- 点击 PTZ 按钮展开面板，再次点击收起

### 7.3 多目广角摄像机直播页

#### 功能应用场景
查看多目广角摄像机（最多4个镜头）的实时画面，支持单画面全景缩放和多画面并列两种模式。

#### 画面模式

| 模式 | 条件 | 行为 |
|------|------|------|
| 单屏全景 | lensCount = 1 | 32:9 宽画布，支持缩放/平移，全景小地图导航 |
| 多屏并列 | lensCount = 2-4 | 等分纵向空间，各画面独立，点击切换活跃屏 |

#### 全景交互（单屏模式）
- 双指捏合/滚轮缩放（1x - 8x）
- 单指拖动平移
- 双击切换 1x ↔ 2x
- 放大后显示全景小地图，标示当前视口位置
- 小地图可拖拽，点击跳转视口

#### 底部功能按钮
- 语音对讲、变倍、回放、更多
- 录制、子设备、截图、补光
- 「更多」展开：警笛、水平翻转、垂直翻转、预置位

### 7.4 子设备面板

#### 功能应用场景
查看绑定到摄像机的子设备（传感器）的实时状态。

#### 面板结构
- 底部弹出面板（van-popup, 圆角顶部）
- 头部：标题「子设备」+ 关闭按钮
- 内容：按分组排列的子设备列表
- 每行：图标 + 名称 + 在线状态 + 当前读数
- 空态：「暂无子设备」居中提示
- 点击行：查看详情/历史数据

### 7.5 语音对讲

#### 功能应用场景
用户通过 APP 与摄像机端进行实时语音通话。

#### 交互方式
- 按住「对讲」按钮开始说话
- 松手结束并发送语音
- 对讲过程中按钮高亮（激活态背景色）

### 7.6 录制

#### 功能应用场景
用户将当前实时画面保存为本地视频。

#### 交互方式
- 点击「录制」开始录制，按钮变为激活态（红色指示灯）
- 再次点击停止录制
- 录制文件保存到本地相册/文件目录

| 起点 | 触发 | 终点 |
|------|------|------|
| 设备列表 | 点击设备「实时预览」 | 直播页（按类型自动路由） |

## 九、异常说明

| 分类 | 场景 | 处理方式 |
|------|------|------|
| 设备识别 | 设备名为空 | 按标准摄像机处理，Toast 提示 |
| 视频流 | 视频加载失败 | 错误占位图 + 重试按钮 |
| 视频流 | 多目部分流失败 | 仅对应区域显示错误，其他正常 |
| 视频流 | 播放卡顿/断流 | 自动重连3次，失败后手动刷新入口 |
| 网络异常 | 设备信息接口超时 | 重试3次，失败使用缓存数据 |
| 网络异常 | 控制指令超时 | Toast「设备响应超时」 |
| 设备状态 | 设备离线 | 离线占位图 + 文字提示 |
| 多屏边界 | lensCount=0 或未传 | 按1处理 |
| 多屏边界 | lensCount>4 | 按4处理 + Toast 提示 |
| 子设备 | 子设备列表为空 | 空态「暂无子设备」 |
| 子设备 | 子设备离线 | 显示离线标记 + 最后更新时间 |
| 子设备 | 传感器读数为 null | 显示「--」 |
| 操作异常 | 快速重复点击PTZ | 指令去重，300ms内忽略重复 |
| 操作异常 | 录制中切换页面 | 提示「录制将停止」或自动停止 |
