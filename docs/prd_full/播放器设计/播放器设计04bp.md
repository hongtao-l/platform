## 一、核心指标体系

### 1.1 指标总览

| 序号 | 指标名称 | 指标类型 | 计算方式 | 监控周期 |
|------|------|------|------|------|
| 1 | 直播页PV | 行为 | COUNT(page_view) WHERE page=liveview | 日 |
| 2 | 直播页UV | 行为 | COUNT(DISTINCT user_id) WHERE page=liveview | 日 |
| 3 | 平均观看时长 | 行为 | AVG(view_duration) WHERE page=liveview | 日 |
| 4 | 视频加载成功率 | 质量 | COUNT(stream_start_success) / COUNT(stream_start) × 100 | 实时 |
| 5 | 视频首帧耗时 | 质量 | AVG(first_frame_time) WHERE event=stream_start_success | 日 |
| 6 | 卡顿率 | 质量 | COUNT(stream_stutter) / COUNT(view_session) × 100 | 日 |
| 7 | PTZ使用率 | 行为 | COUNT(ptz_action) / COUNT(view_session) × 100 | 周 |
| 8 | 语音对讲使用率 | 行为 | COUNT(intercom_start) / COUNT(view_session) × 100 | 周 |
| 9 | 录制使用率 | 行为 | COUNT(record_start) / COUNT(view_session) × 100 | 周 |
| 10 | 截图使用率 | 行为 | COUNT(snapshot) / COUNT(view_session) × 100 | 周 |
| 11 | 子设备查看率 | 行为 | COUNT(subdevice_panel_open) / COUNT(view_session) × 100 | 周 |
| 12 | 清晰度切换率 | 行为 | COUNT(quality_switch) / COUNT(view_session) × 100 | 周 |
| 13 | 全屏使用率 | 行为 | COUNT(fullscreen_enter) / COUNT(view_session) × 100 | 周 |
| 14 | 各设备类型占比 | 业务 | COUNT(view_session) GROUP BY device_type | 周 |
| 15 | 多屏模式下各屏点击率 | 行为 | COUNT(screen_tap) GROUP BY screen_index | 周 |

### 1.2 核心指标说明

**视频加载成功率**：衡量视频流播放的稳定性。成功率低于 98% 需排查 CDN/设备端推流问题。

**平均观看时长**：反映用户对直播功能的使用深度。分设备类型对比可判断不同类型设备的用户粘性差异。

**PTZ使用率**：衡量云台控制功能的渗透率，影响后续 PTZ 相关功能的优先级决策。

**卡顿率**：每100次观看会话中发生卡顿的次数，是核心体验质量指标。

## 二、埋点方案

### 2.1 事件清单

| 事件ID | 事件名称 | 触发时机 | 优先级 |
|------|------|------|------|
| liveview_enter | 进入直播页 | 页面 onMounted | P0 |
| liveview_leave | 离开直播页 | 页面 onUnmounted | P0 |
| stream_start | 开始拉流 | 调用拉流接口时 | P0 |
| stream_start_success | 拉流成功 | 收到首帧画面时 | P0 |
| stream_start_fail | 拉流失败 | 拉流接口失败/超时 | P0 |
| stream_stutter | 播放卡顿 | 缓冲区欠载触发 | P0 |
| quality_switch | 清晰度切换 | 用户点击清晰度选项 | P1 |
| fullscreen_enter | 进入全屏 | 点击全屏按钮 | P1 |
| fullscreen_exit | 退出全屏 | 退出全屏操作 | P1 |
| ptz_action | PTZ操作 | 点击方向键/滑块变焦 | P1 |
| intercom_start | 开始对讲 | 按住对讲按钮 | P1 |
| intercom_end | 结束对讲 | 松手对讲按钮 | P1 |
| record_start | 开始录制 | 点击录制按钮 | P1 |
| record_stop | 停止录制 | 再次点击录制/自动停止 | P1 |
| snapshot | 截图 | 点击截图按钮 | P1 |
| subdevice_panel_open | 打开子设备面板 | 点击子设备按钮 | P2 |
| subdevice_detail | 查看子设备详情 | 点击子设备行 | P2 |
| screen_tap | 多屏点击切换 | 多屏模式下点击某画面 | P2 |
| panorama_zoom | 全景缩放 | 广角画面缩放操作 | P2 |
| panorama_pan | 全景平移 | 广角画面平移操作 | P2 |
| light_toggle | 补光开关 | 点击补光按钮 | P2 |
| siren_toggle | 警笛开关 | 点击警笛按钮 | P2 |
| device_type_mismatch | 设备类型误判 | 用户反馈/自动检测类型不匹配 | P2 |

### 2.2 公共参数（所有事件必带）

| 参数 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|------|
| event_id | String | 是 | 事件ID | "liveview_enter" |
| event_time | Int64 | 是 | 事件发生时间戳(ms) | 1715846400000 |
| user_id | String | 是 | 用户唯一标识 | "usr_123456" |
| session_id | String | 是 | 观看会话ID（进入直播页时生成） | "sess_abc123" |
| device_id | String | 是 | 设备ID | "dev_001" |
| device_name | String | 是 | 设备名称 | "Front Door" |
| device_type | String | 是 | 识别出的设备类型 | "standard" / "wide_angle" |
| app_version | String | 是 | APP版本号 | "2.1.0" |
| os_type | String | 是 | iOS / Android | "iOS" |

### 2.3 业务参数（按事件）

**liveview_enter / liveview_leave**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lens_count | Number | 是 | 镜头数量 |
| enter_from | String | 是 | 来源页面（device_list / notification / widget） |

**liveview_leave**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| view_duration | Number | 是 | 观看时长(秒) |

**stream_start / stream_start_success / stream_start_fail**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stream_quality | String | 是 | 清晰度档位 (SD/HD/UHD) |
| first_frame_time | Number | 否 | 首帧耗时(ms)，仅success时 |
| fail_reason | String | 否 | 失败原因，仅fail时 |

**ptz_action**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action_type | String | 是 | 操作类型 (direction / zoom) |
| direction | String | 否 | 方向 (up/down/left/right/center)，仅direction时 |
| zoom_level | Number | 否 | 变焦级别，仅zoom时 |

**quality_switch**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| from_quality | String | 是 | 切换前清晰度 |
| to_quality | String | 是 | 切换后清晰度 |

**screen_tap**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| screen_index | Number | 是 | 点击的屏幕序号(0-3) |
| total_screens | Number | 是 | 当前总屏数 |

### 2.4 去重策略

| 场景 | 策略 |
|------|------|
| liveview_enter 重复上报 | 同一 session_id 仅保留第一条 |
| stream_start 重复 | 同一 session_id + device_id 5秒内去重 |
| ptz_action 高频 | 同一方向指令 300ms 内去重 |

## 三、数据看板

### 3.1 看板概览

**看板名称**：播放器运营看板

**看板用途**：监控直播功能的使用情况和视频播放质量，发现异常及时排查。

**目标查看人群**：产品经理、运营经理、技术运维

### 3.2 看板布局

```
┌─────────────────────────────────────────────┐
│ 区域1: 概览指标卡片（4列）                      │
│ [日活UV] [观看次数] [平均时长] [加载成功率]       │
├─────────────────────────────────────────────┤
│ 区域2: 趋势图（2列）                           │
│ [每日PV/UV趋势]        [每日加载成功率]         │
├─────────────────────────────────────────────┤
│ 区域3: 设备类型分布                            │
│ [饼图: 标准 vs 广角 占比]                      │
├─────────────────────────────────────────────┤
│ 区域4: 功能使用排行榜                          │
│ [柱状图: 各功能按钮点击率排名]                   │
├─────────────────────────────────────────────┤
│ 区域5: 视频质量监控                            │
│ [首帧耗时P50/P90/P99]  [卡顿率趋势]            │
├─────────────────────────────────────────────┤
│ 区域6: 设备类型对比                            │
│ [表格: 标准 vs 广角 各项指标对比]               │
└─────────────────────────────────────────────┘
```

### 3.3 各区域说明

**区域1 - 概览指标卡片**：
- 今日直播UV、今日观看次数(PV)、平均观看时长、视频加载成功率
- 每个卡片同比昨日变化（绿色↑ / 红色↓）

**区域2 - 趋势图**：
- 近30天 PV/UV 折线图（双Y轴）
- 近30天加载成功率折线图（含 98% 告警线）

**区域3 - 设备类型分布**：
- 饼图展示标准 vs 广角占比
- 按日聚合

**区域4 - 功能使用排行**：
- 柱状图：PTZ/对讲/录制/截图/子设备/全屏等功能的点击率
- 支持按设备类型筛选

**区域5 - 视频质量**：
- 首帧耗时 P50/P90/P99 三线图
- 卡顿率趋势

**区域6 - 设备类型对比**：
- 标准 vs 广角的各项指标对比表（UV、时长、加载成功率、功能使用率）

### 3.4 告警规则

| 告警名称 | 触发条件 | 级别 | 通知方式 |
|------|------|------|------|
| 视频加载成功率骤降 | 成功率 < 95% | P0 | 站内 + 推送 |
| 首帧耗时异常 | P90 > 3s | P1 | 站内消息 |
| 卡顿率飙升 | 卡顿率 > 5% | P1 | 站内 + 推送 |
| 直播页UV骤降 | 环比下降 > 30% | P1 | 站内消息 |
| 设备类型识别异常 | device_type_mismatch 事件 > 10次/天 | P2 | 站内消息 |

## 四、埋点测试验证

| 测试场景 | 预期结果 | 验证方式 |
|------|------|------|
| 从设备列表进入标准摄像机 | liveview_enter 上报，device_type=standard | 抓包验证 |

| PTZ 方向点击 | ptz_action 上报，direction 正确 | 抓包验证 |
| PTZ 变焦滑动 | ptz_action 上报，zoom_level 正确 | 抓包验证 |

| 打开子设备面板 | subdevice_panel_open 上报 | 抓包验证 |
