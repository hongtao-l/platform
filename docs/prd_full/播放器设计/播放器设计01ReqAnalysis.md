## 一、业务背景

CareCam Pro 需要支持不同类型的 IPC 摄像机设备。不同设备的**物模型属性**不同——有的单镜头，有的多镜头（最多4目）；有的支持 PTZ 云台，有的带补光灯/警笛。目前的做法是在直播页内手动切换摄像机品类，存在以下问题：

- **操作冗余**：用户需手动选择设备品类才能看到对应的功能按钮
- **扩展性差**：新增设备类型需要新增品类选项和页面分支
- **多目设备支持不足**：当前只支持单画面，无法适配多目设备多个镜头同时展示
- **功能按钮硬编码**：PTZ、补光、警笛等功能写死在页面中，而非根据设备实际能力动态显示

**核心目标**：基于设备物模型属性动态渲染直播页——画面数量由 `lensCount` 决定（1-4路），功能按钮由设备能力属性（hasPTZ/hasLight/hasSiren 等）决定。统一页面框架自适应不同设备能力组合。

---

## 二、角色与职责

| 角色 | 职责 | 关键约束 |
|------|------|----------|
| APP（客户端） | 根据物模型属性渲染对应数量的视频画面 + 功能按钮 | 不预判设备类型，纯属性驱动 |
| 后台（iot-platform） | 提供设备物模型（属性、能力、子设备列表）+ 视频流地址 | 物模型定义设备全部能力 |
| 用户 | 进入直播页查看实时画面，操作可用功能 | 无需手动选择设备类型 |

---

## 三、名词解释

| 术语 | 说明 |
|------|------|
| 物模型 | 设备的数字化定义，包含属性（Property）、动作指令（Action）、告警事件（Event）。APP 根据物模型属性决定页面布局和可用功能 |
| 镜头数 lensCount | 物模型属性，决定渲染几路视频画面（1-4） |
| PTZ | Pan/Tilt/Zoom 云台控制，包含方向旋转 + 变焦 |
| 子设备 | 绑定到主摄像机的附属传感器（温湿度、烟雾、门磁、毫米波雷达等），物模型 `subDevices` 列表 |
| 多目设备 | lensCount ≥ 2 的设备，多路画面从上到下垂直排列 |
| 清晰度切换 | 用户在画面底部切换视频清晰度（标清/高清/超清） |

## 四、业务实体说明

### 核心实体

**摄像机设备（Camera Device）**
- 物模型属性决定页面布局和功能：
  - `lensCount`：镜头数量（1-4），决定渲染几路视频画面
  - `hasPTZ`：是否支持云台控制
  - `hasLight`：是否支持补光灯
  - `hasSiren`：是否支持警笛
  - `hasIntercom`：是否支持语音对讲
  - `presetPositions`：预置位列表
  - `subDevices`：子设备列表
- 设备名称仅作展示用，不参与任何布局/路由逻辑

**视频画面（Video Panel）**
- 竖屏全宽，16:9 比例
- 多目设备：多个 16:9 画面垂直叠放，画面间 2px 分隔
- 每路画面独立的清晰度和全屏控制

**子设备（Sub Device）**
- 类型由物模型定义：温湿度传感器、烟雾传感器、门磁、毫米波雷达等
- 属性：子设备ID、类型、在线状态、当前读数/状态

### 实体关系

```
摄像机设备 1──1 物模型（定义全部能力和属性）
物模型 1──N 视频画面（lensCount 决定数量）
物模型 0──N 子设备
物模型 0──N 功能能力（PTZ/补光/警笛/对讲/预置位）
```

## 五、核心业务流程

### 5.1 页面初始化

```
用户在设备列表点击某设备「实时预览」
  → APP 请求设备物模型 + 视频流地址
  → 解析物模型属性：
    - lensCount → 渲染 N 路 16:9 视频画面，从上到下排列
    - 功能属性 → 底部工具栏动态显示对应按钮
  → 初始化各路视频流
  → 进入竖屏全屏直播页
```

### 5.2 单目标准流程（lensCount=1）

```
进入直播页
  → 1 路 16:9 视频画面（全宽，居中偏上）
  → 画面底部：清晰度切换 + 全屏按钮
  → 底部工具栏：根据物模型属性动态显示功能按钮
  → 例：hasPTZ=true → 显示 PTZ 按钮，点击展开 3x3 方向键 + 变焦滑块
  → 例：hasIntercom=true → 显示对讲按钮
  → 例：hasLight=true → 显示补光按钮
```

### 5.3 多目流程（lensCount=2-4）

```
进入直播页
  → N 路 16:9 视频画面，从上到下垂直排列，等分屏幕空间
  → 每路画面底部：清晰度切换 + 全屏按钮
  → 点击某画面 → 该画面全屏（横屏模式），其他画面隐藏
  → 全屏下支持手势返回 → 回到多屏并列
  → 底部工具栏同单目，由物模型属性决定
```

## 六、功能架构

```
播放器系统（APP端）
├── 1. 物模型解析层
│   └── 从设备详情中提取 lensCount + 功能能力列表 + 子设备列表
├── 2. 直播页面（统一布局框架）
│   ├── 顶部栏（设备名称、返回、设置入口）
│   ├── 视频画面区（1-N 路 16:9，垂直排列）
│   │   ├── 清晰度切换
│   │   └── 全屏切换
│   └── 底部功能区（物模型驱动的动态按钮组）
├── 3. 功能面板（按能力条件渲染）
│   ├── PTZ 面板（hasPTZ=true 时可用）
│   ├── 对讲功能（hasIntercom=true 时可用）
│   ├── 补光灯（hasLight=true 时可用）
│   ├── 警笛（hasSiren=true 时可用）
│   └── 预置位（presetPositions 非空时可用）
└── 4. 子设备面板
    ├── 子设备列表弹窗
    └── 各类型子设备状态展示
```

## 七、详细功能分析

### 7.1 物模型解析

**功能应用场景**：进入直播页前，APP 从设备详情中提取物模型属性，决定页面渲染方式。

**正常场景**：
- SC-01：获取设备详情 → 提取 `lensCount`（默认1）→ 决定画面数量
- SC-02：提取功能属性 `hasPTZ/hasLight/hasSiren/hasIntercom` → 决定底部按钮可见性
- SC-03：提取 `subDevices` → 子设备面板内容
- SC-04：提取 `presetPositions` → 预置位列表

**异常场景**：
- EX-01：物模型字段缺失 → 使用默认值（lensCount=1, 功能全false），Toast 提示部分功能不可用
- EX-02：lensCount=0 → 按 1 处理
- EX-03：lensCount>4 → 按 4 处理 + Toast 提示

### 7.2 单路视频（lensCount=1）

**功能应用场景**：查看单镜头摄像机的实时画面。

**布局**：
- 竖屏全屏，视频区域 16:9 比例，居中偏上
- 视频画面下方：清晰度选择 + 全屏按钮
- 画面下方黑色区域 → 功能按钮工具栏

**正常场景**：
- SC-01：进入页面自动拉流播放，默认高清
- SC-02：点击清晰度切换（标清/高清/超清）
- SC-03：点击全屏按钮 → 横屏全屏播放
- SC-04：视频区域上方覆盖顶部栏（半透明渐变）

**异常场景**：
- EX-01：视频流加载失败 → 画面显示重试按钮 + 提示
- EX-02：设备离线 → 离线占位图

### 7.3 多路视频（lensCount=2-4）

**功能应用场景**：查看多目摄像机的多路实时画面。

**布局**：
- 竖屏全屏，N 路 16:9 画面从上到下垂直排列
- 等比例均分可用空间
- 画面间 2px 深色分隔线

**正常场景**：
- SC-01：lensCount=2 → 两个 16:9 画面上下排列
- SC-02：lensCount=3 → 三个 16:9 画面上下排列
- SC-03：lensCount=4 → 四个 16:9 画面上下排列
- SC-04：每路画面底部独立清晰度切换 + 全屏按钮
- SC-05：点击某画面全屏 → 该画面进入横屏全屏，顶部显示「镜头 N」
- SC-06：全屏下返回 → 回到多屏并列

**异常场景**：
- EX-01：部分流加载失败 → 仅对应区域错误占位，其他正常
- EX-02：lensCount 超出 1-4 范围 → clamp 到 1-4

### 7.4 底部功能区（物模型驱动）

**功能应用场景**：提供设备支持的所有功能入口。

**功能按钮按物模型属性动态显示**：

| 按钮 | 显示条件 | 行为 |
|------|------|------|
| 语音对讲 | `hasIntercom=true` | 按住说话，松手发送 |
| PTZ 控制 | `hasPTZ=true` | 展开 PTZ 3x3 方向键面板 + 变焦滑块 |
| 回放 | 始终显示 | 跳转回放页 |
| 录制 | 始终显示 | 开始/停止本地录制 |
| 截图 | 始终显示 | 截取当前活跃画面 |
| 子设备 | `subDevices` 非空 | 弹出子设备列表面板 |
| 补光 | `hasLight=true` | 切换补光灯开关 |
| 警笛 | `hasSiren=true` | 触发警笛 |
| 预置位 | `presetPositions` 非空 | 选择预置位，设备自动旋转 |
| 更多 | 始终显示 | 展开更多功能行 |

**按钮排列规则**：
- 第一行 4 个：核心功能（对讲、PTZ、回放、更多 → 按可用性从左到右）
- 第二行 4 个：辅助功能（录制、子设备、截图、补光等）

**「更多」展开行**：容纳第一行放不下的功能（警笛、预置位等）

**异常场景**：
- EX-01：功能指令超时 → Toast「设备响应超时」
- EX-02：设备不支持某功能（物模型字段为 false）→ 按钮不渲染

### 7.5 PTZ 控制面板

**显示条件**：`hasPTZ=true`，点击 PTZ 按钮展开

**面板内容**：
- 3x3 方向键网格（上/下/左/右 + 中心复位，120×120px）
- 变焦滑块（1x-10x，实时数值显示）
- 点击其他区域或再次点击 PTZ → 面板收起

### 7.6 子设备面板

**显示条件**：`subDevices` 列表非空

**正常场景**：
- SC-01：底部弹出面板（van-popup，圆角顶部，最大高度 60vh）
- SC-02：按类型分组展示（环境传感器 / 安防传感器 / 其他）
- SC-03：每行显示图标 + 名称 + 在线状态 + 实时读数
- SC-04：无子设备 → 不显示「子设备」按钮

**子设备类型展示**：

| 类型 | 显示数据 |
|------|------|
| 温湿度传感器 | 温度(°C) + 湿度(%) |
| 烟雾传感器 | 正常/报警 |
| 门磁 | 开/关 |
| 毫米波雷达 | 有人/无人 |

**异常场景**：
- EX-01：子设备离线 → 灰色 + 最后更新时间
- EX-02：传感器数据为空 → 显示「--」

## 八、业务规则

### 8.1 物模型驱动规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 画面数量由物模型决定 | `lensCount` = 几就渲染几个 16:9 画面 |
| R02 | 功能按钮由物模型决定 | 每个按钮都有对应的能力属性，false 则不渲染 |
| R03 | 默认值兜底 | lensCount 默认 1，功能属性默认 false |
| R04 | 设备名仅展示 | 设备名不参与任何布局/路由/功能判断 |

### 8.2 多屏规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R05 | 排列方向 | 从上到下垂直排列 |
| R06 | 空间分配 | 等比例均分可用空间 |
| R07 | 画面比例 | 每路均为 16:9，全宽 |
| R08 | 全屏行为 | 某路全屏时隐藏其他路，返回即恢复 |

### 8.3 视频播放规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R09 | 自动播放 | 进入页面自动拉流 |
| R10 | 清晰度默认 | 默认高清（HD），切换后记住偏好 |

## 九、异常场景总览

| 分类 | 场景 | 处理方式 |
|------|------|------|
| 物模型 | lensCount 字段缺失 | 默认 1 |
| 物模型 | 功能属性缺失 | 默认 false（按钮不显示） |
| 物模型 | subDevices 为空 | 不显示「子设备」按钮 |
| 视频流 | 视频流加载失败 | 错误占位 + 重试按钮 |
| 视频流 | 多目部分流失败 | 仅对应区域显示错误 |
| 网络异常 | 控制指令超时 | Toast 提示 |
| 设备状态 | 设备离线 | 离线占位图 |
| 多屏边界 | lensCount=0 | 按 1 处理 |
| 多屏边界 | lensCount>4 | 按 4 处理 + Toast 提示 |

## 十、跨角色联动分析

### APP ↔ 后台协作

| 环节 | APP职责 | 后台职责 |
|------|------|------|
| 能力获取 | 从物模型解析 lensCount + 功能能力 + 子设备 | 提供完整物模型 JSON |
| 视频播放 | 根据 lensCount 拉对应数量视频流 | 提供各路视频流地址 |
| PTZ/指令 | 发送控制指令 | 转发指令到设备 |
| 子设备 | 展示子设备状态 | 提供子设备实时数据 |

### 关键约束
- **所有布局和功能由物模型属性驱动，APP 不做设备类型预判**
- **设备名称仅用于页面展示，不参与任何逻辑判断**
- **视频流地址、子设备列表均由物模型/后台下发**
