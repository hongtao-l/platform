# 物模型全链路流程 — 以「工作模式」为例

## 数据回顾

平台管理员定义的工作模式：

```json
{
  "capType": "prop",
  "name": "工作模式",
  "identifier": "WorkMode",
  "dataDef": {
    "dataType": "enum",
    "accessMode": "rw",
    "enumValues": [
      { "name": "低功耗模式", "val": 0 },
      { "name": "AOV模式", "val": 1 },
      { "name": "长电模式", "val": 2 },
      { "name": "自定义模式", "val": 3 }
    ],
    "defaultVal": "0"
  }
}
```

---

## 1. 设备添加 → 物模型下发

```mermaid
sequenceDiagram
    participant U as 👤 用户
    participant APP as 📱 APP
    participant Platform as 🖥️ 平台
    participant Device as 📷 设备

    U->>APP: 添加设备，扫描设备二维码/输入SN
    APP->>Platform: GET /thing-model/product/{productId}/capabilities
    Note over Platform: 根据产品ID查关联能力<br/>返回物模型 JSON
    Platform-->>APP: 返回能力列表（含 WorkMode dataDef）
    Note over APP: 本地缓存物模型

    APP->>Device: 配网 + 绑定请求
    Device-->>APP: 绑定成功

    Note over APP: 按物模型 defaultVal 初始化设备<br/>下发 {identifier:"WorkMode", value:"0"}
    APP->>Platform: 平台转发 → 设备
    Platform->>Device: 下发初始值 WorkMode=0
    Device->>Device: 写入默认值，切换为低功耗模式
    Device-->>Platform: 确认
    Platform-->>APP: 确认

    APP-->>U: 设备添加完成
```

> 设备没有预置状态，初始值由平台 `defaultVal` 决定，APP 在绑定后通过平台下发写入设备。

---

## 2. APP 控件动态加载

```mermaid
sequenceDiagram
    participant APP as 📱 APP
    participant Cache as 📦 本地物模型缓存

    APP->>Cache: 读取 WorkMode 的 dataDef
    Cache-->>APP: {dataType:"enum", enumValues:[...], accessMode:"rw", defaultVal:"0"}

    Note over APP: 判断渲染规则：
    Note over APP: dataType=enum, 4项 ≤5 → 分段选择器

    APP->>APP: 渲染分段选择器<br/>[低功耗模式 | AOV模式 | 长电模式 | 自定义模式]
    APP->>APP: 按 defaultVal="0" 默认选中「低功耗模式」
```

控件渲染决策树：

```
dataDef 解析
├── dataType = "enum"
│   ├── enumValues.length ≤ 5 → 分段选择器（Segmented Control）
│   │   选项: enumValues[i].name, 值: enumValues[i].val
│   └── enumValues.length > 5 → 下拉列表（Picker）
│
├── dataType = "int"
│   ├── 可选值 ≤ 6 → 下拉选择
│   ├── 可选值 > 6, max-min ≤ 100 → 滑动条（min/max/step/unit）
│   └── 可选值 > 6, max-min > 100 → 数字输入（Stepper）
│
├── dataType = "boolean" → 开关（Switch）
│   文案: trueLabel / falseLabel
│
└── accessMode = "r" → 上述控件全部置灰只读
```

## 3. 用户修改参数 → 设备生效

```mermaid
sequenceDiagram
    participant U as 👤 用户
    participant APP as 📱 APP
    participant Platform as 🖥️ 平台
    participant Device as 📷 设备

    U->>APP: 点击分段选择器「AOV模式」

    Note over APP: accessMode="rw", 允许操作
    APP->>APP: 本地乐观更新 UI<br/>高亮「AOV模式」

    APP->>Platform: POST /device/{deviceId}/property<br/>{identifier:"WorkMode", value:1}
    Platform->>Device: 转发属性设置指令

    Device->>Device: 切换工作模式为 AOV
    Device-->>Platform: 上报确认 {identifier:"WorkMode", value:1}
    Platform-->>APP: 推送确认 / APP 轮询结果

    APP->>APP: 确认 UI 状态一致
    APP-->>U: 显示「AOV模式」
```

### 异常处理

```mermaid
sequenceDiagram
    participant U as 👤 用户
    participant APP as 📱 APP
    participant Platform as 🖥️ 平台

    U->>APP: 点击「自定义模式」(val=3)
    APP->>APP: 乐观更新 UI

    APP->>Platform: POST {identifier:"WorkMode", value:3}
    Platform->>Platform: 设备离线 / 超时

    Platform-->>APP: 返回失败 {code: "DEVICE_OFFLINE", msg:"设备未响应"}

    APP->>APP: 回滚 UI 到修改前的值
    APP-->>U: Toast「设备未响应，请重试」
```

## 4. 设备主动上报（物理按键 / 自动切换）

```mermaid
sequenceDiagram
    participant Device as 📷 设备
    participant Platform as 🖥️ 平台
    participant APP as 📱 APP

    Note over Device: 用户按下设备物理按键<br/>切换为「长电模式」

    Device->>Platform: 主动上报 {identifier:"WorkMode", value:2}
    Platform->>APP: 推送属性变更通知

    Note over APP: 匹配本地物模型<br/>val=2 → enumValues[2].name="长电模式"

    APP->>APP: 更新分段选择器<br/>选中「长电模式」
```

## 5. 完整链路时序总图

```mermaid
sequenceDiagram
    participant Admin as 🖥️ 平台管理员
    participant Platform as 🖥️ 平台
    participant APP as 📱 APP
    participant Device as 📷 设备

    rect rgb(240, 248, 255)
        Note over Admin,Platform: ① 平台侧：能力池配置（开发阶段）
        Admin->>Platform: 创建能力模块「工作模式模块」
        Admin->>Platform: 添加能力 WorkMode（enum, 4项, defaultVal=0）
        Admin->>Platform: 导入到对应类目（类目→产品→设备）
    end

    rect rgb(255, 248, 240)
        Note over APP,Device: ② 设备添加（用户操作）
        APP->>Platform: 请求产品物模型 GET /product/{productId}/capabilities
        Platform-->>APP: 返回 WorkMode dataDef（含 defaultVal）
        APP->>APP: 缓存物模型
        APP->>Device: 配网绑定
        APP->>Platform: 下发默认值 WorkMode=0 → 设备
        Platform->>Device: 写入 defaultVal
        Device->>Device: 初始化为「低功耗模式」
    end

    rect rgb(240, 255, 240)
        Note over APP: ③ APP 渲染（动态控件）
        APP->>APP: 解析 dataDef → enum 4项 → 分段选择器
        APP->>APP: 渲染 [低功耗|AOV|长电|自定义]
        APP->>APP: defaultVal=0 → 默认选中「低功耗模式」
    end

    rect rgb(255, 240, 240)
        Note over APP,Device: ④ 用户修改参数（APP→平台→设备）
        APP->>Platform: POST /device/{id}/property WorkMode=1
        Platform->>Device: 转发属性设置
        Device->>Device: 切换为 AOV模式
        Device-->>Platform: 确认
        Platform-->>APP: 推送结果
        APP->>APP: UI 更新为「AOV模式」
    end

    rect rgb(248, 240, 255)
        Note over Device,APP: ⑤ 设备主动上报（设备→平台→APP）
        Device->>Platform: 上报 WorkMode=2（长电模式）
        Platform->>APP: 推送属性变更
        APP->>APP: UI 更新为「长电模式」
    end
```

---

| 关键原则 | 说明 |
|------|------|
| 默认值由平台定义 | `defaultVal` 在平台配置能力时设定，设备绑定后由 APP 通过平台下发写入 |
| 按产品请求物模型 | APP 通过 `productId` 获取物模型，平台解析产品所属类目→关联能力 |
| APP 不直连设备 | 所有属性读写均走 **APP→平台→设备** 链路，平台统一鉴权、转发、记录 |
| 控件动态渲染 | APP 不硬编码任何控件，完全由 `dataDef` 决定渲染什么、如何渲染 |
