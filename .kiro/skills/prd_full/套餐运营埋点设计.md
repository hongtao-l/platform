## 设计原则

- 以**转化漏斗**为核心：曝光 → 点击 → 进入商城 → 选择套餐 → 发起支付 → 支付成功
- 每个事件携带足够的上下文属性，支持按策略、运营位、套餐、用户分群多维度分析
- 属性命名统一使用 snake_case

---

## 公共属性（所有事件均携带）

| 属性名 | 类型 | 说明 |
|---|---|---|
| app_id | string | 应用标识，如 APP_001 |
| user_id | string | 用户唯一标识 |
| device_id | string | 设备唯一标识 |
| user_group | string | 用户所属分群编码，如 user_group_high_value |
| strategy_id | string | 当前命中的套餐运营策略 ID |
| slot_code | string | 触发来源的运营位编码，如 pkg_slot_cloud_001 |
| platform | string | iOS / Android |
| app_version | string | App 版本号 |
| timestamp | long | 事件发生时间戳（ms） |

## 一、套餐运营弹窗埋点

### 1.1 弹窗曝光

**事件名：** `pkg_popup_show`

**触发时机：** 弹窗出现在用户屏幕上（可见）

| 属性名 | 类型 | 说明 |
|---|---|---|
| trigger_scene | string | 触发场景：sd_full（SD卡循环覆盖）/ cloud_full（云存循环覆盖）/ manual（主动进入商城） |
| strategy_type | string | 策略类型：static（静态指定）/ dynamic（动态匹配） |
| pkg_slot_count | int | 本次弹窗展示的套餐位数量（1/2/3） |
| pkg_list | array | 展示的套餐列表，每项包含 pkg_code、pkg_name、pkg_price、pkg_type、label（运营标签） |
| match_strategy | string | 动态匹配时的匹配策略：step_up / best_discount / history_prefer，静态时为空 |

---

### 1.2 弹窗关闭

**事件名：** `pkg_popup_close`

**触发时机：** 用户点击关闭按钮或点击弹窗外区域关闭

| 属性名 | 类型 | 说明 |
|---|---|---|
| close_type | string | 关闭方式：btn_close（点关闭按钮）/ mask_click（点遮罩）/ not_interested（不感兴趣） |
| view_duration_ms | int | 弹窗展示时长（ms） |
| pkg_list | array | 同曝光事件 |

---

### 1.3 套餐点击

**事件名：** `pkg_popup_pkg_click`

**触发时机：** 用户在弹窗中点击某个套餐卡片

| 属性名 | 类型 | 说明 |
|---|---|---|
| pkg_code | string | 被点击的套餐代码 |
| pkg_name | string | 套餐名称 |
| pkg_price | string | 套餐价格 |
| pkg_type | string | 套餐类型：cloud_storage / ai_service |
| pkg_position | int | 套餐所在位置（1/2/3号位） |
| label | string | 运营标签：discount / new_user / free_trial，无标签为空 |

---

### 1.4 立即开通点击

**事件名：** `pkg_popup_buy_click`

**触发时机：** 用户点击弹窗中的【立即开通】按钮

| 属性名 | 类型 | 说明 |
|---|---|---|
| pkg_code | string | 选中的套餐代码 |
| pkg_name | string | 套餐名称 |
| pkg_price | string | 套餐价格 |
| pkg_type | string | 套餐类型 |
| pkg_position | int | 套餐位置 |

## 四、转化漏斗模型

```
弹窗曝光 (pkg_popup_show)
    ↓ 漏斗1：弹窗点击率 = 套餐点击 / 弹窗曝光
套餐点击 (pkg_popup_pkg_click)
    ↓ 漏斗2：开通意向率 = 立即开通点击 / 套餐点击
立即开通点击 (pkg_popup_buy_click)
    ↓ 漏斗3：支付发起率 = 发起支付 / 立即开通点击
发起支付 (pay_submit)
    ↓ 漏斗4：支付成功率 = 支付成功 / 发起支付
支付成功 (pay_result: success)
```

## 五、核心分析指标

| 指标 | 计算方式 | 分析维度 |
|---|---|---|
| 弹窗触达率 | 弹窗曝光 UV / 目标分群 UV | 策略、运营位、触发场景 |
| 弹窗点击率（CTR） | 套餐点击 UV / 弹窗曝光 UV | 策略、套餐位、套餐类型、运营标签 |
| 弹窗转化率 | 支付成功 UV / 弹窗曝光 UV | 策略、运营位、用户分群 |
| 套餐位效果对比 | 各号位 CTR 对比 | 1号位 vs 2号位 vs 3号位 |
| 运营标签效果 | 各标签 CTR 对比 | 折扣 vs 新用户 vs 免费试用 |
| 关闭原因分布 | 各 close_type 占比 | 策略、触发场景 |
| 支付成功率 | 支付成功 / 发起支付 | 支付方式、套餐价格段 |
| 策略 ROI | 支付金额 / 策略覆盖用户数 | 策略 ID |

## 六、平台端数据展示建议

生态平台套餐运营策略列表中，建议在每条策略后增加数据列：

| 字段 | 说明 |
|---|---|
| 近7日曝光 | pkg_popup_show 去重 UV |
| 近7日 CTR | 套餐点击 UV / 曝光 UV |
| 近7日转化率 | 支付成功 UV / 曝光 UV |
| 近7日支付金额 | pay_result(success) 的金额汇总 |
