## 业务背景

随着IoT设备品类的不断扩展，平台需要建立一套标准化的物模型体系，以支持不同类型设备的快速接入和功能配置。新生态物模型旨在为IPC监控类设备提供统一的功能点定义和配置能力，覆盖工作模式、录制模式、灯光模式等核心功能模块。

通过物模型的标准化定义，实现：
- 后台灵活配置设备功能参数
- APP端统一展示和交互逻辑
- 设备端标准化功能实现

## 名词解释

| 名词 | 说明 |
|------|------|
| 物模型 | 设备功能的数字化定义，包括属性、事件、服务 |
| 属性 | 设备的状态参数，可读取和设置 |
| 服务 | 设备可执行的指令，可调用 |
| 事件 | 设备主动上报的消息，可订阅 |
| 枚举型 | 数据类型之一，表示有限的离散值集合 |
| 数值型 | 数据类型之一，表示可配置范围的连续数值 |
| 布尔型 | 数据类型之一，表示true/false二值状态 |
| AOV模式 | Always On Video，低功耗常开视频模式 |
| 长电模式 | 设备持续供电的工作模式 |
| 事件录制 | 由事件（如移动侦测）触发的录制行为 |

## 详细功能说明

---

### 一、工作模式模块

---

#### 1.1 工作模式 (work_mode)

**功能标识**：`work_mode`

**类型**：属性

**数据类型**：枚举型

**功能描述**：控制设备的工作模式切换，支持低功耗模式、AOV模式、长电模式、自定义模式。

**枚举值定义**：

| 枚举值 | 描述 | 默认值 | 说明 |
|--------|------|--------|------|
| 0 | 低功耗模式 | 否 | 设备进入低功耗休眠状态 |
| 1 | AOV模式 | 否 | Always On Video，常开视频低功耗模式 |
| 2 | 长电模式 | 否 | 设备持续供电工作模式 |
| 3 | 自定义模式 | 否 | 用户自定义工作参数（不可设为默认） |

**后台配置项**：
- 枚举值启用/禁用
- 枚举值描述（APP显示文案）
- 默认值设置

**AOV模式扩展配置**：
- 低功耗模式电量阈值（%）
- 拍照帧率（秒/帧）

**长电模式扩展配置**：
- 低功耗模式电量阈值（%）

**APP端交互**：
- 展示为单选列表或卡片选择
- 选中后立即下发切换指令
- 切换成功后显示Toast提示

---

#### 1.2 AOV自动切换低功耗电量 (aov_auto_switch_power)

**功能标识**：`aov_auto_switch_power`

**类型**：属性

**数据类型**：数值型

**功能描述**：AOV模式下，当设备电量低于此阈值时，自动切换到低功耗模式。

**数值配置**：

| 配置项 | 值 |
|--------|-----|
| 最小值 | 0 |
| 最大值 | 100 |
| 步长 | 1 |
| 单位 | % |
| 默认值 | 30 |

**后台配置项**：
- 数值范围（最小值、最大值）
- 步长
- 单位
- 默认值

**APP端交互**：
- 展示为滑块或输入框
- 实时显示当前设置值
- 修改后需确认保存

---

**功能标识**：`aov_capture_frame_rate`

**类型**：属性

**数据类型**：数值型

**功能描述**：AOV模式下设备拍照的时间间隔，影响设备功耗和监控频率。

**数值配置**：

| 配置项 | 值 |
|--------|-----|
| 最小值 | 1 |
| 最大值 | 10 |
| 步长 | 1 |
| 单位 | 秒/帧 |
| 默认值 | 1 |

**后台配置项**：
- 数值范围（最小值、最大值）
- 步长
- 单位
- 默认值

**APP端交互**：
- 展示为下拉选择（1~10秒/帧）
- 或滑块选择
- 修改后需确认保存

#### 2.1 常规录制 (normal_record)

**功能标识**：`normal_record`

**类型**：属性

**数据类型**：枚举型

**功能描述**：设置设备的常规录制模式，支持全天录制、自定义时段录制、关闭录制。

**枚举值定义**：

| 枚举值 | 描述 | 默认值 | 说明 |
|--------|------|--------|------|
| 0 | 全天 | 否 | 24小时持续录制 |
| 1 | 自定义时段 | 否 | 用户指定时间段录制 |
| 2 | 关闭 | 否 | 不进行常规录制 |

**后台配置项**：
- 枚举值启用/禁用
- 枚举值描述（APP显示文案）
- 默认值设置

**APP端交互**：
- 展示为单选列表
- 选择"自定义时段"后弹出时间段设置
- 修改后需确认保存

---

#### 2.2 事件录制 (event_record)

**功能标识**：`event_record`

**类型**：属性

**数据类型**：布尔型

**功能描述**：控制是否开启事件触发录制功能。开启后，当检测到事件（如移动侦测）时自动开始录制。

**布尔值定义**：

| 值 | 描述 | 默认值 |
|----|------|--------|
| true | 开启 | 是 |
| false | 关闭 | 否 |

**后台配置项**：
- 默认值设置（true/false）

**APP端交互**：
- 展示为开关按钮
- 切换后立即生效
- 显示当前状态（开启/关闭）

---

**后台配置项**：
- 枚举值启用/禁用
- 枚举值描述（APP显示文案）

### B. APP端功能入口建议

建议在APP设置页新增"录制模式"菜单，统一管理以下功能：
- 常规录制
- 事件录制
- 录制媒体模式
- 录制清晰度
- 抓拍清晰度
- 抓拍间隔
- 事件录制时长
- 自动结束事件录制开关

工作模式和灯光模式可作为独立菜单项展示。

---

### C. 后台管理界面说明

后台产品配置页面支持：
1. 从标准功能点库添加物模型
2. 编辑物模型参数配置
3. 删除已添加的物模型
4. 按模块分类展示功能点列表

列表字段包括：序号、所属模块、功能名称、功能标识、数据类型、描述、操作。
