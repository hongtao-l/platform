# 工作流程指南（多角色协作模式）

## 触发关键词

当用户消息中包含 **「大需求」** 时，自动触发完整 5 角色工作流。

小需求（不含关键词）直接改代码，不走文档流程。

---

## 角色顺序

```
需求AI分析师 ──→ UI/UE AI设计师 ──→ 前端开发AI工程师 ──→ 运营AI经理 ──→ PRD生成AI工程师
```

各角色具体职责、输入输出、执行流程详见 `skills/` 目录：

| 顺序 | 角色 | 技能文件 |
|------|------|------|
| 1 | 需求AI分析师 | `skills/requirement-analyst/SKILL.md` |
| 2 | UI/UE AI设计师 | `skills/ui-designer/SKILL.md` |
| 3 | 前端开发AI工程师 | `skills/frontend-developer/SKILL.md` |
| 4 | 运营AI经理 | `skills/operations-manager/SKILL.md` |
| 5 | PRD生成AI工程师 | `skills/prd-generator/SKILL.md` |

---

## 产出物 & 目录结构

```
docs/prd_full/{需求名}/
├── {需求名}01ReqAnalysis.md    ← 角色1：需求分析
├── {需求名}02UIDesign.md       ← 角色2：UI 设计
├── {需求名}03PRD.md            ← 角色5：完整 PRD
└── {需求名}04bp.md             ← 角色4：运营分析（埋点 + 数据看板）
```

---

## 核心规则

1. **严格按顺序执行**，不得跳过任何阶段
2. **每阶段输出后必须等待用户确认**，才能进入下一阶段
   - ❌ 一口气执行完所有阶段再汇报
   - ✅ 每阶段产出后停下来，明确询问「是否确认，进入下一步？」
3. **角色间上下文隔离**：每个角色只关注自己的输入和输出
4. **修改时只针对用户指出的点**，不全部重做
5. **所有文档保存到对应目录**

---

## 大需求触发流程

```
用户消息包含「大需求」
    ↓
[1] 需求AI分析师 → 输出 01ReqAnalysis.md → 等待确认
    ↓
[2] UI/UE AI设计师 → 输出 02UIDesign.md → 等待确认
    ↓
[3] 前端开发AI工程师 → 开发代码 → 等待确认
    ↓
[4] 运营AI经理 → 输出 04bp.md（埋点 + 数据看板）→ 等待确认
    ↓
[5] PRD生成AI工程师 → 输出 03PRD.md → 完成
```
