# 本协议尽为小IPC产品考虑，仅实现设备配网、设备注册、实时视频点播、本地录制、录制回放、云存储、事件上报、本地抓图、双向语音对讲、设备控制、错误监测数据上报、P2P等功能。报文结构仍然采取之前的报文格式。

## 1. 报文结构定义

### 1.1 报文组成

报文由起始字、类型字、方法字、数据长度、加密方式标记字、保留字、数据组成。

**表1 报文结构**

| 组成 | 类型 | 说明 |
|---|---|---|
| 起始字 | 2字节 | 摄像头报文交互头为：`#$` |
| 类型字 | 1字节 | 用来区分报文的类型 |
| 方法字 | 1字节 | 用来区分信令方法 |
| 数据长度 | 2字节 | 网络字节序，报文长度（去除报文头后数据内容的长度） |
| 加密标记字 | 1字节 | 用来表示报文数据的加密模式 |
| 保留字 | 1字节 | =0，保留 |
| 数据 | 不定 | 数据内容长度可变，最长不超过65535字节 |

> **说明**：如果加密标记字不为0，则数据内容为加密后的数据，获取到数据后，需要根据加密标记字标记的加密模式进行对应的解密过程才能获取到原始数据。

### 1.2 类型字与方法字

用来区分报文所携带的数据类型与数据类型下对应的方法，定义见表1。
每个方法都有一个比当前指令方法在取值上比当前方法大一的请求回应。

### 1.3 加密模式定义

标明所携带数据的加密模式，加密类型定义如下表2
**表2 加密方式**
| 加密方式 | 取值 | 定义 |
|---|---|---|
| 无加密 | 00H | 非加密数据（媒体数据的P帧，不需加密的数据） |
| 轻加密 | 30H | 简单的BLOWFISH轻量级加密模式 |
| 安全加密 | 31H | AES安全级加密模式 |

## 2. 设备注册、登陆和绑定
设备通过这些指令，取得与服务器的连接，设备被添加绑定的通知，以及连接的维护，低功耗下设备进入休眠状态的通知等。
### 2.1 通过顶级域名获取子域名
设备在产测阶段没有烧录子域名，则设备启动后，通过顶级域名获取子域名。可以通过https也可以通过http获取，针对特小系统不支持tls加密的系统：
通用顶级域名：https://topsign.smartcloudcon.com
伊朗地区顶级域名：https://topsign-ir.smartcloudcon.com
生产系统域名：https://factorysys.smartcloudcon.com
获取子域名请求url后缀： /Sign/GetSubCloudDomains
请求报文：
{
    "METHOD":"3228",
    "SEQID":"10001",
    "BODY":{
        "CompanyID":"企业ID",
        "AppID":"产品ID",
        "SDKVersion":"根据SDK版本号分配不同版本的接入服务",
        "CTEI":" 设备授权码",
        "CountryID":"国家码"
    }
}
回应：
{
    "METHOD":"3229",
    "SEQID":"10001",
    "CODE":"0",
    "BODY":{
        "CompanyID":"企业ID",
        "AppID":"产品ID",
        "Subdomains":"子云域名，https://xxxxxxx.com"   
    }
}

{

### 2.7 设备配网成功后，登陆绑定通知 @修改消息实现机制
设备在配网过程中，获取到app绑定设备信息；服务器需要判定设备是否被绑定，若被绑定则返回之前绑定的GID，且通知绑定码关联的用户设备被绑定；若未被绑定，服务端则完成设备绑定，并且返回绑定的GID，同时通知绑定的用户，设备添加绑定成功。
设备通知服务器报文：
{
    "METHOD":"321C",
    "SEQID":"10001",
    "BODY":{
        "BindCode":"绑定码",// 填用户的usertoken
        "GID":"带有GID（可以没有）,ap 配网时候可能携带"
    }
}
回应：
{
    "METHOD":"321D",
    "SEQID":"10001",
    "CODE":"0",
    "BODY":{
        "GID":"设备加入的GID"
    }
}

### 2.8 设备长连接心跳包
链路检测包没有报文内容，只有消息头。
设备请求包头类型字和方法字： 0000；
服务器响应的类型子和方法子：FFFF；

### 2.9 休眠设备唤醒通知
服务器唤醒休眠设备的唤醒包定义。
服务器休眠唤醒设备类型字与方法字分别为： FF00
完整消息： 0x23 0x24 0xff 0x00 0x00 0x00 0x00 0x00

### 2.10 休眠设备心跳检测
设备休眠后，会跟服务器保留一条TCP的长链路来维持心跳，需要发送链路检测包来维护心跳。
链路检测包没有报文内容只有消息头。
链路检测请求包类型字和方法字为：00FF
完整消息： 0x23 0x24 0x00 0xff 0x00 0x00 0x00 0x00

### 14.1 设备侧广播

1. 设备充电入网后，会发送广播消息，只有已经注册激活的设备即拥有设备ID的设备。携带设备ID和设备本地侦听的TCP地址；秘钥和秘钥向量是：“hellonjhemeng123”。
设备广播报文格式：
{
    "METHOD": "3122",
    "SEQID":"10001",
    "BODY":{
      "DID":"设备ID",
      "TCPPort":"本地监听的TCP端口"
    }
}

2. 用户端收到局域网UDP广播信令3112后，信令中携带设备的DID和设备侧的TCP端口，用户和设备建立TCP链接；使用DID作为加密的密钥和向量，发送用户连接报文给设备；
用户连接设备发送的报文格式：
{
    "METHOD":"3114",
    "SEQID":"10001",
    "BODY":{
        "UserToken":"用户的token",
        "SDKVersion":"用户SDK版本号0xAABBCCDD来表示",
        "OsType":"设备系统类型"
    }
}

3. 设备收到报文后，用本地DID解密信令。使用接收到的UserToken作为加密密钥和向量，发送设备相关信息给用户；
设备发送给用户报文格式：
{
    "METHOD":"3115",
    "SEQID":"10001",
    "CODE":"0",
    "BODY":{
        "DID":"设备ID",
        "CTEI":"设备的授权码",
        "GID":"设备GID,若没有则不发送该项",
        "SDKVersion":"设备SDK版本号0xAABBCCDD来表示",
        "OsType":"APP系统类型",
        "EncType":"加密类型",
        "EncKey":"加密Key",
        "EncLoad":"加密向量"
    }
}

4. 用户收到报文后，用本地UserToken解密信令。判断GID，若为空，则表示为新设备，回调给app上层，发现新设备；若不为空，则判断该GID是否在用户GID列表中，若在，则用设备传输过来的加密key和加密向量跟设备在该连接中发送指令；若不在，则关闭连接；

### 14.10 终端NAT类型检测请求
获取到NAT检测服务后，向NAT检测服务端口1和端口2都发送检测请求。NAT检测服务收到该请求后，将终端的外网IP/PORT信息返回给终端。检测指令需要携带检测服务的SvrToken用来校验有效性。更多的情况下，需要用检测服务的密钥对信令进行数据加密。
终端收到两次检测的回应后，判断NAT IP是否一致，NAT PORT是否一致？如果不是一致，则终端为Symetric对称性的NAT。若一致则为Cone锥形的NAT，此时可以上报自身的NAT类型为Port Restrict Cone。然后进一步检测Cone的类型。
请求参数：
{

	"SEQID":"10001",

### 14.22 媒体播放中vod定义
当vod值为1和2 时，Param值：1：视频；2：音频；3：音视频，表示对应的媒体暂停和恢复
当vod值为4、5、6时： Param值无意义，默认为0
当vod值为7时：Param值：统计的TTL值
当vod值为8时： Scale、Speed、SeekTime有效
当vod值为9时：Param值： 0： 关闭坐标功能；1：开启坐标功能
请求参数：
{
	"METHOD":"211C",
	"SEQID":"10001",
	"BODY": {
		  "ChannelID":"通道ID",
		  "Vod":"1:暂停；2：恢复；3：拖动；4：强制I帧；5：开始发送；6：发送完成；7：ttl统计；8：快进；9：人形坐标",
		  "Param":"根据Vod不同，取值不同",
         "Scale":"大于等于0，表示正序；小于0 表示倒续",
         "Speed":"0.5，1，2，4，8倍速度",
         "SeekTime":"当前app缓存帧的时间戳"
	}
}
回应
{
    "METHOD":"211D",
	"SEQID":"10001",
	"CODE":"0",
	"BODY": {
		"ChannelID":"点播通道号，表示同意，即开始传送"，
       "Vod":"",
       "Param":"",
	}
}

### 14.23 关闭当前点播请求
当前用户退出点播时，可以关闭当前点播请求。或者设备收到的ChannelID,在本地找不到时，关闭异常数据的ChannelID。设备收到该指令，找到用户下对应的ChannelID,销毁掉。设备端也可以关闭通道，停止发送所有数据；
当收到请求，寻找到该ChannelID，销毁当前ChannelID。
请求参数：
{
	"METHOD":"211E",
	"SEQID":"10001",
	"BODY": {
		"ChannelID":"通道ID",
		"Code":"关闭原因"
	}
}
回应：
{
	"METHOD":"211F",
	"SEQID":"10001",
	"CODE":"0",
}
